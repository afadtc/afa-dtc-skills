# 工作模式与模板

> 本文件为用户留存引擎的内部参考文件，用于沉淀本模块的工作模式、执行模板与交付骨架。
> **层级声明**：本文件默认作为 **internal-only** 的内部起草模板库使用，不得整段直接粘贴给用户。
> 如需整理为用户可见交付物，应删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
> **渲染规则**：本文件中的评分、行动数量、阶段数量、步数、ROI 预期和任何阈值，默认只可作为参考区间、保守起点或待验证计划；不得写成统一健康线、固定配方或对品牌结果的确定性承诺。

## 1. 触发条件与上下文契约 (Context Contract)

### 1.1 触发条件

afa-retain 在以下场景被 Hub 路由激活：

```
├── 用户提到"复购率""留存""流失""LTV""客户生命周期""忠诚度""会员"
├── 用户提到"订阅""Subscribe & Save""Dunning""催款"
├── 用户提到"召回""Win-back""沉睡客户""再激活"
├── Hub 诊断发现 LTV/CAC 失调，路由至此深挖
└── 用户要求"留存体检""客户分层""群组分析"
```

### 1.2 上下文契约（Hub 传入参数）

```
必需参数：
├── brand_name: 品牌名称
├── product_category: 产品品类（快消/服饰/耐用品）
├── monthly_orders: 月订单量（用于判断数据可用性）
└── business_stage: 业务阶段（启动期/成长期/成熟期）

可选参数（有则更精准）：
├── repurchase_rate_30d / repurchase_rate_90d: 复购率
├── ltv_cac_ratio: LTV:CAC 比率
├── churn_rate: 流失率（订阅制）
├── aov: 平均客单价
├── has_subscription: 是否有订阅模式
├── has_loyalty_program: 是否有忠诚度计划
├── seasonal_mode: off_season（触发淡季策略）
└── cohort_data: 群组分析数据（JSON/CSV）
```

## 2. 五大工作模式 (Work Modes)

### Mode 1: 全面留存体检 (Retention Audit)

```
触发：用户要求"留存体检""留存诊断""复购率分析"

SOP：
1. 收集五维健康数据（复购行为/客户价值/流失模式/互动健康/留存经济学）
2. 与品类基准对标（参考 benchmark-data.md）
3. 运行 RFM 分层（参考 rfm-ltv-framework.md）
4. 匹配诊断决策树（参考 diagnostic-system.md）
5. 按 ICE 或同等优先级框架排序，输出当前最值得优先执行的行动方案，数量按证据密度与执行容量确定

输出：留存体检报告（参考 report-templates.md 中的体检报告模板）
```

### Mode 2: 生命周期架构设计 (Lifecycle Architecture)

```
触发：用户要求"设计客户旅程""生命周期自动化""购后序列"

SOP：
1. 确认产品品类和消耗周期
2. 设计六阶段生命周期触点（参考 core-frameworks.md）
3. 为每个阶段定义触发器、内容框架和渠道
4. 设计跨渠道编排规则（Email/SMS/Push 优先级）
5. 输出完整的生命周期蓝图

输出：生命周期架构蓝图（参考 report-templates.md 中的蓝图模板）
```

### Mode 3: 忠诚度计划设计 (Loyalty Program Design)

```
触发：用户要求"设计会员体系""忠诚度计划""积分系统"

SOP：
1. 评估品牌适合的忠诚度模式（积分/等级/付费/混合）
2. 设计积分经济学（赚取率/兑换率/返现率）
3. 设计等级阶梯（门槛/权益/分布）
4. 规划非交易类赚取途径（评价/社媒/推荐）
5. 计算财务影响（积分负债/会员 LTV 提升预期）

输出：忠诚度计划设计方案（参考 loyalty-program-playbook.md）
```

### Mode 4: 流失预防与订阅优化 (Churn Prevention)

```
触发：用户提到"流失率高""订阅取消""Dunning"

SOP：
1. 拆解主动 vs 被动流失比例
2. 设计取消挽留流（动态路由）
3. 设计 Dunning 序列（步数按产品周期、失败原因与用户耐受度校准）
4. 建立流失风险评分模型
5. 输出完整的防流失方案

输出：流失预防方案（参考 subscription-management.md）
```

### Mode 5: 召回体系编排 (Win-back Orchestration)

```
触发：用户要求"召回流失客户""Win-back""再激活"

SOP：
1. 按 RFM 分层定义召回优先级
2. 设计分层召回序列（标准/VIP/订阅）
3. 定义每步的渠道、时机、Offer
4. 设计日落政策（Sunset Policy）
5. 评估召回 ROI 的参考方向与待验证假设

输出：召回体系方案（参考 win-back-workflows.md）
```

## 4. 淡季留存策略 (Off-Season Retention)

> **激活条件**：当 Hub 传入 `seasonal_mode = off_season` 时自动激活。

### 4.1 淡季留存三大原则

```
1. 存量变现优先于拉新
   → 获取新客成本通常高于留存老客
   → 淡季流量少 = 拉新效率低
   → 聚焦提升现有客户的复购率和 LTV

2. 关系深度优先于交易频次
   → 淡季不要频繁促销（会训练客户只等折扣）
   → 投资客户教育、社群建设、品牌忠诚
   → 目标：旺季来临时客户主动回来，而非被折扣吸引回来

3. 数据收集优先于短期收入
   → 淡季是收集客户反馈、NPS、偏好数据的最佳时机
   → 客户有更多时间回复调查和参与互动
   → 这些数据为旺季的个性化营销提供弹药
```

### 4.2 淡季留存策略矩阵

| 客户分层 | 淡季策略 | 目标 KPI |
|---------|---------|---------|
| **VIP（高价值客群）** | 专属预览、早鸟通知、1对1关怀 | 以高价值客群的复购与参与深度为主 |
| **活跃客户** | 教育内容、社群互动、忠诚积分加倍 | 以复购与参与质量的持续改善为主 |
| **沉睡客户（按品类周期定义）** | 温和再激活序列、个性化推荐 | 以再激活信号与回复质量为主 |
| **流失风险客户** | Win-back 序列、调查问卷、特别优惠 | 以挽回信号与风险下降趋势为主 |
| **新客（首购后）** | 加速第二次购买的培育序列 | 以二购推进与早期满意度改善为主 |

### 4.3 淡季忠诚度计划优化

```
1. 积分加倍活动
   → 淡季特定时段积分翻倍，激励淡季购买
   → 限定品类或限定时段，避免利润侵蚀

2. 非交易性积分获取
   → 写评价/社媒分享/推荐朋友/完成调查 = 积分
   → 目标：淡季不买也能互动

3. VIP 专属体验
   → 新品优先试用权、创始人 1 对 1 交流、幕后内容
   → 目标：让 VIP 感到被重视

4. 游戏化参与
   → 游戏化转盘、签到、任务挑战和徽章收集等元素可能改善转化与参与，但提升幅度应以真实测试验证
   → 淡季适合测试游戏化元素（低风险）
   → 每日签到、任务挑战、徽章收集
```

### 4.4 淡季 KPI 调整

| 维度 | 旺季 KPI | 淡季 KPI |
|------|---------|---------|
| **核心指标** | 复购率、LTV、客单价 | 客户参与度、NPS、留存率 |
| **邮件/SMS** | 转化率、Revenue per Send | 打开率、回复率、列表增长 |
| **忠诚度** | 积分兑换率、会员消费占比 | 积分获取率、会员活跃率 |
| **社群** | 社群驱动的收入 | 社群参与度、UGC 产出量 |
| **反馈** | CSAT（交易后） | NPS、产品反馈收集量 |
