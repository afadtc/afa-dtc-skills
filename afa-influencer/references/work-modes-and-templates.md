# 工作模式与输出模板
> 本文件为达人合作模块的内部参考文件，用于沉淀本模块的工作模式、KPI 体系与输出模板。
> **层级声明**：本文件默认作为 **internal-only** 的内部起草模板库使用，不得整段直接粘贴给用户。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
> **渲染规则**：本文件中的 KPI 基准、候选数量、触点数量、时间线和任何 ROI 预期，默认只可作为参考区间、保守起点或待验证计划；不得写成统一健康线、固定配方或对品牌结果的确定性承诺。

## 一、衡量与 KPI：数据驱动决策

> **核心目标**：建立清晰的指标体系，准确衡量网红营销和联盟计划的业务贡献。

### 1.1 核心指标仪表盘

| 指标类别 | 关键指标 (KPI) | 使用方式 | 业务意义 |
| :--- | :--- | :--- | :--- |
| **漏斗顶部 (曝光)** | 触达人数 (Reach) | 结合预算、平台分发特性和创作者层级观察 | 品牌知名度扩展 |
| | 每次展示成本 (CPM) | 作为效率参考，不套用统一价格线 | 曝光效率评估 |
| **漏斗中部 (互动)** | 参与度 (Engagement Rate) | 结合创作者体量、平台和内容类型判断 | 内容共鸣与受众质量 |
| | 每次互动成本 (CPE) | 作为效率参考，不写成固定好坏线 | 互动效率评估 |
| | 链接点击率 (CTR) | 结合落地页、受众意图和素材匹配判断 | 流量引导能力 |
| **漏斗底部 (转化)** | 转化率 (Conversion Rate) | 结合商品价格、创作者流量质量和归因方式解释 | 流量变现能力 |
| | 客户获取成本 (CAC) | 与其他渠道做可比口径对照，不预设统一结论 | 获客效率评估 |
| | 广告支出回报率 (ROAS) | 只作参考观察，不写成统一盈利线 | 整体盈利能力 |

### 1.2 ROI 计算模型

**基础 ROI 计算公式**：
`ROI = (活动带来的总收入 - 活动总成本) / 活动总成本 * 100%`

**成本构成 (Total Costs)**：
*   创作者固定费用 (Flat Fees)
*   联盟佣金支出 (Commissions Paid)
*   产品样品成本 (COGS of Gifted Products) + 运费
*   专属折扣码带来的利润折损 (Discount Cost)
*   平台或代理机构服务费 (Platform/Agency Fees)

**收入归因 (Total Revenue)**：
*   **直接归因**：通过专属联盟链接或折扣码追踪到的直接销售额。
*   **间接归因 (光环效应)**：活动期间品牌词搜索量提升带来的自然流量销售增长；通过白名单投放 (Whitelisting) 带来的额外销售额。

## 二、工作模式 (Work Modes)

afa-influencer 提供 6 种标准化的工作模式，以应对不同的业务场景。

### 模式 1：[DISCOVER] 创作者发现与筛选模式
*   **触发条件**：用户需要寻找特定类型的网红进行合作，或表达"帮我找网红""推荐创作者""谁适合推广我的产品"等意图。
*   **输入要求**：目标市场、产品品类、预算范围、期望的创作者层级（如微型、头部）。
*   **执行流程**：
    1.  从用户输入中提取品牌、产品、目标受众、预算等关键信息。
    2.  定义理想创作者画像 (ICP)：内容风格、受众重叠度、平台偏好和适合当前预算与目标的体量层级。
    3.  应用 3C 框架设定筛选标准：内容质量 (Content)、社区健康度 (Community)、转化潜力 (Conversion)。
    4.  对每个候选人进行假粉检测、参与度审计和历史合作记录检查。
    5.  输出包含足够支持当前阶段决策的结构化短名单，按综合匹配度排序。
    6.  为每个候选人提供建议的合作模式和预估报价范围。
*   **输出格式**：`creator-shortlist.md` 模板。
*   **质量检查**：每个候选人必须包含完整的 3C 评估，禁止仅列出名字和粉丝数。

### 模式 2：[OUTREACH] 冷拓展与邀约模式
*   **触发条件**：用户需要联系已选定的创作者，或表达"帮我写邮件""怎么联系网红""提高回复率"等意图。
*   **输入要求**：创作者背景信息、品牌能提供的价值 (Offer)、合作目标。
*   **执行流程**：
    1.  研究创作者的最新内容，提取个性化钩子（具体作品、观点、成就）。
    2.  根据产品特性选择最佳文案框架：解决痛点用 PAS，强调转变用 BAB，需要强钩子用 AIDA。
    3.  撰写高度个性化的初始触达邮件/私信，长度按渠道语境和沟通目标控制。
    4.  设计适合当前合作目标的跟进序列，每个触点提供新价值。
    5.  为最后一封邮件设计"分手邮件"策略，礼貌结束并留下未来合作的可能性。
    6.  根据创作者体量、联系方式和合作复杂度推荐最佳触达渠道，不固化为单一路径。
*   **输出格式**：`outreach-sequence.md` 模板。
*   **质量检查**：每封邮件必须包含至少 1 个个性化元素，禁止生成群发模板。

### 模式 3：[BRIEF] 创作者内容简报模式
*   **触发条件**：合作已确认，需要向创作者下发内容制作指南。
*   **输入要求**：产品核心卖点、品牌语调、必须包含的元素、平台政策要求。
*   **执行流程**：
    1.  整合 Brand Brain 信息。
    2.  明确视觉要求、关键信息点和 CTA。
    3.  加入广告披露提示和平台政策注意事项。
*   **输出格式**：`creator-brief.md` 模板。

### 模式 4：[AFFILIATE] 联盟计划设计模式
*   **触发条件**：品牌准备从零启动或优化现有的联盟营销计划。
*   **输入要求**：产品利润率、竞品联盟政策、目标受众。
*   **执行流程**：
    1.  设计具有竞争力的佣金结构（基础 + 阶梯）。
    2.  规划招募策略和入驻流程。
    3.  列出所需的素材清单。
*   **输出格式**：`affiliate-program-design.md` 模板。

### 模式 5：[DIAGNOSE] 渠道诊断与优化模式
*   **触发条件**：网红合作 ROI 不达标，联盟计划缺乏活力，或用户表达"网红合作效果不好""联盟计划没人参与"等意图。
*   **输入要求**：当前的核心数据（回复率、参与度、转化率、成本）。
*   **执行流程**：
    1.  收集用户提供的核心数据，与 benchmark-data.md 中的行业基准进行对标。
    2.  调用 6 大诊断决策树（见 diagnostic-system.md），逐一排查。
    3.  定位核心瓶颈（是选人问题、内容问题、转化问题还是运营问题）。
    4.  为每个问题提供具体的优化和恢复方案，包含阶段安排与待验证效果假设。
    5.  生成优先级排序的行动清单（Quick Win + 中期优化 + 长期战略）。
*   **输出格式**：诊断报告（包含问题摘要、根因分析、行动清单）。
*   **质量检查**：诊断报告必须包含数据对标，禁止无数据支撑的主观判断。

### 模式 6：[COMMUNITY] 社区飞轮与品牌倡导者培育模式
*   **触发条件**：品牌希望建立品牌大使项目、培育社区领袖、或将客户转化为品牌倡导者。
*   **输入要求**：品牌阶段、现有客户基础、社区目标。
*   **执行流程**：
    1.  评估品牌当前的社区成熟度（如无社区 / 初始社区 / 活跃社区等分层），并按实际情况校准分层规则。
    2.  应用社区参与阶梯模型，设计从潜水者到领导者的升级路径。
    3.  设计品牌大使项目的完整架构：招募标准、激励机制、沟通节奏、内容要求。
    4.  制定 UGC 规模化策略：品牌标签挑战、社区竞赛、再分享策略。
    5.  建立社区健康度监控指标。
*   **输出格式**：社区建设方案文档。
*   **质量检查**：方案必须包含可追踪的成功指标和阶段安排。

## 三、输出格式规范

### 3.1 创作者候选人短名单 (creator-shortlist.md)

```markdown
# 创作者候选人短名单：[项目名称]

## 核心筛选标准 (ICP)
- 目标受众：[描述]
- 垂直领域：[描述]
- 粉丝量级：[描述]

## 候选人推荐（按当前阶段列出必要候选人）

### 1. [创作者姓名/Handle]
- **平台链接**：[URL]
- **粉丝量**：[数量] | **预估参与度**：[百分比]
- **3C 评估**：
  - **Content (内容)**：[风格描述，品牌契合度分析]
  - **Community (社区)**：[受众画像，互动质量分析]
  - **Conversion (转化)**：[过往商业合作表现，带货潜力]
- **推荐理由**：[一句话总结为什么选择 TA]
- **建议合作模式**：[如：产品置换 / 专属折扣码 / 赞助视频]

### 2. [创作者姓名/Handle]
...
```

### 3.2 多触点跟进序列 (outreach-sequence.md)

```markdown
# 创作者触达序列：[项目名称]

## 核心 Offer 与价值主张
- [描述品牌能提供的价值]

## 触达序列蓝图

### 触点 1：初始邀约（首轮触达）
- **渠道**：[按创作者偏好选择，如 Email / Instagram DM]
- **主题行**：[引人注目的标题，避免营销感]
- **正文**：
  [个性化开场白，提及具体作品]
  [使用 PAS/BAB 框架引入产品和价值]
  [明确且低门槛的 CTA]

### 后续跟进：价值叠加与收口（按渠道节奏安排）
- **渠道**：[按触达反馈与可用联系路径选择]
- **正文**：
  [提供新信息，如媒体报道、其他创作者的成功案例]
  [在合适窗口内重申 CTA 或礼貌结束对话]
```

### 3.3 创作者内容简报 (creator-brief.md)

```markdown
# 创作者内容简报：[品牌名称] x [创作者名称]

## 1. 品牌与产品概览
- **品牌使命**：[一句话描述]
- **推广产品**：[产品名称及链接]
- **核心卖点 (USPs)**：
  1. [卖点 1]
  2. [卖点 2]

## 2. 创意指南 (Creative Direction)
- **内容格式**：[按平台和创意目标定义，如短视频、轮播图或混合素材]
- **视觉基调**：[如：自然光、生活方式、避免过度滤镜]
- **关键信息 (Talking Points)**：
  - [必须提及的信息]
  - [鼓励用自己的话表达的感受]
- **行动号召 (CTA)**：[明确指示用户点击链接或使用折扣码]

## 3. 注意事项与红线 (Do's & Don'ts)
- **必须做 (Do's)**：
  - 按目标平台与当地法规要求清晰完成广告披露（如 `#ad` 或 `#sponsored`）。
  - 在适合该创意的前提下展示产品实际使用过程。
- **绝对禁止 (Don'ts)**：
  - 避免引入会造成品牌混淆的竞争对手产品。
  - 禁止做出未经证实的健康或功效声明（如"100%治愈"）。
  - 避免使用过于生硬的电视购物式推销语气。

## 4. 交付与时间表
- **草稿审核截止日期**：[按合作节奏定义]
- **最终发布日期**：[按合作节奏定义]
- **专属链接/折扣码**：[Code]
```
