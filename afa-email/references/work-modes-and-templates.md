# 工作模式与模板

> 本文件为邮件策略模块的内部参考文件，用于沉淀本模块的工作模式、执行模板与交付骨架。
> **层级声明**：本文件默认作为 **internal-only** 的内部起草模板库使用，不得整段直接粘贴给用户。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
> **渲染规则**：本文件中的版本数量、发送节奏、KPI 目标、发送窗口、RPR 预期和任何假设值，默认只可作为参考区间、保守起点或待验证计划；不得写成统一健康线、固定配方或对品牌结果的确定性承诺。

## 1. 上下文契约 (Context Contract)

在启动任何邮件工作模式前，必须确认以下前置信息：

| 必需信息 | 说明 |
|:---|:---|
| ESP 平台 | Klaviyo / Mailchimp / Omnisend / 其他 |
| 当前列表规模 | 总订阅者数量 + 活跃比例 |
| 品牌阶段 | 冷启动 / 成长期 / 成熟期 |
| 品类 | 消耗品 / 耐用品 / 时尚 / 数字产品 |
| 当前痛点 | 用户自述的核心问题 |
| SMS 能力 | 是否启用 SMS 渠道（Klaviyo SMS / Postscript / Attentive / 无） |

如果用户未提供，主动询问。如果用户拒绝提供，先说明信息缺口并输出保守版建议；如必须使用假设，须明确标注为待验证假设，不把品类常见值写成品牌真实数据。

## 2. 四大工作模式

### Mode 1: /build-flow — 自动化流构建

**触发**：用户要求创建或优化某个自动化流。

**输出结构**：
1. 流程拓扑图（使用第 3 章的 Flow Blueprint 执行级语法）
2. 每封邮件的：标题（按当前测试需求提供足够的变体）+ 预览文本 + 正文框架 + CTA
3. 条件分支逻辑与抑制规则
4. SMS 层叠触点（如果品牌启用了 SMS）
5. KPI 目标与监控点

### Mode 2: /write-campaign — Campaign 文案撰写

**触发**：用户要求撰写单次发送的 Campaign 邮件。

**输出结构**：
1. 标题（按当前测试需求提供足够的变体，可包含 Safe / Bold / Personal 等风格）
2. 预览文本（与标题互补，不重复）
3. 正文（使用 PAS/BAB/AIDA 框架之一）
4. CTA 按钮文案
5. 发送建议：目标分段 + 最佳发送时间 + A/B 测试方案

### Mode 3: /fix-deliverability — 可交付性修复

**触发**：用户报告邮件进垃圾箱、打开率暴跌、被列入黑名单。

**输出结构**：
1. 12 步可交付性检查清单（逐项诊断）
2. 问题根因分析
3. 修复行动方案（按紧急度排序）
4. 预热/恢复计划（如需要）

### Mode 4: /plan-calendar — 邮件日历规划

**触发**：用户要求规划月度/季度邮件发送计划。

**输出结构**：
1. 月度日历视图（日期 × 类型 × 分段 × 主题）
2. Flow 与 Campaign 的协调逻辑
3. 频率治理规则（每周最大触达次数）
4. 关键节点提醒（BFCM、节假日等）

## 3. 输出模板

### 模板 A：自动化流设计文档 (Flow Blueprint) — 执行级拓扑

> **设计原则**：Flow Blueprint 必须达到"照图施工"精度——运营人员可以直接在 ESP 后台按照本文档搭建完整的自动化流，无需额外猜测任何参数。

#### A.1 Flow Blueprint 拓扑语法

每个 Flow Blueprint 必须使用以下标准化节点语法：

**触发节点 (Trigger)**：
```
[TRIGGER] 事件名称
  └─ 条件过滤：[具体过滤条件]
  └─ 排除条件：[正在经历哪些 Flow 的用户不进入]
```

**延迟节点 (Delay)**：
```
[WAIT] X 小时/天
  └─ 发送窗口：仅在收件人当地时间 [HH:MM]-[HH:MM] 发送
  └─ 智能发送：[是/否] — 如果启用，ESP 在用户历史最活跃时段发送
```

**条件分支节点 (Conditional Split)**：
```
[SPLIT] 条件名称
  ├─ YES (满足条件) → [下一节点]
  └─ NO (不满足条件) → [下一节点]
```

**邮件节点 (Email)**：
```
[EMAIL] 编号: 邮件名称
  └─ 标题变体：A (Safe) / B (Bold) / C (Personal)
  └─ 预览文本："..."
  └─ 正文框架：[PAS/BAB/AIDA] + 核心卖点
  └─ CTA："..." → [链接目标]
  └─ 动态内容块：[是/否] — 如果是，描述个性化逻辑
```

**SMS 节点 (SMS)**（仅当品牌启用 SMS 时）：
```
[SMS] 编号: 短信名称
      └─ 正文："..."（长度按渠道承载和合规要求控制）

  └─ 链接：[短链目标]
  └─ 发送窗口：仅在收件人当地时间 [HH:MM]-[HH:MM] 发送
```

### 2. 触发与入口条件

### 3. 流程拓扑

[TRIGGER] → [节点 1] → [节点 2] → ... → [EXIT]

| 点击率 | 结合品牌历史、名单质量与活动目标解释 | 自有历史或行业参考 |
| RPR | 结合品牌历史、名单质量与活动目标解释 | 自有历史或行业参考 |

- **优先级**：高优先（因收入回收价值高，但仍需结合当前 Flow 冲突关系判断）

### 2. 触发与入口条件
[TRIGGER] Started Checkout
  └─ 条件过滤：触发后未完成购买
  └─ 排除条件：
      • 正在经历 Welcome Series 的用户不进入（避免新客同时收到两条流）

  │   ▼
  │   [EMAIL] #1A: "Your cart is waiting" (高价值版)

## 4. KPI 体系

### 4.1 三维度 KPI 框架

**效率维度**：
- Revenue per Recipient (RPR)：结合 Flow 与 Campaign 类型、名单质量和品牌历史参考解释
- Click-to-Open Rate (CTOR)：结合品牌历史和创意强度参考解释
- Conversion Rate：结合流量质量、报价强度和活动类型参考解释
- List Growth Rate：结合获客结构和订阅入口健康度参考解释

**健康维度**：
- Spam Complaint Rate：按 ESP 警戒线、自有历史与市场法规共同判断
- Unsubscribe Rate：按发送场景、名单耐受度与近期频率共同判断
- Bounce Rate：区分 Hard / Soft，按平台健康阈值和自有历史判断
- Deliverability Rate：结合送达趋势、域名健康和列表质量判断

**营收维度**：
- Email Revenue Share：只作结构性观察，不设统一占比真理
- Flow vs Campaign Split：按品牌阶段、活动节奏和自动化成熟度判断
- LTV Impact：按订阅者与非订阅者长期价值差异观察，不写固定倍数
- Repeat Purchase Rate：结合品类复购周期和自有历史判断

## 5. 淡季邮件策略 (Off-Season / Q1 Detox)

BFCM 后的淡季阶段常被视为邮件营销的"排毒期"。用户可能经历过高频触达，因此应结合品牌真实发送历史、名单耐受度和当季目标调整策略，而不默认所有品牌处于同一状态。

### 5.1 核心原则
- **降低频率**：从高峰期频率回落到当前名单仍能承受的水平
- **转变内容比例**：减少纯促销内容，提升教育、故事和关系维护内容的占比
- **朋友式纯文本邮件**：可在合适窗口尝试纯文本创始人来信，并以自有数据验证其是否优于设计邮件

### 5.2 淡季策略矩阵

| 策略 | 目的 | 频率 |
|:---|:---|:---|
| 创始人来信 (纯文本) | 建立人格化连接 | 按品牌内容节奏安排 |
| 教育内容 (How-to) | 提供价值，维持互动 | 按内容产能和名单耐受度安排 |
| UGC 精选 | 社会认同 + 社区感 | 按素材储备和活动节奏安排 |
| 新品预告 | 维持期待感 | 按需 |
| FOMO 再激活 | 唤醒沉默用户 | 按沉默窗口定义、名单耐受度与当前商业优先级安排 |

### 5.3 淡季 KPI 调整
- 打开率目标可按名单质量和内容形态重新设定，不默认维持不变
- 转化率目标可按季节需求和购买意愿变化适度校准，不套用固定降幅
- 重点关注列表健康度指标（退订率、投诉率）
