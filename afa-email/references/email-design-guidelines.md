# 邮件设计规范指南

> **来源说明**：本文件为当前生效的模块参考资料，供执行时按需加载；用于指导 DTC 品牌邮件在移动端可读性、图文结构、视觉层级、无障碍体验与品牌一致性上的设计决策。若与当前模块 `SKILL.md` 或 `_system/` 全局协议冲突，以系统级规则为准。

---

## 1. 设计目标与使用原则

邮件设计的目标，不是把页面做得“像海报一样好看”，而是让用户**快速看懂、顺畅阅读、愿意点击，并且不会因为设计问题影响送达与转化**。在邮件场景里，视觉设计必须服务于信息传达、点击推进与品牌识别，而不能为了视觉效果牺牲可读性、可访问性或可交付性。

因此，邮件设计应同时满足五个目标：第一，移动端优先；第二，图文结构健康；第三，视觉引导明确；第四，无障碍可访问；第五，整体风格稳定一致。任何偏离这五项基础原则的设计，即使短期看起来更“吸睛”，长期也更容易带来删除、误触、低点击或送达风险。

| 设计维度 | 核心问题 | 设计目标 |
|---|---|---|
| 移动端可读性 | 用户能否在手机上快速读懂？ | 保证正文、按钮、段落结构清晰 |
| 图文比例 | 邮件是否过度依赖图片表达？ | 降低被判定为低质量邮件的风险 |
| 视觉层级 | 用户是否知道先看哪里、点哪里？ | 把视线自然引导到主 CTA |
| 无障碍体验 | 屏幕阅读器与弱视用户能否顺利理解？ | 提升普适可读性与合规性 |
| 品牌一致性 | 用户是否一眼认出这是品牌邮件？ | 稳定建立识别感与信任感 |

## 2. 移动端优先设计原则

大多数邮件会优先在手机端被打开，因此邮件设计默认应从移动端开始，而不是先做桌面版再被动压缩。只要邮件在手机上出现字号太小、按钮难点、段落过长、布局拥挤的问题，用户通常不会花时间适应，而是直接划走、删除，甚至降低对品牌邮件的后续打开意愿。

### 2.1 采用单栏主结构

在邮件场景中，多栏布局虽然在桌面端看起来更“丰富”，但在移动端极易出现缩放后难读、图片与文案错位、按钮过小等问题。因此，主内容区域应优先采用**单栏垂直结构**，确保信息按自然顺序向下展开，让用户无需放大即可完成阅读。

| 设计项 | 建议做法 | 原因 |
|---|---|---|
| 主内容布局 | 使用单栏纵向结构 | 更适合手机端阅读与滚动浏览 |
| 模块排列 | 一屏只强调一个主要信息点 | 降低认知负担，提升点击效率 |
| 内容节奏 | 标题 → 视觉 → 说明 → CTA | 符合用户扫描习惯 |

### 2.2 控制字号与段落长度

正文过小会迫使用户放大阅读，标题过小则会削弱信息层级。建议正文最小字号不低于 **16px**，标题建议不低于 **22px**。同时，段落应保持短句与短段落，避免在手机端形成大段密集文本墙。

| 元素 | 建议标准 | 说明 |
|---|---|---|
| 正文字号 | 至少 16px | 保证无需缩放即可阅读 |
| 标题字号 | 至少 22px | 建立清晰层级，便于快速扫读 |
| 段落长度 | 每段尽量控制在 2–3 句内 | 适合移动端分屏阅读 |

### 2.3 让 CTA 适合手指点击

按钮不是装饰，而是转化动作的核心触发点。CTA 如果过小、周围过挤，用户即使想点也可能误触或放弃。按钮建议高度不低于 **44px**，并保留足够留白，让用户在单手操作时也能顺畅点击。

| 设计项 | 建议标准 | 说明 |
|---|---|---|
| 按钮高度 | 至少 44px | 满足移动端点击友好性 |
| 按钮周围留白 | 充足留白，避免贴近其他元素 | 减少误触与视觉拥挤 |
| CTA 数量 | 一屏内优先只有一个主 CTA | 减少分散注意力 |

## 3. 图文比例与可交付性原则

邮件不是海报，也不是社媒图片合集。过度依赖图片会同时带来两个问题：一是邮箱客户端和垃圾过滤器无法像读取 HTML 文本那样稳定识别图片中的信息；二是当图片默认被拦截时，用户会看到大片空白，核心信息直接丢失。因此，设计必须保证**即使图片不显示，邮件仍然能传递核心意思**。

### 3.1 遵循图文 60/40 原则

默认建议将邮件控制在约 **60% 活文本（HTML 文本）+ 40% 图片** 的结构附近。这个比例不是机械模板，而是提醒设计者：**价值主张、主体文案与 CTA 必须主要依靠活文本表达，而不是写进图片里。**

| 项目 | 建议做法 | 风险提示 |
|---|---|---|
| 主标题 | 使用 HTML 活文本 | 不要把主标题只做成图片的一部分 |
| 正文说明 | 使用 HTML 活文本 | 便于阅读、索引与无障碍识别 |
| CTA 按钮 | 使用真实按钮或 HTML 文本按钮 | 避免把按钮画在图片里 |
| 图片作用 | 辅助展示氛围、产品与场景 | 不承担全部信息表达 |

### 3.2 避免“整封邮件就是一张图”

如果整封邮件只是由一张大图或多张切片图组成，即使视觉上统一，实际也极易被判为低质量内容。更重要的是，一旦用户邮箱默认屏蔽图片，邮件就会失去绝大部分信息价值。

因此，任何时候都不要把整封邮件的关键信息只存在一张主视觉里。图片可以增强表达，但**不能替代信息结构本身**。

### 3.3 每张图片都补全 Alt Text

Alt Text 不只是无障碍要求，也是图片加载失败时的最低保底信息。每张关键图片都应具备准确、简洁、能传达作用的 Alt Text，而不是空白占位。

| 图片类型 | Alt Text 写法建议 |
|---|---|
| 产品图 | 说明产品名称或使用场景 |
| Banner 图 | 概括活动主题或主价值主张 |
| Logo 图 | 标明品牌名称 |
| 功能性图片 | 说明其动作或意图，而不是只写“image” |

## 4. 视觉层级与阅读路径

高转化邮件的核心，不是塞入更多元素，而是让用户**一眼知道先看什么、再看什么、最后点哪里**。如果视觉层级混乱，用户就会在 Logo、主图、正文、多个按钮和次级链接之间来回跳转，最终失去行动意愿。

### 4.1 使用清晰的扫描路径

在英语语境下，用户通常按从左到右、从上到下的方式浏览内容，因此邮件结构通常适合遵循接近 **Z-Pattern** 的阅读路径：顶部识别品牌，中部承接核心价值，底部推动点击动作。

| 视觉区域 | 推荐内容 | 作用 |
|---|---|---|
| 左上 | Logo / 品牌识别 | 先确认发件品牌 |
| 右上 | 次级链接或“View in Browser” | 可选辅助入口，不抢主焦点 |
| 中部核心区 | 主图 + 主标题 | 承担最主要的信息传递 |
| 下方中心区 | 主 CTA 按钮 | 推动下一步动作 |

### 4.2 突出主 CTA，而不是让所有元素都抢眼

CTA 必须是整封邮件中最明确、最容易识别的动作区域。按钮颜色应与背景形成明显对比，并在页面中拥有足够视觉权重。如果标题、图片、折扣标语和多个按钮都同样“吵”，用户往往反而不会点击。

### 4.3 用留白提升聚焦，而不是浪费空间

留白不是空，而是帮助用户区分层级、停顿节奏和理解重点的设计工具。模块之间需要有足够间距，按钮周围也需要有呼吸空间。过度拥挤会让邮件看起来廉价、急躁、压迫感强，并降低阅读顺畅度。

## 5. 邮件无障碍规范（Accessibility / A11y）

无障碍不是“额外加分项”，而是现代邮件设计的基本要求。一个真正可用的邮件，必须让使用屏幕阅读器的用户、视力较弱的用户，以及在不同显示环境中的用户，都能稳定获得主要信息。

### 5.1 保证足够的文字与背景对比

正文文字与背景之间应具备足够对比度。对于普通文本，建议至少满足 **WCAG AA 4.5:1** 的对比要求。浅灰字配白底、浅色按钮配浅色背景，都属于常见但应避免的设计错误。

| 设计项 | 建议 |
|---|---|
| 正文字色 | 避免低对比浅灰色 |
| 按钮文字 | 保证按钮背景与文字有明显反差 |
| 次要说明 | 即使是辅助文字，也不能牺牲到难以辨认 |

### 5.2 使用语义清晰的 HTML 结构

即便最终视觉上看起来只是“排版”，底层结构仍应有清晰的标题层级，例如合理使用 `<h1>`、`<h2>`、`<h3>`。这不仅有助于屏幕阅读器理解内容顺序，也能让整封邮件在结构上更稳定。

### 5.3 链接文字要说明动作，不要只写“点击这里”

链接文案本身应说明点进去会看到什么，而不是只写模糊动作。例如，比起“Click Here”，更好的写法是让链接直接表达目的，如查看系列、阅读指南、进入新品页面等。这样既更有助于转化，也更利于无障碍阅读。

### 5.4 考虑 Dark Mode 下的显示效果

许多用户会在 Dark Mode 下阅读邮件。如果 Logo、图标或插图背景不透明，就很容易在深色背景中出现白块、锯齿或边缘异常。建议品牌 Logo 与关键图标尽量使用透明背景素材，并检查品牌色在 Dark Mode 下的反转效果是否仍然可读。

| 检查项 | 建议做法 |
|---|---|
| Logo / 图标背景 | 优先使用透明背景 PNG |
| 深色模式按钮 | 测试反差是否仍清晰 |
| 品牌主色 | 检查反转后是否仍保持识别与可读性 |

## 6. 品牌一致性规范

邮件不是一次性页面，而是品牌识别系统的一部分。用户每次打开邮件，都应当迅速感受到“这是同一个品牌在说话”。如果不同活动之间头图、字体、按钮风格、页脚结构完全失控，即使单封邮件局部表现不错，也会削弱整体信任感和品牌资产积累。

### 6.1 统一 Header 与 Footer

Header 和 Footer 是邮件系统里最稳定的识别锚点。Logo 位置、社媒链接、法律说明、退订入口等基础元素应长期保持一致，不要在不同活动中频繁变动结构。

| 区域 | 应保持一致的内容 |
|---|---|
| Header | Logo 位置、基础导航逻辑、顶部留白风格 |
| Footer | 社媒链接、品牌地址、法律文本、退订链接 |

### 6.2 字体选择以稳定兼容为先

邮件客户端对字体支持并不完全一致，因此即使品牌有自定义字体，也必须准备安全的回退字体。常见可用的 web-safe 字体包括 Arial、Helvetica、Georgia、Times New Roman 等。设计时应确保即便品牌字体没有加载，邮件依然不会在可读性和层级上明显失真。

### 6.3 颜色系统保持稳定，不为单次活动随意漂移

即使遇到单次 Campaign，也不要为了“新鲜感”随意引入与品牌无关的颜色。邮件中的主色、辅助色、强调色应始终服务于品牌识别，而不是让用户每次都像看到不同品牌的内容。

## 7. 执行检查清单

在正式发送前，应对设计稿或模板进行一次快速检查，确保没有为了追求视觉效果而破坏邮件的基础表现。

| 检查项 | 通过标准 |
|---|---|
| 移动端阅读 | 无需缩放即可阅读，结构清晰 |
| 正文字号 | 不低于 16px |
| 标题字号 | 不低于 22px |
| CTA 按钮 | 高度不低于 44px，易于点击 |
| 图文比例 | 主体信息由活文本承载，避免整封图片化 |
| Alt Text | 所有关键图片已填写 |
| 对比度 | 文本与背景对比足够清晰 |
| HTML 结构 | 标题层级与模块结构清晰 |
| Dark Mode | Logo、图标、按钮已做显示检查 |
| 品牌一致性 | Header、Footer、色彩与字体风格保持统一 |

邮件设计的最终标准，不是“看起来像不像广告图”，而是：**用户是否能轻松读懂、顺畅点击、稳定识别品牌，并在不同设备与显示环境下都获得一致体验。** 任何设计决策，只要同时提升了这四点，就更接近高质量邮件设计。
