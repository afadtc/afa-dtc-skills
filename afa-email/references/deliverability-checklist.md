# 邮件送达率检查清单

> 本文件为邮件营销模块的内部参考文件，用于沉淀提升邮件送达率所需的技术配置、名单管理、内容结构与发送节奏检查要点。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。

---

## 1. 使用原则

送达率（Deliverability）不是某一个按钮、某一个设置或某一次活动的结果，而是一套持续累积的系统表现。邮箱服务商不会只看你这一封邮件写得如何，而是综合判断：**你是谁、你有没有合法身份、你平时发给谁、这些人会不会互动、你的内容像不像垃圾邮件、你的发送节奏是否稳定。**

因此，送达率问题不能只靠“换标题”或“少写促销词”解决，而要从技术基础、名单健康、内容质量和发送纪律四个维度同时管理。下面这份清单可以作为日常发送前后的最低标准。

| 维度 | 主要问题 | 目标 |
|---|---|---|
| 技术身份 | 发信身份是否真实、可验证？ | 先解决“能不能被信任地接收” |
| 名单健康 | 发送对象是否愿意接收？ | 降低投诉、退信和低互动风险 |
| 内容与设计 | 邮件是否像正常商业邮件而非垃圾邮件？ | 提升可读性并降低过滤风险 |
| 发送节奏 | 发送量是否稳定、可预期？ | 避免突然异常峰值伤害域名/IP 信誉 |

## 2. 技术配置基础（Technical Setup）

如果技术身份没有建立好，后面的内容再优秀也可能直接进垃圾箱，甚至被接收服务器拒收。技术配置属于所有发送工作的底座，不可省略。

### 2.1 SPF

**SPF（Sender Policy Framework）** 的作用，是声明哪些服务器被授权代表你的域名发送邮件。没有 SPF，接收方很难判断这封邮件是不是“冒充你的域名发出的”。

| 检查项 | 合格标准 |
|---|---|
| SPF 已配置 | 发信域名有有效 SPF 记录 |
| 授权范围正确 | 包含当前 ESP / 发信服务商需要的发送源 |
| 未出现冲突 | 记录未因重复或错误拼接而失效 |

### 2.2 DKIM

**DKIM（DomainKeys Identified Mail）** 会为邮件添加加密签名，用于证明邮件在传输过程中未被篡改，也帮助接收方确认邮件确实由受信任的发信实体发出。

| 检查项 | 合格标准 |
|---|---|
| DKIM 已启用 | 当前发信服务商已完成 DKIM 签名配置 |
| 签名正常生效 | 发出邮件能被正确验证 |
| 与品牌域名一致 | 优先使用品牌自身域名体系，而非完全依赖共享域 |

### 2.3 DMARC

**DMARC（Domain-based Message Authentication, Reporting, and Conformance）** 用于告诉接收服务器：如果 SPF 或 DKIM 验证失败，应该如何处理这封邮件。它不仅是身份验证的延伸，也能帮助品牌监控域名被冒用的风险。

| 检查项 | 建议 |
|---|---|
| DMARC 已配置 | 应尽快完成配置，避免身份验证链路长期缺口 |
| 起始策略 | 可先从 `p=none` 开始观察与监控 |
| 目标策略 | 逐步过渡到 `p=quarantine` 或 `p=reject`，建立更完整防护 |

### 2.4 BIMI（可选但推荐）

**BIMI（Brand Indicators for Message Identification）** 可在部分支持的邮箱环境中展示已验证品牌 Logo，有助于提升品牌可信度与识别度。它不是送达率的根本变量，但在身份建设成熟后，通常是值得补齐的一环。

### 2.5 独立发信域名

大批量营销邮件不应使用共享免费域名，如 `@gmail.com` 或 `@yahoo.com`。应使用品牌自己的发信域名，例如 `hello@yourbrand.com`。这不仅是专业度问题，更关系到身份控制与信誉积累。

## 3. 名单健康与 Sunset Policy

送达率很大程度上取决于你发给谁。即使技术配置齐全，如果持续向长期不互动、无效、甚至不存在的地址发送邮件，接收方也会认定你的邮件对用户价值低，从而损害发信信誉。

### 3.1 Double Opt-In（建议）

Double Opt-In 的作用，是让用户订阅后再做一次确认。这样可以显著减少拼错邮箱、虚假订阅、机器人注册和 Spam Trap 混入名单的风险。对于长期健康运营来说，这一步通常非常值得。

### 3.2 Sunset Policy（关键）

Sunset Policy 指的是：**对于一段时间内完全不打开、不点击的订阅者，自动进入抑制或停发状态。** 继续向这些人发送邮件，只会不断拉低互动率，并增加垃圾邮件风险。

| 情况 | 建议动作 |
|---|---|
| 标准情况 | 对持续较长时间无互动的人群转入 Suppress 或 Sunset 管理 |
| 送达率较差时 | 可进一步缩短观察窗口，更早停止向无互动人群继续发送 |
| 特殊高价值人群 | 可先进入再唤醒流程，再决定是否彻底停发 |

### 3.3 Easy Unsubscribe

退订入口必须清晰、易见、低阻力。不要隐藏在长段法律文本里，也不要让用户经历多步跳转。对于 Gmail / Yahoo 等环境，一键退订（one-click unsubscribe）已经成为越来越重要的基础要求。

### 3.4 Bounce Management

| Bounce 类型 | 中文说明 | 处理建议 |
|---|---|---|
| Hard Bounce | 永久性无效地址，如邮箱不存在 | 应尽快移出活跃发送池 |
| Soft Bounce | 临时性问题，如邮箱已满、服务器异常 | 持续监控，若反复出现则移除 |

名单管理的底层逻辑非常简单：**宁可名单更小，也不要把大量无效或低质量地址继续留在活跃发送池里。**

## 4. 内容与设计对送达率的影响

送达率不仅看身份，也看邮件“像不像一封正常、有价值、用户可能愿意阅读的邮件”。如果内容高度像垃圾邮件，或结构让系统难以理解，就更容易被过滤。

### 4.1 图文比例

建议邮件保持相对健康的图文平衡，让核心信息主要由可解析文本承载，而不是完全依赖图片。完全由大图组成的邮件更容易被判为低质量，因为过滤系统难以理解图片中的实际内容。

| 做法 | 建议 |
|---|---|
| 主标题 | 用 HTML 活文本，而不是只放进图片里 |
| 正文说明 | 优先使用活文本承载核心信息 |
| CTA | 使用真实按钮或 HTML 文本按钮，不要画在图上 |
| 图片作用 | 用于增强表现，不负责承载全部信息 |

### 4.2 垃圾邮件触发词

不要过度依赖高压促销词和夸张格式，例如全大写、连续多个感叹号，以及过于典型的 Spam Trigger Words。常见风险表达包括：

`FREE` / `GUARANTEE` / `ACT NOW` / `WIN`

这些词不是绝对禁止，但如果与夸张承诺、紧迫感轰炸、全大写排版一起出现，风险会明显上升。

### 4.3 图片 Alt Text

所有关键图片都应填写清晰的 Alt Text。这不仅有助于无障碍体验，也有利于在图片无法加载时保留最低限度的信息表达，并向系统传递更完整的邮件结构信号。

### 4.4 HTML 质量

邮件代码必须干净、结构清晰、链接无损坏。过度嵌套、乱七八糟的格式、失效链接或奇怪脚本都会增加异常风险。若使用成熟 ESP（如 Klaviyo）构建邮件，通常能减少很多底层编码问题，但仍应在发送前做好预览与测试。

## 5. 发送节奏与域名/IP 预热

即使内容正常，如果发送节奏极不稳定，也可能损害送达率。邮箱服务商非常关注一个品牌是否突然从低量发送跃升到大规模群发，尤其是对长时间未发信后突然大规模重启的账号而言，这种风险更高。

### 5.1 保持发送量相对稳定

避免短期内出现巨大峰值。如果必须放量，应分批推进，而不是一次性把所有沉睡名单重新打满。

### 5.2 域名 / IP 预热

如果很久没有发送，或准备扩大发送规模，建议先从最活跃、最愿意互动的人群开始，逐步扩大范围。这样能向接收方释放更健康的早期互动信号。

| 预热顺序 | 建议 |
|---|---|
| 第一步 | 先发给最近高互动人群 |
| 第二步 | 再逐步加入中等互动人群 |
| 第三步 | 最后才考虑较久未互动人群，且应谨慎控制规模 |

## 6. 日常送达率检查清单

在常规发送前，可用下表快速自检：

| 检查项 | 通过标准 |
|---|---|
| SPF | 已正确配置并生效 |
| DKIM | 已签名并能被验证 |
| DMARC | 已配置，策略合理 |
| 发信域名 | 使用品牌独立域名 |
| Double Opt-In | 如条件允许，已启用 |
| Sunset Policy | 对长期不互动用户已停发或抑制 |
| 退订入口 | 清晰、可见、低阻力 |
| Bounce 管理 | Hard Bounce 已清理，Soft Bounce 已监控 |
| 图文比例 | 核心信息由文本承载，避免整封图片化 |
| 垃圾词风险 | 未使用过度夸张表达 |
| Alt Text | 关键图片已填写 |
| HTML 质量 | 无损坏链接、无异常格式 |
| 发送节奏 | 无突然异常峰值 |

---

## 7. 结论

送达率工作的本质，是让邮箱服务商逐渐形成一种稳定判断：**这个发件人身份真实、发送对象合理、内容正常、用户愿意互动。** 只要技术身份、名单纪律、内容质量与发送节奏同时在线，邮件进入主收件箱的概率就会长期向好的方向累积。

如果出现送达率恶化，不要只盯着主题行或正文措辞，而应回到整条链路排查：**域名身份是否完整、名单是否过脏、是否长时间给不互动人群群发、是否突然放量、内容是否过度促销化。** 只有把问题放回系统里看，送达率修复才会真正有效。
