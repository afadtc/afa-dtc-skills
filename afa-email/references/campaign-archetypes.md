# 邮件 Campaign 类型手册

> **来源说明**：本文件为当前生效的模块参考资料，供执行时按需加载；用于指导单次发送类邮件 Campaign（如 Newsletter、Sale、Announcement）的常见类型选择、节奏设计与结构搭建。若与当前模块 `SKILL.md` 或 `_system/` 全局协议冲突，以系统级规则为准。

---

## 1. 使用原则

单次发送类 Campaign 的核心，不是“这周再发一封邮件”，而是根据业务目标选择最匹配的发送类型。不同 Campaign 的价值并不相同：有些负责直接收单，有些负责教育培育，有些负责强化品牌关系，还有些负责为后续自动化流提供信号和转化助推。因此，在规划任何一次 Campaign 之前，先回答三个问题：**这封邮件要解决什么问题、应该发给谁、用户收到后最希望他做什么动作。**

如果这三点不清楚，即使邮件写得漂亮，也很容易出现目标模糊、节奏混乱、列表过度打扰的问题。相反，只要类型匹配正确，即便结构不复杂，也通常能取得更稳定的表现。

| Campaign 类型 | 主要目标 | 更适合的使用场景 |
|---|---|---|
| 新品发布型 | 制造期待、快速起量、完成首波销售 | 上新、联名、限量系列发布 |
| 限时促销型 | 快速拉收入、清库存、推动短期转化 | 节日促销、清仓、短窗口活动 |
| 深度教育型 | 建立认知、解释价值、减少购买阻力 | 新品教育、复杂品类、成分/技术解释 |
| 创始人来信型 | 建立情感连接、提升品牌信任 | 里程碑、品牌故事、重大变化说明 |
| UGC 精选型 | 提供社会认同、增强社区感 | 真实用户反馈、穿搭/使用场景展示 |
| 补货提醒型 | 推动复购、降低断货流失 | 消耗型产品、补货节奏清晰的品类 |

## 2. 新品发布型 Campaign

新品发布类邮件的目标不是单纯“宣布有新品”，而是通过连续节奏让用户从**被提醒 → 被种草 → 被触发购买**。这类 Campaign 更适合做成短序列，而不是只发一封“新品上线了”。如果产品本身具备明显新意、视觉冲击或限量属性，分阶段推进通常比一次性讲完全部信息更有效。

| 项目 | 建议 |
|---|---|
| **核心目标** | 制造期待感并推动首波销售 |
| **建议频率** | 视上新节奏而定，通常每月 1–2 次 |
| **建议受众** | 全名单可发，但需排除明显低互动人群；VIP 可提前触达 |
| **适用前提** | 产品确实“值得发布”，不是普通库存补充却硬包装成大事件 |

### 2.1 推荐结构

| 阶段 | 作用 | 中文策略说明 |
|---|---|---|
| 预热（T-3） | 先种下期待 | 只强调即将到来的变化或价值，不急于揭晓全貌 |
| 抢先访问（T-1） | 奖励高价值人群 | 向 VIP 或高互动人群提前开放，强化优待感 |
| 正式发布（Day 0） | 完成主信息曝光 | 视觉、核心卖点、购买理由和 CTA 一次说清 |
| 社会认同 / 深度解释（Day 2） | 增强转化信心 | 用评论、成分、功能细节、场景说明补足决策依据 |
| 最后提醒（Day 4） | 促成犹豫用户转化 | 若为限量或短期发售，强化稀缺与时间窗口 |

### 2.2 英文示例文案

以下内容属于可直接用于业务的消费者文案示例，因此保留英文：

| 场景 | 英文示例 |
|---|---|
| 预热主题 | `Something big is coming.` |
| VIP 抢先访问 | `You get first dibs.` |
| 教育型主题 | `Why everyone is loving [Product].` |
| 限量提醒 | `Last chance to get yours.` |
| CTA | `Shop the Launch` / `Get Early Access` |

## 3. 限时促销型 Campaign

限时促销类邮件适合在短时间内集中释放购买动能，用于快速拉动营收或清理库存。但这类邮件不能过度使用；如果品牌每隔几天就做一次“最后机会”，用户会迅速学会忽略紧迫感，甚至延后正常价格下单。

| 项目 | 建议 |
|---|---|
| **核心目标** | 快速回收收入、推动短窗口转化 |
| **建议频率** | 通常每 6–8 周一次，避免过密 |
| **建议受众** | 优先面向高互动人群、历史购买者、已验证促销响应人群 |
| **适用前提** | 促销理由明确、窗口真实、优惠表达清晰 |

### 3.1 推荐结构

| 阶段 | 作用 | 中文策略说明 |
|---|---|---|
| 宣布邮件 | 明确提出优惠与截止时间 | 标题直接、利益明确，不要绕弯 |
| 中段提醒 | 防止用户遗忘 | 重新唤起注意力，并用热卖品或畅销品增加说服力 |
| 最后数小时 | 强化最后行动窗口 | 可用更简洁、更直给的结构，推动立即点击 |

### 3.2 英文示例文案

| 场景 | 英文示例 |
|---|---|
| 宣布主题 | `48-Hour Flash Sale: 30% Off Sitewide.` |
| 提醒主题 | `Halfway there. Top picks are selling out.` |
| 最后召回 | `Last chance. Sale ends at midnight.` |
| CTA | `Shop the Sale` / `Grab the Deal` |

执行时要特别注意：紧迫感必须真实。如果活动实际上会被反复延长，就不要继续使用“今晚截止”“最后机会”作为核心话术，否则长期会损害信任。

## 4. 深度教育型 Campaign

深度教育型邮件适合那些购买决策需要解释、认知门槛较高、或品牌希望通过内容建立专业权威的场景。这类邮件不以“立刻成交”为唯一目标，而是通过提供有价值的信息，把用户从“感兴趣但不理解”推进到“理解并愿意相信”。

| 项目 | 建议 |
|---|---|
| **核心目标** | 建立专业性、培育潜在线索、提升后续转化质量 |
| **建议频率** | 每月 2–4 次，根据内容能力与受众耐受度调整 |
| **建议受众** | 可覆盖全名单，也可优先触达新品兴趣人群或浏览高频人群 |
| **适用前提** | 品类确实需要教育，且品牌能提供有价值而非空泛的信息 |

### 4.1 推荐结构

| 模块 | 作用 | 中文策略说明 |
|---|---|---|
| Hook | 抓住阅读意愿 | 从一个误区、反常识观点或真实痛点切入 |
| Lesson | 提供清晰解释 | 用 2–3 个短段落讲清关键概念，避免写成长文教材 |
| Tie-In | 与产品建立自然关联 | 说明品牌产品如何解决刚才提到的问题 |
| Soft CTA | 给予下一步动作 | 不强压成交，可引导去看详情页、博客或产品页 |

### 4.2 英文示例文案

| 场景 | 英文示例 |
|---|---|
| 教育型标题 | `What most people get wrong about [Topic]` |
| 软性 CTA | `Learn More` / `Read the Full Guide` / `Shop the Solution` |

教育型邮件最怕两种错误：第一，信息全是品牌自夸，没有真正解释问题；第二，前面讲得很长，最后却没有把内容与产品价值连起来。

## 5. 创始人来信型 Campaign

创始人来信的价值不在于“让老板出镜”，而在于通过更个人化、低营销感的表达，建立品牌温度与关系信任。这类邮件尤其适合品牌有明确故事、阶段性节点或重要变化需要说明时使用。

| 项目 | 建议 |
|---|---|
| **核心目标** | 增强品牌亲近感与信任感 |
| **建议频率** | 每季度一次，或在重大节点时触发 |
| **建议受众** | 全名单，尤其适合高价值客户与长期订阅者 |
| **适用前提** | 内容必须真实、具体、有情绪价值，不能写成官话公告 |

### 5.1 推荐结构

| 模块 | 作用 | 中文策略说明 |
|---|---|---|
| 形式 | 营造个人邮件感 | 采用简洁排版，避免过度设计 |
| 故事 | 提供真实连接点 | 可以是幕后经历、品牌转折、一次困难或一个重要节点 |
| 感谢 | 强化关系 | 真诚感谢用户的支持和陪伴 |
| 结尾动作 | 保持克制 | 不做硬性销售；如需放链接，宜用低压方式放在 P.S. |

### 5.2 英文示例文案

| 场景 | 英文示例 |
|---|---|
| 开场语气 | `I wanted to write to you personally today.` |
| 感谢表达 | `Thank you for being part of this journey.` |
| 柔和收尾 | `P.S. If you'd like to see what we've been working on, you can take a look here.` |

## 6. UGC 精选型 Campaign

UGC（用户生成内容）精选邮件的价值在于把“品牌自己说好”转变为“用户替品牌证明”。这类邮件非常适合解决信任、效果、风格、搭配或实际使用场景等问题，也能增强社区感和参与感。

| 项目 | 建议 |
|---|---|
| **核心目标** | 提供社会认同、增强真实感、推动转化 |
| **建议频率** | 每月 1 次左右 |
| **建议受众** | 全名单，特别适合浏览高频但转化不足的人群 |
| **适用前提** | 已积累足够质量的用户内容、评价或使用场景素材 |

### 6.1 推荐结构

| 模块 | 作用 | 中文策略说明 |
|---|---|---|
| 主题 | 设定聚焦场景 | 例如某款产品如何被搭配、真实用户如何使用、前后对比等 |
| 主体内容 | 展示真实用户素材 | 选 3–4 个高质量案例即可，不要堆满素材 |
| CTA | 承接转化或参与 | 可引导去购买、去看完整故事、或鼓励用户投稿 |

### 6.2 英文示例文案

| 场景 | 英文示例 |
|---|---|
| 主题文案 | `Real results from real customers.` |
| 参与邀请 | `Share your story for a chance to be featured.` |
| 转化 CTA | `Shop Their Favorites` / `See the Looks` |

## 7. 补货提醒型 Campaign（手动发送版）

对于消耗型产品，如果暂时没有完全自动化的 Replenishment Flow，手动补货提醒 Campaign 可以承担重要的复购触发作用。这类邮件不应被写成普通促销，而应围绕“你可能快用完了”“现在补货最方便”来设计。

| 项目 | 建议 |
|---|---|
| **核心目标** | 促进复购、减少断货流失 |
| **建议频率** | 按产品消耗周期发送 |
| **建议受众** | 已购买某产品且处于合理补货窗口的人群 |
| **适用前提** | 产品具有明确复购节奏，且可按购买时间进行分段 |

### 7.1 推荐结构

| 模块 | 作用 | 中文策略说明 |
|---|---|---|
| Nudge | 温和提醒库存可能见底 | 用“你可能快用完了”的语气开启，而非直接推销 |
| Convenience | 强调补货轻松 | 突出一键回购、快速下单、熟悉产品等便利性 |
| Incentive（可选） | 轻度推动行动 | 如有需要，可加入小额优惠或订阅优惠 |

### 7.2 英文示例文案

| 场景 | 英文示例 |
|---|---|
| 提醒主题 | `Running low on [Product]?` |
| CTA | `Restock Now` / `Reorder in Seconds` |
| 订阅引导 | `Subscribe & Save` |

## 8. 类型选择的执行建议

真正高效的 Campaign 系统，不是每周都靠“促销型邮件”硬推收入，而是让不同类型的 Campaign 各自承担不同职责。新品发布负责造势与起量，限时促销负责短期收单，教育型内容负责降低决策门槛，创始人来信负责建立关系，UGC 负责提供信任，补货提醒负责提升复购。只要这些类型协同得当，品牌的邮件体系就不会只剩下“打折”这一种表达方式。

| 当前业务目标 | 优先选择的 Campaign 类型 |
|---|---|
| 需要快速拉新产品销量 | 新品发布型 |
| 需要短期冲营收或清库存 | 限时促销型 |
| 品类复杂、教育成本高 | 深度教育型 |
| 想增强品牌关系与信任 | 创始人来信型 |
| 需要强化社会认同 | UGC 精选型 |
| 需要推动消耗型产品复购 | 补货提醒型 |

在实际执行中，可以同时运行多个类型，但不要在同一封邮件里同时承担太多角色。**一封邮件只服务一个主目标**，通常比“什么都想讲一点”的设计更有效。
