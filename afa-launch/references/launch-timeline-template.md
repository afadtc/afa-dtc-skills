# 启动时间线模板

> 本文档为上市启动流程的参考文件，提供从验证到优化的完整时间线模板。
> **使用边界**：面向用户展示的时间线、清单与责任栏位，必须使用业务角色、团队名称或渠道名称；内部模块代号与路由映射不得直接出现在用户交付版本中。如需内部执行映射，统一放在文末的内部编排说明中使用。

---

## 一、用户可见时间线模板

### 阶段 0：验证与准备（D-90 至 D-45）

#### Week 1-2：客户研究

| 日期 | 任务 | 负责 | 交付物 |

| D-90 | 定义目标客群画像初稿 | 品牌团队 | 客群画像文档 |

| D-82 至 D-76 | 执行访谈（每天 5-7 人） | 品牌团队 | 访谈录音 + 笔记 |
| D-75 | 整理访谈发现，提炼痛点优先级 | 上市负责人 | 痛点排序报告 |

#### Week 3-4：市场验证

| 日期 | 任务 | 负责 | 交付物 |
|------|------|------|--------|
| D-74 | 设计定量调研问卷 | 上市负责人 | 问卷链接 |
| D-73 至 D-67 | 发放问卷，收集 500+ 回复 | 品牌团队 | 原始数据 |
| D-66 | 分析问卷数据，量化支付意愿 | 上市负责人 | 支付意愿报告 |
| D-65 | Google Trends + Amazon 品类分析 | 市场分析负责人 | 市场规模评估 |
| D-64 | 竞品深度扫描 | 竞争情报负责人 | 竞品矩阵 |
| D-63 | 综合评估，输出 Go / No-Go 决策 | 上市负责人 | Go / No-Go 报告 |

#### Week 5-6：Beta 测试与 GTM 策略

| 日期 | 任务 | 负责 | 交付物 |
|------|------|------|--------|
| D-62 至 D-48 | Beta 测试（100-200 用户） | 品牌团队 | Beta 数据 |
| D-47 | Beta 测试报告 + NPS 评估 | 上市负责人 | Beta 报告 |
| D-46 | 制定 GTM 策略 + 假设清单 | 上市负责人 | GTM 一页纸 |
| D-45 | **里程碑检查点 1：阶段 0 完成** | 项目组 | 启动决策 |

#### 阶段 0 → 阶段 1 的 Go / No-Go 检查清单

```text
□ 痛点强度：>60% 用户主动提及相同痛点
□ 支付意愿：>40% 受访者愿意以目标价格购买
□ Beta NPS：≥ 40
□ 竞品空间：存在明确的未被满足的需求缺口
□ 搜索趋势：相关关键词搜索量呈上升趋势
□ GTM 策略已定稿，假设清单已确认
□ 产品已准备就绪（库存 / 供应链 / 网站）

判定：5/7 以上为 Go → 进入阶段 1
      3/7 以下为 Go → 返回产品优化
```

### 阶段 3：诊断与优化（D+15 至 D+30）

#### 周度节奏

| 周次 | 核心任务 | 决策点 |
|------|----------|--------|
| Week 3 | 更新 PMF 仪表盘 v2、创意焕新、受众扩展 | PMF 得分 ≥ 75 → 开始规模化 |
| Week 4 | 输出完整 PMF 评估报告、最终决策 | Scale / Pivot / Kill |

#### 阶段 3 最终决策检查清单

```text
Scale 条件（全部满足）：
□ PMF 得分 ≥ 75
□ 连续 2 周 CPA ≤ 目标
□ 至少 2 个渠道盈利
□ 创意可复制性得到验证
□ 复购信号出现（30 天复购率 > 15%）

Pivot 条件（满足 2+ 项）：
□ PMF 得分 40-74
□ 仅 1 个渠道有效
□ CPA 不稳定（周波动 > 30%）
□ 复购信号弱

Kill 条件（满足 2+ 项）：
□ PMF 得分 < 40
□ 所有渠道 CPA > 目标 200%
□ 网站 CVR 持续 < 1%
□ 无复购，无有机增长
□ 客户反馈普遍负面
```

## 二、内部编排说明（仅供系统使用，不直接进入用户交付物）

> 本节仅用于内部职责映射与路由参考。若生成对用户可见的路线图、甘特图、周计划或执行清单，必须只保留上方“用户可见时间线模板”的角色和渠道表达，不得直接暴露内部模块代号。

| 用户可见角色 / 任务类型 | 内部承接模块 |
|------|------|
| 上市负责人 | `afa-launch` |
| 竞争情报负责人 | `afa-compete` |
| Meta 广告负责人 | `afa-fb` |
| Google 广告负责人 | `afa-gg` |
| 邮件营销负责人 | `afa-email` |
| SMS 营销负责人 | `afa-sms` |
| 短视频运营负责人 | `afa-tt` |
| 转化负责人 | `afa-convert` |

> 若某次交付需要进一步细化执行负责人与审批链，可在用户版本中改写为品牌团队、增长团队、投放团队、CRM 团队或项目组，不得把内部模块 ID 直接放进表格单元格、标题、说明文字或责任字段中。
