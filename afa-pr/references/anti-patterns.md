# PR 反模式、边界处理与降级策略

## 1. 绝对禁止的操作

```
公关与声誉管理中的致命错误（零容忍）：
├── ❌ 危机中删除真实的负面评论或拉黑用户（恶意谩骂除外）
│   └── 掩盖问题会引发更大的"史翠珊效应"（Streisand effect）
├── ❌ 在 Pitch 邮件中群发（Bcc）数百名记者
│   └── 记者能看出来，这会永久损害品牌信誉
├── ❌ 未经明确授权将用户的有机 UGC 用于付费广告
│   └── 存在严重的法律风险和声誉反噬
├── ❌ 在危机声明中使用"如果有人感到被冒犯，我道歉"这种非道歉话术
│   └── 必须承担责任，而不是指责受众的感受
├── ❌ 承诺可以"保证"在特定顶级媒体（如福布斯）上获得报道
│   └── Earned Media 无法保证，只能提高概率
└── ❌ 在输出末尾添加任何个人推广信息
```

## 2. 常见错误模式

```
媒体关系错误：
├── ❌ Pitch 邮件全篇都在谈论产品有多好
│   └── ✅ 正确：谈论产品如何解决当前受众关心的问题（Relevance）
├── ❌ 寄送未经请求的昂贵样品
│   └── ✅ 正确：先用 CPR 邮件建立联系，记者感兴趣后再寄送
├── ❌ 获得报道后不进行复用
│   └── ✅ 正确：一次报道 = 社交帖子 + 邮件 Newsletter + 官网背书徽章

UGC 错误：
├── ❌ 认为 UGC 必须是专业级的高清视频
│   └── ✅ 正确：真实、略带粗糙感的手机实拍往往转化率更高
├── ❌ 只在产品页底部展示评价
│   └── ✅ 正确：将精选 UGC 放在首页、结账页和废弃购物车邮件中

危机处理错误：
├── ❌ 反应迟缓，等待内部层层审批
│   └── ✅ 正确：在 2-4 小时内给出初步回应，哪怕只是"我们正在调查"
├── ❌ 创始人隐身，让客服背锅
│   └── ✅ 正确：重大危机必须由最高负责人出面承担
```

## 3. 边界处理规则

```
当用户要求购买虚假评价或虚假媒体报道时：
├── 明确拒绝，解释这违反平台规则
├── 警告这会导致封店或长期的声誉毁灭
└── 提供正当的替代方案：通过激励机制获取真实 UGC，或使用 CPR 框架争取真实报道

当用户面临复杂的法律诉讼（如专利侵权被告）时：
├── 明确声明 afa-pr 只能提供基础的 DMCA/C&D 模板和声誉降温策略
├── 强烈建议用户立即咨询专业律师
└── 不提供具体的法律抗辩策略

当用户要求代发 Pitch 邮件时：
├── 说明 afa-pr 提供策略、模板和记者画像，但不直接操作用户的邮箱发送
├── 强调由创始人/品牌官方邮箱发送的回复率远高于第三方代发
```

## 4. 降级策略（PR 专有）

```text
Level 1（完整数据）：品牌故事 + 媒体关系数据 + 产品信息 + 新闻素材
  → 执行全维度 PR 策划，输出完整 Pitch 包 + 媒体列表 + 危机预案

Level 2（部分数据）：有品牌信息但无媒体关系数据
  → 基于品牌定位生成 PR 叙事角度和 Pitch 模板
  → 提供目标媒体画像指南，标注"需补充记者联系信息"

Level 3（最少数据）：仅有产品描述，无品牌故事
  → 输出品牌故事构建框架 + PR 角度发现清单
  → 提供 CPR 冷推模板（通用版），引导用户完善品牌叙事

终端无联网：
  → 基于用户描述生成 PR 策略框架和 Pitch 草稿
  → 输出离线可用的媒体套件模板和危机响应 SOP
```

## 5. Dropshipping 适配

```
当 supply_chain_mode = dropshipping 时：

  整体优先级调整：
    • PR 在 Dropshipping 阶段的整体优先级较低
    • 原因：PR 需要品牌故事和差异化，Dropshipping 早期通常缺乏这些
    • 温和提示：
      「PR 通常需要有品牌故事和差异化才能发挥最大效果。
      当前阶段建议优先把精力放在付费广告和社交媒体上。
      以下 PR 建议依然有效，你可以根据精力选择性执行。」

  提升优先级（如果用户坚持做 PR）：
    • 社交证明收集（客户评价、用户反馈）
    • 微型网红/KOL 合作（低成本曝光）
    • 产品评测博客/视频合作（产品验证）

  降低优先级（但仍然提供）：
    • 主流媒体关系建设（需要品牌故事）
    • 思想领导力内容（需要行业专业度）
    • 大规模 PR 活动（需要预算和品牌基础）
```

```
当 supply_chain_mode = wholesale 时：

  提升优先级：
    • B2B 行业媒体关系（最高优先级）：行业垂直媒体、贸易媒体、行业协会刊物
    • 行业展会 PR（高优先级）：围绕行业展会做 PR 活动，展示供应链能力
    • 客户案例研究（高优先级）：发布合作客户的成功案例，建立 B2B 信任

  降低优先级（但仍然提供）：
    • 消费者媒体关系（批发客户不读消费者媒体）
    • 网红合作（B2B 客户不受网红影响）
```

```
当 supply_chain_mode = manufacturing 时：

  提升优先级：
    • 品牌故事型 PR（最高优先级）：制造商有真实的品牌故事——工厂故事、工艺传承、质量承诺
    • 思想领导力内容（最高优先级）：制造商有行业专业度，适合做行业洞察型 PR
    • 主流媒体关系建设（高优先级）：制造商有独特的故事角度，更容易获得媒体关注

  降低优先级（但仍然提供）：
    • 低价促销型 PR（制造商品牌应保护价格形象）
```

```
当 supply_chain_mode = dtc 时：
  → 保持现有所有 PR 策略和优先级不变（DTC 是系统的默认设计目标）
```

## 6. 危机模式止血策略

```
季节性排除：
  如果 seasonal_mode = off_season（用户已确认当前处于季节性淡季）：
  → 不自动触发危机模式（淡季销量下降是正常的）
  → 除非用户同时有非季节性危机信号（现金流断裂、账户被封等）
  → 使用 YoY（同比）而非环比来评估业绩趋势

当 crisis_mode ≠ none 时（即 cash_crisis 或 pr_crisis），PR 策略应聚焦在「声誉保护」：
  ① 如果危机涉及公关问题（产品质量、客户投诉），立即启动危机公关 SOP
  ② 不建议大规模 PR 投入（危机期不应增加支出）
  ③ 可以做「品牌韧性故事」（将危机转化为正面叙事）
  ④ 利用现有媒体关系做低成本 PR（而不是新建关系）
  ⑤ 监控社交媒体负面言论（危机期更容易被放大）

重要补充：
  以上是「止血建议」的方向指引，不是「禁止用户做其他事」。
  如果用户在危机期坚持要做非止血类的事，尊重用户意愿，正常执行。
```
