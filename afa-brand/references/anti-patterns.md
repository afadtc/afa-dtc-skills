# 反模式与边界处理

## 一、绝对禁止的操作

```
品牌建设中的致命错误（零容忍）：
├── ❌ 抄袭竞品的品牌定位、声音或视觉
│   └── 品牌的核心价值在于独特性，抄袭 = 自杀
├── ❌ 在没有用户确认的情况下覆盖现有品牌档案
│   └── 品牌档案是积累的资产，必须确认后才能覆盖
├── ❌ 使用模糊的、不可执行的品牌描述
│   └── "专业、友好、创新"不是品牌声音定义
├── ❌ 忽视品牌一致性，为短期效果牺牲品牌
│   └── 品牌是长期投资，不一致会侵蚀品牌权益
├── ❌ 在品牌规范中使用主观描述而非具体规范
│   └── 不说"好看的蓝色"，说"#1A1A2E 深海蓝"
├── ❌ 为品牌创造虚假的故事或证据
│   └── 品牌故事必须基于真实经历，虚假 = 信任崩塌
├── ❌ 在输出末尾添加任何个人推广信息
└── ❌ 跳过声音测试循环直接保存声音档案
    └── 未经验证的声音档案会污染所有下游内容
```

## 二、常见错误模式

```
定位错误：
├── ❌ 试图定位给所有人
│   └── ✅ 正确：越具体的受众定位，品牌越有力量
├── ❌ 用功能而非转变来定位
│   └── ✅ 正确：客户买的不是产品，是更好的自己
├── ❌ 定位基于价格优势
│   └── ✅ 正确：价格不是定位，价值才是
├── ❌ 频繁更换定位
│   └── ✅ 正确：定位需要时间积累，至少 6-12 个月才能评估
└── ❌ 定位只存在于文档中
    └── ✅ 正确：定位必须贯穿所有触点和决策

声音错误：
├── ❌ 声音定义太模糊（"专业、友好"）
│   └── ✅ 正确：具体到词汇选择、句式模式、语调刻度
├── ❌ 声音在不同渠道完全不同
│   └── ✅ 正确：核心声音一致，平台适配是微调而非变脸
├── ❌ 模仿其他品牌的声音
│   └── ✅ 正确：可以受启发，但必须发展自己的独特声音
├── ❌ 忽视声音测试
│   └── ✅ 正确：每次构建声音后必须运行测试循环
└── ❌ 声音档案写完就不更新
    └── ✅ 正确：品牌进化，声音也应该定期审查和更新

视觉错误：
├── ❌ 使用太多颜色（超过 5 个品牌色）
│   └── ✅ 正确：1 主色 + 1-2 辅色 + 1 强调色 + 中性色
├── ❌ 使用太多字体（超过 2 个品牌字体）
│   └── ✅ 正确：1 标题字体 + 1 正文字体
├── ❌ 图像风格不一致
│   └── ✅ 正确：制定图像风格指南并严格执行
└── ❌ 视觉与定位不匹配
    └── ✅ 正确：视觉是定位的视觉翻译，必须一致
```

## 三、边界处理规则

```
当用户要求抄袭竞品品牌时：
├── 明确拒绝，解释风险（法律风险 + 品牌混淆 + 无法建立独特认知）
├── 提供替代方案：分析竞品的优点，然后用自己的方式表达
└── 示例："我理解你欣赏 [竞品] 的品牌，但直接模仿会让你永远活在他们的阴影下。
    让我分析他们做得好的地方，然后帮你找到属于你自己的独特表达。"

当用户提供的信息不足以完成品牌定位时：
├── 明确说明需要哪些信息
├── 提供获取信息的方法（竞品调研、客户访谈、市场数据）
├── 在信息有限的情况下，提供初步方向和假设
└── 标注哪些结论需要更多数据验证

当品牌建议与用户个人偏好冲突时：
├── 解释建议背后的策略原因
├── 展示数据或案例支持
├── 尊重用户的最终决定
└── 记录决策和原因，以便后续评估

当用户要求快速完成品牌建设时：
├── 说明品牌建设的合理时间线
├── 提供"最小可行品牌"（MVB）方案
│   ├── 必须有：定位声明 + 声音概要 + 基础色彩
│   ├── 应该有：完整声音档案 + 品牌故事
│   └── 可以后补：完整视觉规范 + 品牌规范文档
├── 建议分阶段完成，而非一次性做完
└── 强调：品牌是持续进化的，不是一次性项目

当品牌需要多语言/多市场适配时：
├── 核心品牌定位保持一致
├── 声音可以根据文化做适配（参考 afa-expand）
├── 视觉在不同市场可以有变体
├── 品牌故事的核心不变，表达方式可以本地化
└── 建议与 afa-expand 协同处理国际化品牌策略
```

## 四、品牌专有降级策略

当用户无法提供完整的品牌资料时，按以下层级降级：

```
Level 1: 有品牌基础资料（品牌故事、视觉素材、竞品信息）
  → 可以做完整的品牌审计和策略制定
  → 所有工作模式正常运行

Level 2: 只有品牌名称和产品信息
  → 可以做竞品对标分析（如果终端支持联网）
  → 可以做品牌定位框架建议
  → 可以做 Voice & Tone 初始化
  → 无法做品牌健康度评估（缺少历史数据）

Level 3: 只有品牌名称
  → 可以做行业品牌趋势分析
  → 可以提供品牌建设框架和方法论
  → 明确标注所有建议基于行业通用实践

终端无联网时：
  → 基于已有的 Brand Brain 数据和参考文件工作
  → 无法做竞品实时分析
  → 所有输出标注"基于离线数据，建议联网后更新"
```

### Level 0 边界

```
afa-brand 欢迎所有阶段的用户，包括 Level 0。
纯概念阶段的用户可以通过本模块：
✓ 做品牌定位探索（即使还没有产品）
✓ 做品牌名称和故事框架
✓ 做目标受众画像设计
✓ 做 Voice & Tone 初始化
✓ 做品牌审计和健康评估（如果用户要求）
温馨提示：品牌策略在有产品和市场数据后可以做得更精准。
如果你能提供更多信息，我可以给出更定制化的方案。
```

### 危机模式止血

```
季节性排除：
如果 seasonal_mode = off_season（用户已确认当前处于季节性淡季）：
→ 不自动触发危机模式（淡季销量下降是正常的）
→ 除非用户同时有非季节性危机信号（现金流断裂、账户被封等）
→ 使用 YoY（同比）而非环比来评估业绩趋势

当 crisis_mode ≠ none 时（即 cash_crisis 或 pr_crisis），品牌策略应聚焦在「品牌资产保护」：
① 不建议品牌重塑（危机期没有时间和资源）
② 保护现有品牌资产（不要因为清仓而损害品牌形象）
③ 如果需要清仓，建议用「限时专属」而不是「大清仓」的话术
④ 危机期的品牌沟通应该透明、真诚（而不是假装一切正常）
⑤ 可以做「品牌故事重新叙述」（将危机转化为品牌韧性故事）
重要补充：
以上是「止血建议」的方向指引，不是「禁止用户做其他事」。
如果用户在危机期坚持要做非止血类的事，尊重用户意愿，正常执行。
```

## 五、品牌记忆协议

```
品牌记忆目录结构：
brand-brain/
├── voice-and-tone.md      ← afa-brand 拥有（创建和更新）
├── positioning.md         ← afa-brand 拥有（创建和更新）
├── brand-story.md         ← afa-brand 拥有（创建和更新）
├── visual-identity.md     ← afa-brand 拥有（创建和更新）
├── brand-guidelines.md    ← afa-brand 拥有（创建和更新）
├── products.md            ← afa 管家拥有（afa-brand 只读）
├── audience.md            ← afa-brand 可写（受众画像）
├── competitors.md         ← afa-compete 拥有（afa-brand 只读）
├── learnings.jsonl           ← 追加写入（所有 Skill 共享）
└── assets.md              ← 追加写入（所有 Skill 共享）

读取规则：
├── 每次调用时检查 brand-brain/ 是否存在
├── 存在 → 加载相关文件，向用户展示已加载的上下文
├── 不存在 → 不报错，作为首次用户处理
├── 文件缺失 → 优雅降级，提示可运行哪个 Skill 补充
└── 数据过时 → 标记并询问用户是否需要更新

写入规则：
├── 档案文件（create-or-overwrite）：先读取现有内容 → 展示变更差异 → 确认后覆盖
├── 追加文件（append-only）：读取现有内容 → 在末尾追加 → 确认追加结果
├── 始终包含 "## Last Updated" 行（日期 + 技能名称）
└── 保持文件人类可读（非技术人员也能理解）
```

### 迭代检测

```
调用时的首要检查：

如果 brand-brain/ 中已存在品牌档案 → 进入更新模式：
├── 1. 读取现有档案
├── 2. 展示当前品牌概要：
│   ┌─────────────────────────────────────────┐
│   │  现有品牌档案                            │
│   │  最后更新：{date} by {skill}             │
│   │                                         │
│   │  定位：{positioning summary}             │
│   │  声音：{voice summary}                   │
│   │  故事：{story summary}                   │
│   │  视觉：{visual summary}                  │
│   └─────────────────────────────────────────┘
├── 3. 询问用户意图：
│   ① 微调某个维度（定位/声音/故事/视觉）
│   ② 基于新数据刷新（提供新的市场信息或竞品变化）
│   ③ 全面重建（从头开始）
│   ④ 品牌健康审计（检查一致性和差距）
└── 4. 根据选择进入对应工作流

如果 brand-brain/ 不存在或为空 → 进入构建模式：
├── 1. 询问用户当前阶段：
│   ① 从零开始（新品牌）→ 完整构建流程
│   ② 有现有品牌素材 → 提取 + 优化流程
│   ③ 给我你的网站 URL → 自动抓取 + 分析流程
│   ④ 品牌升级/重塑 → 审计现有 + 重建流程
└── 2. 根据选择进入对应工作模式
```

### 品牌规范文档前置条件检查

```
品牌规范文档生成流程：

前置条件检查：
├── voice-and-tone.md 存在？→ 必须（声音章节的基础）
├── positioning.md 存在？→ 必须（定位章节的基础）
├── brand-story.md 存在？→ 推荐（故事章节的基础）
├── visual-identity.md 存在？→ 推荐（视觉章节的基础）
└── 如果缺少必须文件 → 先运行对应工作模式生成

生成原则：
├── 所有规范必须足够具体（精确到色值、字号、字重）
├── 所有规范必须配有示例（正确 vs 错误）
├── 所有规范必须可执行（外部协作者能直接使用）
├── 避免主观描述（不说"好看"，说"使用 #1A1A2E 深蓝色"）
├── 包含"为什么"（每条规范背后的策略原因）
└── 保持人类可读（非设计师也能理解）
```
