# 品牌健康审计工具包

> 本文件提供品牌健康审计模式所需的完整审计方法论、评分标准和报告模板。

---

## 一、审计准备

### 1.1 审计范围定义

```
全面审计（推荐首次使用）：
├── 网站（首页、关于页、产品页、博客）
├── 社交媒体（所有活跃平台）
├── 邮件（近期代表性营销邮件）
├── 广告（当前投放的广告素材）
├── 包装（产品包装照片）
├── 客服（客服回复样本）
└── 预计时间：按触点数量与资料完整度灵活评估

快速审计（定期检查）：
├── 网站首页 + 代表性产品页
├── 最新若干社交媒体帖子
├── 最新若干营销邮件
└── 预计时间：按样本量与复杂度灵活评估

单维度审计（针对性检查）：
├── 只检查声音一致性
├── 只检查视觉一致性
├── 只检查定位传达
└── 预计时间：按单一维度复杂度灵活评估
```

### 1.2 审计基准加载

```
审计前宜优先加载的品牌基准：
├── voice-and-tone.md → 声音审计的基准
├── positioning.md → 定位审计的基准
├── visual-identity.md → 视觉审计的基准
├── brand-story.md → 故事审计的基准
└── brand-guidelines.md → 综合审计的基准

如果品牌基准不存在：
├── 基于网站首页和关于页面建立临时基准
├── 在审计报告中标注"基于推断的基准"
└── 建议先运行品牌定义流程，再进行正式审计
```

## 二、五维审计详细标准

| 检查项 | 评分标准 | 分值 |

| 关于页面 | 完整品牌故事 = 2分，基本信息 = 1分，空洞 = 0分 | /2 |
| 产品描述 | 反映品牌价值观 = 1分，纯功能 = 0.5分，无 = 0分 | /1 |

## 五、品牌健康评分体系

```
品牌健康总分计算：

总分 = Σ（各维度评分 × 对应权重）

说明：各维度权重应结合品牌阶段、当前问题集中区和触点重要性动态设定，避免写死统一配比。

评级标准：
├── 高分段：品牌一致性较强，主要触点较稳定
├── 中高分段：大部分触点一致，仍有局部偏差
├── 中分段：核心触点基本一致，但边缘触点存在偏差
├── 中低分段：存在较明显的不一致，需要系统性修复
└── 低分段：品牌认知混乱，建议优先补齐基准与核心触点

审计报告可视化输出格式：

  ┌─────────────────────────────────────────────────┐
  │  品牌健康审计报告                                │
  │  品牌：{brand_name}                              │
  │  日期：{date}                                    │
  │  审计人：品牌审计顾问                            │
  │                                                  │
  │  总分：{score}/10  评级：{rating}                 │
  │                                                  │
  │  ┌─────────────────────────────────────────┐     │
  │  │ 定位一致性    ████████░░  {score}/10    │     │
  │  │ 声音一致性    ██████░░░░  {score}/10    │     │
  │  │ 视觉一致性    █████████░  {score}/10    │     │
  │  │ 故事一致性    ████░░░░░░  {score}/10    │     │
  │  │ 触点一致性    ███████░░░  {score}/10    │     │
  │  └─────────────────────────────────────────┘     │
  │                                                  │
  │  关键发现：                                      │
  │  ├── 强项：{top strengths}                       │
  │  ├── 改进：{improvement areas}                   │
  │  └── 高优先级：{critical issues}                 │
  │                                                  │
  │  优先修复路线图：                                │
  │  ├── 近期：{quick wins}                          │
  │  ├── 中期：{medium fixes}                        │
  │  └── 长期：{strategic improvements}              │
  └─────────────────────────────────────────────────┘
```

## 六、品牌规范文档生成

```
完整品牌规范文档目录：

1. 品牌概述
   ├── 品牌使命与愿景
   ├── 品牌价值观
   ├── 品牌定位声明
   └── 品牌故事（精简版）

2. 品牌个性
   ├── 品牌原型
   ├── 核心人格特质
   └── 品牌"是/不是"对照表

3. 品牌声音
   ├── 声音概要
   ├── 语调光谱
   ├── 词汇指南（使用/避免）
   ├── 写作风格指南
   ├── 平台适配指南
   └── 示例（品牌内 vs 品牌外）

4. 视觉识别
   ├── Logo 使用规范
   │   ├── 主 Logo + 变体
   │   ├── 最小尺寸
   │   ├── 安全区域
   │   ├── 颜色变体（全彩/单色/反白）
   │   └── 禁止用法
   ├── 色彩系统
   │   ├── 主色 + 辅色 + 强调色 + 中性色
   │   ├── 色值（HEX/RGB/CMYK）
   │   └── 使用比例和场景
   ├── 字体系统
   │   ├── 标题字体 + 正文字体
   │   ├── 字体层级规范
   │   └── Web 字体 + 替代字体
   ├── 图像风格
   │   ├── 摄影指南
   │   ├── 插画指南（如适用）
   │   └── 图标风格
   └── 布局与间距
       ├── 网格系统
       ├── 间距规范
       └── 留白原则

5. 应用示例
   ├── 网站页面
   ├── 社交媒体
   ├── 邮件模板
   ├── 广告素材
   └── 包装设计

6. 做与不做
   ├── 视觉做与不做
   ├── 文案做与不做
   └── 常见错误示例

生成规则：
├── 前置条件：优先具备 voice-and-tone.md + positioning.md
├── 推荐：brand-story.md + visual-identity.md
├── 所有规范应足够具体（精确到色值、字号、字重）
├── 所有规范宜配有示例（正确 vs 错误）
├── 所有规范应保持可执行（外部协作者能直接使用）
├── 避免主观描述（不说"好看"，说"使用 #1A1A2E 深蓝色"）
├── 包含"为什么"（每条规范背后的策略原因）
└── 保持人类可读（非设计师也能理解）
```
