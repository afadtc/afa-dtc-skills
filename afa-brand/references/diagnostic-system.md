# 品牌诊断体系

> 核心理念：不同的品牌问题需要不同的诊断路径。先诊断，再处方。

## 诊断模式 1：品牌定位模糊

```
触发信号：
├── 用户说"不知道怎么描述我的品牌"
├── 用户说"客户不理解我们做什么"
├── 用户说"我们和竞品看起来一样"
├── 网站首页没有清晰的价值主张
└── 不同渠道的品牌描述不一致

诊断决策树：
├── 有没有明确的目标受众？
│   ├── 没有 → 先做受众定义（Step 1 of 定位画布）
│   └── 有 → 继续
├── 有没有清晰的差异化？
│   ├── 没有 → 运行竞品品牌分析 + 白空间识别
│   └── 有 → 继续
├── 差异化是否可信？
│   ├── 没有证据支撑 → 收集/创造差异化证据
│   └── 有证据 → 继续
├── 定位是否被一致地传达？
│   ├── 不一致 → 品牌一致性审计 + 统一定位表达
│   └── 一致 → 继续
└── 定位是否被目标受众理解？
    ├── 不理解 → 简化定位表达 + 测试
    └── 理解 → 定位没问题，问题在其他地方

处方：
├── 轻度（定位存在但不清晰）→ 定位声明优化 + 一致性检查
├── 中度（定位模糊或不差异化）→ 完整定位画布 + 竞品分析
└── 重度（没有定位或完全错误）→ 从零开始的完整品牌定位流程
```

## 诊断模式 2：品牌声音不一致

```
触发信号：
├── 用户说"我们的文案感觉不像一个人写的"
├── 用户说"不同渠道的语调差异很大"
├── 用户说"新的文案写手总是写不对"
├── 客户反馈"品牌感觉不真实"
└── 内容参与度低（可能是声音问题）

诊断决策树：
├── 有没有声音档案（voice-and-tone.md）？
│   ├── 没有 → 这就是问题。进入声音构建流程
│   └── 有 → 继续
├── 声音档案是否足够具体？
│   ├── 太模糊（只有"专业、友好"这种词）→ 重建声音档案
│   └── 足够具体 → 继续
├── 声音档案是否被使用？
│   ├── 团队不知道它的存在 → 培训 + 分发
│   └── 知道但不遵循 → 继续
├── 声音档案是否过时？
│   ├── 品牌已经进化但声音没更新 → 更新声音档案
│   └── 仍然准确 → 继续
└── 是否有平台适配指南？
    ├── 没有 → 添加平台适配指南
    └── 有 → 检查执行层面的问题

处方：
├── 轻度（有声音但不够具体）→ 深化声音档案 + 添加更多示例
├── 中度（声音过时或不被使用）→ 更新声音档案 + 团队培训材料
└── 重度（没有声音定义）→ 完整声音构建流程（提取/构建/测试）
```

## 诊断模式 3：品牌视觉混乱

```
触发信号：
├── 用户说"我们的视觉看起来不专业"
├── 用户说"不同平台的视觉风格不统一"
├── 用户说"设计师每次做出来的东西都不一样"
├── 网站、社交媒体、邮件的视觉风格明显不同
└── 品牌看起来"廉价"或"不可信"

诊断决策树：
├── 有没有视觉识别规范？
│   ├── 没有 → 这就是问题。进入视觉识别构建流程
│   └── 有 → 继续
├── 色彩系统是否完整？
│   ├── 只有主色，没有完整色板 → 扩展色彩系统
│   └── 完整 → 继续
├── 字体系统是否规范？
│   ├── 使用了太多字体或不一致 → 统一字体系统
│   └── 规范 → 继续
├── 图像风格是否一致？
│   ├── 不一致 → 制定图像风格指南
│   └── 一致 → 继续
└── 视觉是否与品牌定位匹配？
    ├── 不匹配（如高端定位但视觉廉价）→ 视觉升级
    └── 匹配 → 视觉没问题，问题在其他地方

处方：
├── 轻度（有规范但不完整）→ 补充缺失的视觉规范
├── 中度（有规范但不被执行）→ 审计 + 修正 + 模板化
└── 重度（没有视觉规范或完全不匹配）→ 完整视觉识别系统构建
```

## 诊断模式 4：品牌故事缺失

```
触发信号：
├── 用户说"不知道怎么讲我们的故事"
├── 关于页面空洞或只有公司信息
├── 没有创始人故事或起源叙事
├── 品牌感觉"没有灵魂"
└── 客户不知道品牌代表什么

诊断决策树：
├── 有没有起源故事？
│   ├── 没有 → 挖掘创始人经历，构建起源故事
│   └── 有 → 继续
├── 起源故事是否有情感共鸣？
│   ├── 太平淡/太商业化 → 重写，注入个人经历和情感
│   └── 有共鸣 → 继续
├── 有没有使命叙事？
│   ├── 没有 → 从品牌信念出发构建使命叙事
│   └── 有 → 继续
├── 有没有客户故事？
│   ├── 没有 → 设计客户故事收集框架
│   └── 有 → 继续
└── 品牌故事是否在关键触点上被使用？
    ├── 没有 → 制定故事分发策略
    └── 有 → 故事没问题，问题在其他地方

处方：
├── 轻度（有故事但不够好）→ 优化叙事结构和情感深度
├── 中度（有部分故事元素但不完整）→ 补充缺失的故事层次
└── 重度（完全没有品牌故事）→ 完整品牌故事框架构建
```

## 诊断模式 5：品牌溢价不足

```
触发信号：
├── 用户说"客户总是嫌贵"
├── 用户说"只能靠打折卖"
├── 用户说"竞品价格更低，我们没优势"
├── 品牌搜索量低，直接流量占比低
└── 复购率低，客户忠诚度差

诊断决策树：
├── 品牌是否有清晰的价值主张？
│   ├── 没有 → 回到定位画布，明确价值主张
│   └── 有 → 继续
├── 价值主张是否被有效传达？
│   ├── 没有 → 优化所有触点的价值传达
│   └── 有 → 继续
├── 品牌是否有情感连接？
│   ├── 纯功能定位 → 增加情感层次（故事、价值观、社区）
│   └── 有情感连接 → 继续
├── 品牌是否有社会证明？
│   ├── 不足 → 建立社会证明体系（评价、案例、媒体）
│   └── 充足 → 继续
└── 品牌体验是否匹配价格？
    ├── 不匹配 → 升级品牌体验（包装、客服、开箱）
    └── 匹配 → 可能是市场/产品问题，转交 afa-product

处方：
├── 轻度（有品牌但溢价空间小）→ 强化差异化 + 社会证明
├── 中度（品牌感知与价格不匹配）→ 品牌升级 + 体验优化
└── 重度（完全靠价格竞争）→ 品牌重塑 + 定位重建
```

## 诊断模式 6：品牌重塑需求

```
触发信号：
├── 用户说"品牌需要升级/重塑"
├── 用户说"品牌形象过时了"
├── 用户说"我们的受众变了"
├── 品牌经历了重大变化（合并、转型、扩品类）
└── 品牌遭遇了声誉危机

诊断决策树：
├── 重塑的原因是什么？
│   ├── 受众变化 → 重新定义受众 + 调整定位
│   ├── 市场变化 → 竞品分析 + 重新定位
│   ├── 产品变化 → 品类重新定义 + 品牌延伸策略
│   ├── 形象过时 → 视觉升级 + 声音现代化
│   └── 声誉危机 → 危机后品牌重建策略
├── 重塑的范围是什么？
│   ├── 微调（Refresh）：保留核心，更新表达
│   ├── 进化（Evolution）：保留部分，升级大部分
│   └── 革命（Revolution）：几乎全部重建
├── 有什么必须保留的品牌资产？
│   ├── 品牌名称？
│   ├── Logo？
│   ├── 核心色彩？
│   ├── 品牌故事？
│   └── 客户关系？
└── 重塑的时间线和预算？

处方：
├── 微调（Refresh）→ 视觉更新 + 声音微调 + 定位锐化
├── 进化（Evolution）→ 重新定位 + 新声音 + 视觉升级 + 故事更新
└── 革命（Revolution）→ 完整品牌重建流程（所有框架从头开始）
```
