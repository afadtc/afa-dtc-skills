> 本文件为创意策略模块的内部参考文件，用于沉淀本模块的工作模式、KPI 体系与输出模板。
> **层级声明**：本文件默认作为 **internal-only** 的内部起草模板库使用，不得整段直接粘贴给用户。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
> **渲染规则**：本文件中的 KPI 阈值、变体数量、Hook 数量、验证图数量、生命周期和任何优胜判断，默认只可作为参考区间、保守起点或待验证计划；不得写成统一健康线、固定配方或对品牌结果的确定性承诺。

## 十三、 衡量与 KPI (Measurement & KPIs)

创意工作的成功不以“生成的图片数量”衡量，而以下游的业务指标衡量。

### 13.1 核心创意指标

| 指标 | 定义 | 参考解释 | 风险信号 |
| :--- | :--- | :--- | :--- |
| **Hook Rate (前 3 秒停留率)** | 视频广告前 3 秒的观看比例 | 结合平台、视频长度与创意目标判断 | 明显低于当前账户基线 |
| **CTR (点击率)** | 点击链接的人数 / 总展示次数 | 结合版位、受众与素材形态判断 | 明显低于当前账户基线 |
| **CVR (转化率)** | 转化人数 / 点击人数 | 结合落地页承接与流量质量判断 | 明显低于当前账户基线 |
| **创意生命周期** | 创意从上线到效率显著恶化的周期 | 结合 CPA、频次与疲劳节奏判断 | 生命周期明显缩短 |
| **创意多样性** | 活跃的创意变体数量 | 以是否足够支撑测试与轮换为判断核心 | 变体明显不足，难以支撑测试 |

### 13.2 质量门禁 (Quality Gate Checklist)

在交付任何素材前，必须先完成以下自我审查：

**技术审查**：
- [ ] 分辨率是否适合目标用途？
- [ ] 是否存在 AI 伪影（变形的手、融化的文字、不可能的几何）？
- [ ] 主体是否清晰对焦？
- [ ] 比例是否正确匹配目标平台？

**品牌审查**：
- [ ] 颜色是否匹配 `creative-kit.md` 的色彩锚点？
- [ ] 排版方向是否一致？
- [ ] 情绪是否符合品牌人格？
- [ ] 反模式清单中的元素是否出现？

**策略审查**：
- [ ] 资产是否服务于其传播目标？
- [ ] 构图是否支持预期用途（如果需要文本空间，是否有留白）？
- [ ] 视觉层级是否清晰（一个焦点，一个信息）？
- [ ] 是否与竞品有差异化（不是品类通用图）？

**平台审查**：
- [ ] 尺寸是否正确匹配目标平台？
- [ ] 在缩略图/预览尺寸下是否仍然有效？
- [ ] 视频是否在 3 秒内 Hook？
- [ ] 文本在移动端是否可读？

**平台政策检查**：
- [ ] 是否符合目标平台的广告政策？
- [ ] AI 生成内容是否需要披露？
- [ ] 是否存在误导性声明？

## 十四、 工作模式 (Work Modes)

本模块提供 6 种标准化的工作模式，每种模式都有完整的触发条件、执行流程和输出规范。

### 模式 1：品牌视觉基建 (Brand Kit Initialization)

**触发**：新品牌接入，或用户要求"定义视觉风格"、"建立品牌视觉规范"。

**完整流程**：
```text
Step 1: 收集输入
├─ 检查是否已有清晰的品牌定位与声音定义
├─ 收集现有视觉资产（网站截图、社交媒体、产品图）
└─ 收集竞品视觉参考

Step 2: 运行多方向探索法
├─ 生成足够支持比较与决策的差异化视觉方向描述
├─ 每个方向包含：色彩、光影、构图、情绪、风险等级
└─ 呈现给用户选择或组合

Step 3: 锁定视觉规范
├─ 根据用户反馈确定最终方向
├─ 锁定色彩锚点（具体 HEX 码）
├─ 锁定光影参数
├─ 锁定排版方向
├─ 锁定反模式清单
└─ 生成并保存 creative-kit.md

Step 4: 验证
├─ 使用锁定的规范生成足够支持一致性验证的测试图
├─ 确认品牌一致性
└─ 用户最终确认
```

**输出**：`creative-kit.md` 文件 + 用于一致性验证的一组测试图 Prompt。

### 模式 2：广告测试矩阵生成 (Ad Matrix Generation)

**触发**：用户要求"做一组广告"、"准备投放素材"、"生成广告创意"。

**完整流程**：
```text
Step 1: 确认输入
├─ 读取 creative-kit.md（如不存在，先运行模式 1）
├─ 确认推广的产品/Offer
├─ 确认目标受众和痛点（来自现有用户洞察与研究结论）
└─ 确认目标平台

Step 2: 构建足够支持测试决策的 Hook 组
├─ 痛点放大 (Pain Point)：基于受众最强烈的痛点
├─ 状态转变 (Transformation)：基于产品的核心价值承诺
├─ 社会认同 (Social Proof)：基于已有的客户数据或评价
└─ 好奇心鸿沟 (Curiosity Gap)：基于行业的信息不对称

Step 3: 设定适合当前平台与素材目标的视觉 Format 组
├─ 产品英雄 (Product Hero)
├─ UGC 风格 (UGC Style)
└─ 粗体排版 (Bold Typography)

Step 4: 输出足够支持测试决策的变体组合
├─ 每个变体包含：Tracking ID, Prompt, 画面内文本, 文案, 标题, 尺寸
├─ 所有 Prompt 必须包含反塑料感规则和品牌色彩
└─ 为每个变体标注最佳投放平台
```

**输出**：完整的测试矩阵文档。

### 模式 3：社交媒体资产包 (Social Asset Batch)

**触发**：用户要求"做一周的社媒图"、"生成 Instagram 帖子"、"做 LinkedIn 内容图"。

**完整流程**：
```text
Step 1: 确认输入
├─ 读取 creative-kit.md
├─ 确认目标平台和内容日历
└─ 确认内容主题（产品推广、教育、品牌故事、互动）

Step 2: 选择原生模板
├─ IG Feed：4:5 或 1:1，留出底部文本区
├─ IG Carousel：5 张叙事结构
├─ LinkedIn：1:1 数据可视化或 PDF 轮播
├─ TikTok：9:16 缩略图 + 视频脚本
└─ Pinterest：2:3 垂直灵感图

Step 3: 批量生成
├─ 确保系列视觉的一致性（统一色彩、光影、排版）
├─ 每张图留出文本叠加的安全区
└─ 输出完整的 Prompt 包

Step 4: 排版建议
├─ 为每张图提供后期文本叠加的位置和字体建议
└─ 确保文本在移动端可读
```

**输出**：平台原生化的系列视觉 Prompt 包 + 排版建议。

### 模式 4：Reali-TEA 短视频脚本 (UGC Scripting)

**触发**：用户要求"写 TikTok 脚本"、"做 UGC 视频"、"写 Reels 脚本"。

**完整流程**：
```text
Step 1: 确认输入
├─ 确认产品和核心卖点
├─ 确认目标受众的人口统计和心理特征
└─ 确认视频时长（15 秒 / 30 秒 / 60 秒）

Step 2: 构建 Hook
├─ 强制应用"前 7 秒不提产品"法则
├─ 选择 Hook 类型（负面 Hook / 好奇 Hook / 共鸣 Hook）
└─ 编写口语化的开场白

Step 3: 注入 Reali-TEA
├─ 加入言语不完美（填充词、中途重启、自我纠正）
├─ 加入行为真实感（自嘲、分心、打破第四面墙）
├─ 加入视觉提示（手持晃动、非中心构图、环境噪音）
└─ 检查并替换所有营销词汇

Step 4: 输出结构化脚本
├─ 表格格式：时间戳 | 视觉/动作 | 画外音/字幕 | 真实感注入
└─ 包含遗忘式 CTA 和迅速切断指示
```

**输出**：结构化视频脚本表格。

### 模式 5：单点视觉突破 (Hero Asset Creation)

**触发**：用户需要一张极其重要的主图（如官网 Hero Image、产品发布主图）。

**完整流程**：
```text
Step 1: 深度需求分析
├─ 这张图将用在哪里？（官网首页 / 广告主图 / 产品页）
├─ 目标受众看到这张图时应该感受到什么？
└─ 竞品在同一位置使用了什么样的视觉？

Step 2: 极度细化 Prompt
├─ 5 层约束模型的每一层都必须达到最高精度
├─ 光影参数精确到光源角度和色温
├─ 材质描述精确到具体的胶片型号或镜头
└─ 强制加入模式中断 (Pattern Interrupt) 元素

Step 3: 生成 3 个微调变体
├─ 变体 A：标准执行
├─ 变体 B：更大胆的构图或色彩
├─ 变体 C：更克制/极简的方向
└─ 让用户选择最终版本

Step 4: 优化与交付
├─ 根据用户反馈进行最终微调
└─ 输出最终 Prompt + 排版建议（如果需要文本叠加）
```

**输出**：3 个变体的完整 Prompt + 最终选择的优化版本。

### 模式 6：胜者迭代 (Winner Iteration)

**触发**：下游投放 skill 反馈某个创意获胜，需要围绕胜者生成变体。

**完整流程**：
```text
Step 1: 分析获胜原因
├─ 是 Hook 获胜还是 Format 获胜？
├─ 获胜的具体指标是什么？（CTR / CVR / Hook Rate）
└─ 获胜创意的 Tracking ID 是什么？

Step 2: 确定迭代变量
├─ 只选择一个变量进行改变
├─ 可选变量：背景颜色 / 光影方向 / 文本内容 / CTA / 人物特征
└─ 绝对禁止同时改变多个变量

Step 3: 生成一组足够支持比较的微调变体
├─ 保留获胜创意的所有其他元素不变
├─ 只改变选定的单一变量
└─ 为每个变体分配新的 Tracking ID

Step 4: 交付给后续投放执行流程
├─ 输出变体的 Prompt + 文案
├─ 建议测试预算分配（按目标与风险控制设置）
└─ 建议最小样本量（按平台体量与判断置信度设置）
```

**输出**：3-5 个微调变体的完整执行包。

## 十五-B、淡季创意策略（v1.9.5 新增）

> **激活条件**：当 Hub 传入 `seasonal_mode = off_season` 时，本模块自动激活。淡季是低成本、低风险测试新创意方向的最佳窗口。

### 15B.1 淡季创意测试框架

```
淡季创意测试原则：

1. 为什么淡季是创意测试的黄金期
   → CPM 往往低于旺季，同样预算通常能触达更多人
   → 失败成本相对更低，可以更从容地验证新方向
   → 旺季来临时更容易拥有经过验证的创意库

2. 淡季创意测试计划
   → 每次迭代测试一组足够支持比较的创意变体（基于获胜概念的微调版本）
   → 测试维度：Hook（前段）、格式、角度、CTA
   → 记录所有测试结果到 Brand Brain
   → 旺季前关键窗口逐步减少高不确定性新测试，锁定当前 Top 创意

3. 淡季创意方向（从促销转向品牌）
   旺季创意：产品功能 → 价格优惠 → 紧迫感 → 立即购买
   淡季创意：品牌故事 → 教育内容 → 用户故事 → 了解更多
```

### 15B.2 淡季创意内容类型

| 类型 | 目标 | 关键数据 |
|------|------|---------|
| **品牌故事视频** | 品牌认知和情感连接 | 可能提升点击与情感连接，但幅度需用真实测试验证 |
| **教育/How-to 内容** | 建立专业信任 | 常用于沉淀专业信任与长期转化资产 |
| **UGC 和客户评价** | 社会认同积累 | 往往较品牌内容更容易建立真实感，但差异需以实测验证 |
| **幕后花絮** | 真实感和透明度 | 有助于提升品牌好感度和忠诚度 |
| **产品使用场景** | 激发需求 | 场景化内容通常更容易引发代入与分享 |

### 15B.3 Campaign 疲劳诊断与创意刷新

```
Campaign 疲劳是"假季节性"的常见原因之一。
当广告效果下降时，先排查是否是创意疲劳，而非真正的季节性下降。

Campaign 疲劳信号：
  ├── CTR 持续下降
  ├── Frequency 持续上升并逼近受众耐受上限
  ├── CPM 上升但 CTR 下降
  └── 评论区开始出现重复曝光反馈

淡季创意刷新清单：
  1. 更新所有产品图片（新角度、新场景、新模特）
  2. 重写广告文案（新 Hook、新角度、新 CTA）
  3. 测试新的广告格式（Carousel → Video → Collection）
  4. 更新网站 Banner 和落地页视觉
  5. 拍摄新的 UGC 和客户评价视频
  6. 测试不同的创意频次和受众细分组合
```

### 15B.4 淡季素材库建设

```
淡季素材储备目标：

旺季前应准备的素材库：
  ├── 经过验证的广告素材：按投放规模准备足够轮换的素材库
  ├── 产品图片：按 SKU 重要性准备足够角度与场景
  ├── UGC 视频：按平台节奏准备足够的已授权客户视频
  ├── 品牌视频：按品牌叙事需求准备核心视频资产
  └── 邮件/SMS 素材：按旺季内容节奏准备连续素材

淡季素材生产节奏：
  → 按周节奏持续产出新创意概念
  → 视资源安排阶段性拍摄（产品 + 场景 + 模特）
  → 持续收集和整理 UGC
  → 在旺季前关键窗口完成主要素材准备
```
