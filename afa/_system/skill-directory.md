# AFA DTC 模块全局字典
> **协议层级**：全局参考 · Supervisor 越界路由时查阅
>
> **版本**：v2.4.6
> 本文件为 Supervisor 和 Hub 提供全局拓扑视野。当 Worker 通过 `completion.out_of_scope` 结构化回交时，Supervisor 根据本字典判断应将用户引导至哪个方向，并默认沿主问题给出最自然的下一步；只有在出现真实方向分叉、资源投入差异或高风险执行动作时，才请求用户拍板。
>
> **display_name 规范**：`display_name` 是面向用户输出时使用的模块显示名称。根据铁律 2，所有面向用户的输出（包括 Visible Loading、WHAT'S NEXT、Header、话术模板、越界转向说明）必须使用 `display_name`，严禁暴露 `afa-` 前缀的内部代号。
---

## Hub 与 Supervisor 层

| 模块 | display_name | 一句话职责 |
|:---|:---|:---|
| **afa** | AFA 管家 | 全局路由中枢：接收用户需求，检测上下文变量，并在合适时分发给对应 Supervisor。 |
| **afa-foundation** | 品牌基建组 | Foundation 组 Supervisor：协调品牌定位、产品策略、市场探索等基建类 Worker。 |
| **afa-paid** | 付费获客组 | Paid 组 Supervisor：协调 Meta、Google、TikTok 广告和创意生产 Worker。 |
| **afa-organic** | 有机增长组 | Organic 组 Supervisor：协调 SEO、社交媒体、创作者经济、公关和 GEO 搜索 Worker，并负责有机流量协同路由。 |
| **afa-monetize** | 变现留存组 | Monetize 组 Supervisor：协调转化率、客户体验、留存、客单价、邮件和短信 Worker。 |
| **afa-scale** | 运营扩张组 | Scale 组 Supervisor：协调运营效率和全渠道扩张 Worker。 |
