# Reference Authoring Rules（内部参考文件编写规范）

本文件是 AFA DTC Skill 包内 `references/*.md` 与少量配套模板头部的**唯一编写真源**。凡涉及参考文件标题、开头说明、中英文边界、internal-only 标记方式，以及头部是否出现内部模块代号，均以本文件为准；各模块只能继承和实例化，不得平行重定义。

---

## 1. 适用范围

本规范适用于以下资产：

| 资产类型 | 是否适用 | 说明 |
|---|---|---|
| `references/*.md` | 是 | 默认全部适用 |
| 报告模板、工作模式模板、诊断规则模板 | 是 | 若文件位于 `references/` 内，也一并适用 |
| `SKILL.md` | 部分适用 | 仅当需要统一头部骨架与说明方式时参考本文件；模块职责与协议字段仍以各自 `SKILL.md` 为准 |
| `_system/*.md` | 否 | `_system` 自身遵循系统协议文件写法，不受本文件头部格式约束 |

---

## 2. references 标题铁律

所有 `references/*.md` 的标题必须遵循以下规则：

| 规则 | 要求 |
|---|---|
| 标题主语言 | 以中文为主 |
| 是否允许 `afa-*` 出现在标题中 | 默认不允许 |
| 是否允许 `Reference file for afa-xxx` | 不允许 |
| 是否允许 `# afa-xxx — Core Frameworks` 一类旧式标题 | 不允许 |
| 是否允许保留专业英文术语 | 允许，但应作为术语补充，而不是主标题骨架 |

**正确示例**：

| 场景 | 推荐写法 |
|---|---|
| 核心框架文件 | `# 核心框架` |
| 工作模式文件 | `# 工作模式与模板` |
| 诊断规则文件 | `# 诊断规则` |
| 报告模板文件 | `# 报告模板` |

**错误示例**：

| 错误写法 | 问题 |
|---|---|
| `# afa-email — Core Flows Playbook` | 标题暴露内部代号，且整体骨架英文化 |
| `> Reference file for afa-retain (留存引擎)` | 旧式内部命名残留，不是标准中文说明 |
| `# Work Modes & Templates — afa-cx` | 结构顺序与语言主次均不符合当前标准 |

---

## 3. 开头说明写法

references 文件标题下方应紧接一段简短说明，说明本文件的用途、使用边界，以及是否包含 internal-only 区块。

推荐结构如下：

```md
> 本文件为【中文角色名】的内部参考文件，用于【一句话说明用途】。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
```

### 3.1 关于内部模块代号

| 场景 | 是否允许出现 `afa-*` |
|---|---|
| 标题、开头说明、模板抬头 | 不允许 |
| internal-only 小节中的路由说明 | 允许 |
| 正文中作为跨模块职责说明，但会被直接复制到用户版本的内容 | 不允许 |
| 仅供系统使用的映射、交接、路由表 | 允许，但必须显式标记 `仅供系统使用` 或 `internal-only` |

---

## 4. 中英文边界规则

本 Skill 包默认采用**中文说明 + 必要英文术语 / 面向消费者英文示例**的结构。

| 内容类型 | 默认语言 | 备注 |
|---|---|---|
| 规则说明、策略解释、流程步骤 | 中文 | 默认全部中文化 |
| 专业缩写、通用术语 | 可保留英文 | 如 KPI、SEO、LTV、CTA |
| 面向消费者可直接使用的文案示例 | 可保留英文 | 仅限真实需要英文呈现的场景，如 Email / SMS 消费者文案 |
| 模板抬头中的角色名称 | 中文或自然业务角色 | 不使用 `AFA DTC xxx Engine` 作为外显头衔 |

因此，**英文不是被全面禁止**；真正被禁止的是“用内部代号或旧式引擎命名去充当 references 的主标题、开头说明或模板抬头”。

---

## 5. mixed 资产的显式隔离

如果同一文件既承载内部协作说明，又承载可整理成用户交付物的内容，则必须显式分层。

| 层级 | 写法要求 |
|---|---|
| 用户可整理层 | 只写自然语言、业务角色、行动建议 |
| internal-only 层 | 使用二级或三级标题清楚标明“仅供系统使用” |
| 同一段内混写前台与后台信息 | 禁止 |

推荐标记方式：

```md
## 内部映射说明（仅供系统使用，不直接进入用户交付物）
```
