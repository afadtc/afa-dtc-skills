## 一、路由判断框架

> Hub 的核心任务是**快速分诊**：判断用户的问题属于哪个领域，路由到正确的模块。
> 以下框架用于路由判断，不用于给用户下"健康/危险"的结论。

### 1.1 问题领域识别矩阵

```
用户信号                          可能涉及的领域              优先路由
──────────────────────────────────────────────────────────────────────
"转化率低/不出单"                  转化漏斗                    afa-convert / afa-diagnose
"广告花了钱没效果"                 付费广告                    afa-fb / afa-gg / afa-tt
"客单价太低"                       AOV 优化                    afa-aov
"老客不回来"                       留存与复购                  afa-retain
"不知道怎么定价"                   产品与定价                  afa-product
"邮件没人看"                       Email 营销                  afa-email
"想看看数据"                       数据体检                    afa-dashboard
"生意不好，不知道哪里出了问题"     全链路诊断                  afa-diagnose
"想做SEO/自然流量"                 搜索优化                    afa-seo
"想做社交媒体"                     社交媒体                    afa-social / afa-organic
"想做红人营销"                     红人合作                    afa-influencer
"想扩展新市场"                     市场拓展                    afa-expand / afa-geo
```

### 1.2 多信号交叉路由规则

```
当用户同时提到多个问题时：
  ├── 如果涉及 ≥3 个领域 → 优先路由 afa-diagnose（全链路诊断）
  ├── 如果涉及 2 个相关领域 → 路由主要问题领域，附带提示次要领域
  └── 如果问题模糊不清 → 路由 afa-diagnose（Stage 0 问题具体化引擎）
```

## 二、用户数据画像框架（路由级）

> Hub 在首次接触用户时，通过极简问题快速建立用户画像，用于路由判断。
> 详细的指标计算和基准线生成由 afa-dashboard 和 afa-diagnose 负责。

### 2.1 快速画像三问

```
问题 1：你的品类是什么？
  → 用于判断品类特征（高频低客单 vs 低频高客单 vs 订阅制）
  → 影响路由优先级（如订阅制品牌优先关注留存）

问题 2：你现在月销大概多少？
  → 用于判断品牌阶段
  → 影响路由策略（初创期 vs 增长期 vs 规模化期）

问题 3：你最想解决什么问题？
  → 直接路由到对应模块
  → 如果用户说不清 → 路由 afa-diagnose
```

### 2.2 品牌阶段判断框架

```
阶段一：0→1（月销 <$10K）
  ├── 核心关注：产品市场匹配、首批客户获取
  ├── 典型路由：afa-product → afa-launch → afa-convert
  └── 特征：数据量少，更依赖定性判断

阶段二：1→10（月销 $10K-$100K）
  ├── 核心关注：渠道效率、单位经济模型
  ├── 典型路由：afa-diagnose → afa-fb/afa-gg → afa-retain
  └── 特征：开始有足够数据做量化分析

阶段三：10→100（月销 >$100K）
  ├── 核心关注：规模化效率、品牌建设、多渠道协同
  ├── 典型路由：afa-dashboard → afa-scale → afa-brand
  └── 特征：数据丰富，需要系统性监控
```

## 三、指标方向参考（路由辅助）

> 以下不是"行业基准"，而是**方向性参考**——帮助 Hub 判断用户描述的数值大致处于什么水平，以便路由到正确模块。
> 实际的状态判断（健康/警戒/危险）由 afa-dashboard 和 afa-diagnose 基于用户自己的目标和历史数据来完成。

### 3.1 核心指标方向参考

```
指标                  方向性参考                    说明
──────────────────────────────────────────────────────────────────
整体转化率            DTC 电商通常在 1-5% 之间       品类差异极大，不宜用固定数值判断
加购率                通常为转化率的 2-4 倍          与页面设计和产品展示强相关
客单价 (AOV)          取决于品类，$30-$300+          假睫毛和美容仪不可同日而语
CAC                   取决于品类和渠道               必须与 LTV 和毛利率结合看
LTV:CAC               >1:1 才能存活                  具体目标由用户根据商业模式设定
广告 ROAS             必须 > 盈亏平衡 ROAS           盈亏平衡 ROAS = 1 ÷ 毛利率
毛利率                品类差异大（30%-80%）           自制品 vs 代工品差异显著
复购率                取决于品类和产品特性            消耗品 vs 耐用品完全不同
```

### 3.2 盈亏平衡 ROAS 速算（路由级）

```
盈亏平衡 ROAS = 1 ÷ 毛利率

示例：
  毛利率 70% → 盈亏平衡 ROAS = 1.43x
  毛利率 60% → 盈亏平衡 ROAS = 1.67x
  毛利率 50% → 盈亏平衡 ROAS = 2.00x
  毛利率 40% → 盈亏平衡 ROAS = 2.50x
  毛利率 30% → 盈亏平衡 ROAS = 3.33x

用途：当用户说"我的 ROAS 是 X"时，快速判断是否盈利，决定路由优先级。
```

## 四、关键比率关系（路由辅助）

```
比率                          方向性参考              路由信号
──────────────────────────────────────────────────────────────────
LTV:CAC                       >1:1 才能存活           <1:1 → 紧急路由 afa-diagnose
Email Rev:Total Rev            占比越高越健康          极低 → 路由 afa-email
Organic:Paid Traffic           越均衡越好              极度依赖付费 → 路由 afa-seo/afa-organic
New:Returning Visitors         取决于品牌阶段          初创期新客多正常，成熟期应有回头客
Mobile:Desktop Revenue         移动端占比持续上升      移动端转化差 → 路由 afa-convert
COGS:Revenue                   取决于品类和模式        过高 → 路由 afa-product（定价问题）
Ad Spend:Revenue               取决于品牌阶段          过高 → 路由 afa-diagnose（利润树分析）
```

## 五、季节性提醒（路由辅助）

> 季节性影响所有电商品类，但程度不同。Hub 在路由时应考虑当前月份对用户数据的影响。

```
时段              一般影响                          路由注意事项
──────────────────────────────────────────────────────────────────
Q1（1-3月）       节后回落期                        用户说"数据下降"可能是正常季节性
Q2（4-6月）       平稳期                            适合做基线测试和优化
Q3（7-9月）       返校季+备战期                     服装品类可能上升
Q4（10-12月）     旺季（BFCM+圣诞）                 数据普遍上升，不宜与旺季数据做基准
1月 vs 12月       落差可能达 40-60%                  用户说"数据暴跌"需先排除季节性

重要原则：
  ├── 不要用旺季数据作为全年基准
  ├── 不要在淡季告诉用户"你的数据不健康"
  └── 建议用户用同比（去年同期）而非环比来判断趋势
```

## 六、渠道级路由阈值框架

> 以下框架帮助 Hub 判断用户在某个渠道是否需要专项帮助。
> 具体的数值判断由对应的专项模块完成。

### 付费广告

```
路由信号                                          推荐路由
──────────────────────────────────────────────────────────────
用户说 ROAS 低于盈亏平衡线                         afa-fb / afa-gg / afa-tt（按平台）
用户说广告花费占比过高                             afa-diagnose（利润树分析）
用户说 CPM/CPC 持续上升                            对应广告平台模块
用户说广告疲劳/频次过高                            afa-creative（素材问题）
```

### 邮件/SMS 营销

```
路由信号                                          推荐路由
──────────────────────────────────────────────────────────────
用户说邮件没人打开                                 afa-email
用户说 SMS 效果不好                                afa-sms
用户说自有渠道收入占比极低                         afa-email + afa-sms（体系搭建）
用户说自动化序列没有                               afa-email（优先搭建自动化）
```

### SEO / 自然流量

```
路由信号                                          推荐路由
──────────────────────────────────────────────────────────────
用户说自然流量几乎为零                             afa-seo
用户说搜索排名下降                                 afa-seo
用户说想减少对广告的依赖                           afa-seo + afa-organic
```

### 客户体验

```
路由信号                                          推荐路由
──────────────────────────────────────────────────────────────
用户说退货率高                                     afa-cx / afa-product
用户说差评多                                       afa-cx
用户说客服压力大                                   afa-cx
```
