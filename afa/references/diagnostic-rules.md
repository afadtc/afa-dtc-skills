## 一、利润树诊断框架 (Profit Tree)

### 完整利润树

```
净利润
├── 毛利润 = 营收 - COGS
│   ├── 营收 = 流量 × 转化率 × 客单价 × (1 + 复购系数)
│   │   ├── 流量
│   │   │   ├── 付费流量
│   │   │   │   ├── Facebook/Meta Ads
│   │   │   │   ├── Google Ads (Search + Shopping + PMAX)
│   │   │   │   ├── TikTok Ads
│   │   │   │   └── 其他付费渠道
│   │   │   ├── 自然流量
│   │   │   │   ├── SEO (搜索引擎自然流量)
│   │   │   │   ├── 直接访问 (Direct)
│   │   │   │   ├── 社交媒体自然流量
│   │   │   │   └── AI 搜索引擎引荐
│   │   │   └── 自有流量
│   │   │       ├── Email 流量
│   │   │       ├── SMS 流量
│   │   │       └── Push 通知流量
│   │   ├── 转化率 = 加购率 × 结账率 × 支付成功率
│   │   │   ├── 加购率 (Add-to-Cart Rate)
│   │   │   ├── 结账率 (Checkout Rate)
│   │   │   └── 支付成功率 (Payment Success Rate)
│   │   ├── 客单价 (AOV)
│   │   │   ├── 基础产品价格
│   │   │   ├── 追加销售 (Upsell)
│   │   │   ├── 交叉销售 (Cross-sell)
│   │   │   ├── Bundle 策略
│   │   │   └── 阶梯折扣
│   │   └── 复购系数
│   │       ├── 复购率 (Repeat Purchase Rate)
│   │       ├── 复购频次 (Purchase Frequency)
│   │       └── 客户生命周期 (Customer Lifetime)
│   └── COGS (销货成本)
│       ├── 产品采购成本
│       ├── 包装成本
│       ├── 关税/进口税
│       └── 质检成本
├── 获客成本 (CAC)
│   ├── 广告花费
│   │   ├── 媒体花费 (Media Spend)
│   │   └── 代理费/工具费
│   ├── 内容营销成本
│   └── 网红/联盟营销成本
├── 运营成本
│   ├── 平台费用 (Shopify/其他)
│   ├── 工具/SaaS 费用
│   ├── 人力成本
│   └── 仓储成本
├── 物流成本
│   ├── 发货运费
│   ├── 退货运费
│   └── 退货商品损耗
└── 退货/退款成本
    ├── 退货率
    ├── 退款处理费
    └── 退货商品残值
```

### 利润树诊断规则

```
规则 1：自上而下诊断
  从净利润开始，逐层向下拆解
  在每一层找到偏差最大的分支
  继续向下直到找到可操作的根因

规则 2：数学验证
  每一层的子节点之和（或之积）必须等于父节点
  如果不等，说明有遗漏的因素

规则 3：优先级排序
  在同一层级的多个问题中，按以下顺序优先处理：
  ① 对利润影响最大的（杠杆最大）
  ② 最容易修复的（成本最低）
  ③ 最快见效的（时间最短）

规则 4：联动效应
  某些节点的改善会连锁影响其他节点
  例如：提高产品页质量 → 加购率↑ → 转化率↑ → 营收↑
  同时：流量质量提升 → 退货率↓ → 物流成本↓
  优先选择有正向联动效应的改善点
```

### 利润树健康度评分

```
维度              健康          警告          危险
──────────────────────────────────────────────────────
毛利率            >60%          40-60%        <40%
净利率            >15%          8-15%         <8%
CAC/LTV 比率      <1:3          1:3-1:2       >1:2
广告占营收比       <30%          30-45%        >45%
退货率            <5%           5-15%         >15%
复购率            >25%          15-25%        <15%
```

## 二、转化漏斗诊断框架 (Conversion Funnel) — 分诊速查

> 深度转化漏斗诊断（含每步详细检查项和诊断逻辑）→ 见《深度诊断框架》第二章
> 转化优化执行 → 见 afa-convert

### 6 步关键阈值速查

```
Step 1: 流量质量
  跳出率        健康 <45%    警告 45-65%    危险 >65%
  会话时长      健康 >2min   警告 1-2min    危险 <1min
  页面深度      健康 >3页    警告 2-3页     危险 <2页
  付费流量占比   健康 <60%    警告 60-80%    危险 >80%

Step 2: 产品页转化
  加购率        健康 >8%     警告 5-8%      危险 <5%

Step 3: 结账流程
  购物车→结账   健康 >60%    警告 40-60%    危险 <40%
  结账步骤数    最佳 1-2步   可接受 3步     需优化 >3步

Step 4: 定价与价值感
  → 检查：价格锚定、价值堆叠、风险逆转、紧迫感

Step 5: 信任系统
  → 检查：社会证明、权威背书、品牌故事、安全感

Step 6: 技术性能
  加载速度      健康 <2.5s   警告 2.5-4s    危险 >4s
  LCP           <2.5s       FID <100ms      CLS <0.1
```

## 三、付费媒体三支柱诊断框架 (Paid Media Triad) — 分诊速查

> 深度付费媒体诊断（含素材疲劳规则、受众饱和规则、匹配度规则）→ 见《深度诊断框架》第三章
> 平台执行 → 见 afa-fb / afa-gg / afa-tt

### 三支柱模型

```
广告效果 = 素材质量 × 受众精准度 × 落地页匹配度
任何一个支柱出问题，整体效果都会大幅下降。
```

### 关键阈值速查

```
素材质量：
  CTR           健康 >1.5%   警戒 1.0-1.5%  危险 <1.0%
  *行业平均 1.5-2.5%，按品类细分（服装 1.5-2.5%，家居 1.0-2.0%，美妆 2.0-3.5%）*
  Hook Rate     健康 >30%    警告 20-30%    危险 <20%
  频次          健康 <3.0    警戒 3.0-5.0   危险 >5.0
  *注意：仅当频次升高伴随 CTR 下降或 CPA 上升时，才判定为实质性受众疲劳。*
  *如果频次高但 CPA 稳定，说明是高频复购受众，无需干预。*

受众精准度：
  CPM           健康 <$15    警告 $15-30    危险 >$30
  CPC           健康 <$1.5   警告 $1.5-3.0  危险 >$3.0
  受众重叠率     健康 <20%    警告 20-40%    危险 >40%

落地页匹配度：
  CVR           健康 >3.0%   警告 1.5-3.0%  危险 <1.5%
  加购率         健康 >8%     警告 5-8%      危险 <5%
  跳出率         健康 <40%    警告 40-60%    危险 >60%
```

### 三支柱联合诊断矩阵

```
CTR    CVR    诊断结论                    优先行动
──────────────────────────────────────────────────────────────
高     高     系统健康                    扩量（增加预算/拓展受众）
高     低     落地页问题 或 受众偏差       优化落地页 / 收窄受众
低     高     素材问题                    更新素材
低     低     系统性问题                  全面诊断（三支柱同时优化）
下降   稳定   素材疲劳                    更换素材
稳定   下降   落地页问题 或 竞争加剧       检查落地页 + 竞品动态
下降   下降   受众饱和 + 素材疲劳          拓展受众 + 更新素材
```

## 六、邮件健康度 8 维度诊断

```
维度              指标              健康          警告          危险
──────────────────────────────────────────────────────────────────
① 送达率          Delivery Rate    >98%          95-98%        <95%
② 打开率          Open Rate        >25%          18-25%        <18%
③ 点击率          Click Rate       >3.5%         2.0-3.5%      <2.0%
④ 转化率          CVR from Email   >1.5%         0.8-1.5%      <0.8%
⑤ 退订率          Unsubscribe      <0.3%         0.3-0.5%      >0.5%
⑥ 垃圾邮件率      Spam Rate        <0.1%         0.1-0.3%      >0.3%
⑦ 列表增长率      List Growth      >5%/月        2-5%/月       <2%/月
⑧ 收入占比        Email Revenue    >20%          10-20%        <10%

诊断逻辑：
  送达率低 → 域名信誉问题 / 列表质量差 → 清理列表 + 验证域名
  打开率低 → 标题不吸引 / 发送时间不对 / 列表疲劳 → A/B 测试标题
  点击率低 → 内容不相关 / CTA 不清晰 → 优化内容和 CTA
  转化率低 → 着陆页问题 / 优惠不够吸引 → 优化着陆页
  退订率高 → 发送频率过高 / 内容不相关 → 降频 + 分层发送
  垃圾邮件率高 → 列表来源不干净 / 内容触发过滤器 → 清理列表
  列表增长慢 → 缺乏获取渠道 → 优化弹窗/铅磁铁
  收入占比低 → 邮件体系不完整 → 搭建完整的自动化序列
```

## 十、诊断优先级决策矩阵

当多个问题同时存在时，使用以下矩阵决定优先级：

```
                    影响大                  影响小
              ┌──────────────────┬──────────────────┐
              │                  │                  │
  容易修复    │   ★ 立即执行      │   ③ 顺手做       │
              │   （最高优先级）   │   （低优先级）    │
              │                  │                  │
              ├──────────────────┼──────────────────┤
              │                  │                  │
  难以修复    │   ② 规划执行      │   ④ 暂时搁置     │
              │   （高优先级）     │   （最低优先级）  │
              │                  │                  │
              └──────────────────┴──────────────────┘

典型的「立即执行」项目：
  - 添加弃购挽回邮件（容易 + 影响大）
  - 优化产品页首屏信任元素（容易 + 影响大）
  - 修复结账流程中的运费意外（容易 + 影响大）
  - 添加 Guest Checkout（容易 + 影响大）

典型的「规划执行」项目：
  - 重新定位品牌（难 + 影响大）
  - 搭建完整的 SEO 内容体系（难 + 影响大）
  - 开发新产品线（难 + 影响大）

典型的「顺手做」项目：
  - 优化邮件标题（容易 + 影响中等）
  - 添加产品页 FAQ（容易 + 影响中等）

典型的「暂时搁置」项目：
  - 完全重建网站（难 + 影响不确定）
  - 进入全新市场（难 + 影响不确定）
```

## v1.8 诊断行为准则

> 以下规则优先级高于上述所有诊断框架。当冲突时，以本节为准。

### 准则 1：三段式诊断流程（取代旧的"假设驱动直接判断"）

```
Step 1：框架拆解 + 常见原因列表
  指出异常指标，并列出导致该异常的常见可能原因清单。
  示例：「你的转化率偏低，常见原因包括：
  A. 流量质量问题（广告受众不精准）
  B. 落地页体验问题（加载慢、信任元素缺失）
  C. 定价问题（价格与感知价值不匹配）
  D. 支付流程问题（支付方式不足、运费意外）」

  禁止：直接跳到「我判断你的问题是 X」而不先展示可能性清单。

Step 2：向用户索取细分数据
  「为了把判断做得更准，请优先补充：
  ① 各流量渠道的分别转化率（可从 GA4 > 流量获取 > 概览获取）
  ② 落地页加载速度（可从 PageSpeed Insights 获取）
  ③ 购物车到结账的流失率（可从 Shopify 后台 > Analytics 获取）」

  如果用户无法提供 → 输出数据缺口清单 + 当前保守判断 + 最小验证顺序。
  禁止：用行业基准估算并当作用户实际数据使用。

Step 3：拿到数据后再做最终判断
  基于用户提供的实际数据，给出深度诊断结论。
  必须展示推理链：「因为 [A 数据] + [B 数据]，所以判断根因是 [C]」
```

### 准则 2：推导过程透明化

```
规则 1：任何量化预测必须展示推导过程
  ✗ 禁止：「预计你的 ROAS 可以达到 3.5」
  ✓ 正确：「基于以下假设推算：
     • 假设转化率从 1.8% 优化到 2.5%（行业中位数）
     • 假设客单价保持 $65 不变
     • 假设 CPC 保持 $1.2 不变
     → 推算 ROAS = $65 × 2.5% / $1.2 = 1.35」

规则 2：关键变量防漏检查
  在任何财务计算中，必须检查是否考虑了：
  ☐ 退货率（DTC 平均 15-30%）
  ☐ 支付处理费（通常 2.9% + $0.30）
  ☐ 平台费用（Shopify 等）
  ☐ 税费（销售税 / VAT）
  ☐ 物流成本（发货 + 退货）
  ☐ 工具/SaaS 费用
  如果用户未提供某个变量，不要默认忽略，而是：
  → 明确标记该变量缺失，并说明这会影响哪些判断
  → 如必须给方向性判断，只能给保守区间或情景化假设，不把行业平均值写成用户实际值
  → 在输出中列出所有假设，让用户可以核实

规则 3：数据基础声明（取代「置信度」）
  ✗ 禁止：「置信度：低」「以下结论仅供参考」「信心度：70%」
  ✓ 正确：「数据基础：基于你提供的 7 天广告数据 + 行业基准对照（未替代你的实际数据）」
  ✓ 正确：「如果能提供 30 天数据，诊断会更精准」
```

### 准则 3：零数据场景处理

```
当用户无法提供任何数据时：

  ✗ 禁止：用行业基准估算并当作用户实际数据使用
  ✗ 禁止：编造品牌名称或业务数据
  ✗ 禁止：主动推送长篇工具设置教程

  ✓ 正确做法：先给「当前保守可执行版」+「数据缺口清单」
  「基于你目前给的信息，我先给一版最稳妥的排查顺序；
  以下数据会影响精度，但不妨碍先启动第一步。

  优先补充：
  ① [具体数据 A] — 可从 [具体来源和路径] 获取
  ② [具体数据 B] — 可从 [具体来源和路径] 获取

  如果你现在拿不到，我也会先按保守路径继续；
  如果你愿意补充，我可以把诊断升级得更精确。」
```

### 准则 4：小样本警告

```
当输入数据样本极小时（如广告花费 <$1000 或数据仅 <7 天）：

  必须在结论前加上数据基础声明：
  「注意：以下分析基于 {X 天} 的数据，样本较小。
  建议累积至少 30 天数据后重新诊断，以获得更稳定的结论。」

  禁止：基于 3 天数据就做出「你的广告策略有问题」这样的定性判断。
```
