## 二、2026 Google Ads 新范式

> **核心认知转变**：从"关键词竞价 + 手动管理"到"AI 协同 + 信号质量 + 混合策略"。

### 2.1 范式转变全景

```
旧范式（2020-2023）                    新范式（2025-2026）
───────────────────                    ───────────────────
ABGD 关键词分层结构              →     混合式 PMax + 标准购物协同
PMax 唯一策略                    →     PMax + 标准购物 + 搜索 多系列协同
手动出价 + 精确匹配              →     智能出价 + 广泛匹配 + AI 优化
最终点击归因                     →     DDA 2.0 数据驱动归因
SEO 为关键词排名                 →     GEO 为 AI 生成答案的信息源
展示广告做再营销                 →     Demand Gen 做视觉化需求创造
单一 ROAS 考核                   →     利润导向 + LTV 导向的混合考核
Pixel 单一追踪                   →     增强型转化 + 服务器端追踪
```

### 2.2 三大核心支柱

```
支柱 1：信号质量（Signal Quality）
├── 增强型转化（Enhanced Conversions）：上传第一方数据提升匹配率
├── 服务器端追踪（Server-Side Tracking）：绕过浏览器限制
├── 客户数据平台（CDP）整合：打通 CRM + 广告数据
├── 转化价值规则（Conversion Value Rules）：按受众/地区/设备调整价值
└── GA4 深度整合：事件追踪 + 用户属性 + 跨渠道归因

支柱 2：AI 协同（AI Collaboration）
├── PMax 信号输入：高质量受众信号 + 素材组 + 搜索主题
├── 广泛匹配 + 智能出价：让 AI 发现人工遗漏的搜索意图
├── AI 创意工具：Product Studio + Veo 视频生成
├── 自动创建素材（ACA）：AI 生成的标题和描述
└── 搜索主题（Search Themes）：引导 PMax 的搜索方向

支柱 3：混合策略（Hybrid Strategy）
├── PMax 负责规模化获客和跨渠道覆盖
├── 标准购物负责精准控制和利润保护
├── 搜索广告负责高意图关键词捕获
├── Demand Gen 负责上层漏斗需求创造
└── YouTube 负责品牌建设和视频教育
```

## 三、核心框架一：混合式 PMax 与标准购物协同框架

> **核心理念**：放弃"PMax 唯一"的思维定式，将 PMax 的"肌肉"（规模化获客）与标准购物的"手术刀"（精准控制）相结合。

### 3.1 框架背景

自 2025 年底，Google Ads 的广告排序机制更新，PMax 和标准购物广告系列可以在同等条件下基于 Ad Rank 进行公平竞争，而非 PMax 绝对优先。这为混合策略提供了技术基础。

### 3.2 五大核心策略

#### 策略 1："僵尸"复活（The Zombie Resurrection）

```
目标：激活在 PMax 中无法获得曝光的长尾或新品

适用场景：
├── 拥有较多 SKU、长尾覆盖不足的 DTC 品牌
├── 新品上架后一段时间内点击明显偏低
├── 季节性产品在非旺季被算法忽略
└── 长尾产品有搜索量但 PMax 不分配预算

执行步骤：
1. 识别"僵尸产品"
   ├── 导出 PMax 产品报告
   ├── 筛选：过去一段时间点击明显偏低的产品
   └── 排除：已下架或缺货的产品

2. 创建独立标准购物广告系列
   ├── 命名：[品牌]_Shopping_Zombie_Revival
   ├── 出价策略：最大化点击次数（设置 CPC 上限）
   ├── 日预算：按 SKU 数量、单次点击成本和验证目标调整
   └── 产品组：仅包含"僵尸产品"

3. 数据积累与毕业机制
   ├── 监控周期：留出足够验证窗口
   ├── 毕业标准：达到稳定转化与可接受效率后再移回主结构
   ├── 毕业操作：将产品移回 PMax
   └── 淘汰标准：验证窗口结束后仍无有效信号 → 检查产品页面、定价或需求匹配
4. 持续运营
   ├── 每月更新"僵尸"清单
   ├── 新品自动进入"复活"系列
   └── 追踪毕业率和整体贡献
```

#### 策略 2："品牌"守护者（The Brand Guardian）

```
目标：控制品牌搜索词的 CPC，防止 PMax 过度竞价

适用场景：
├── 品牌词搜索量已具备独立管理价值
├── PMax 品牌词 CPC 明显高于历史正常区间
├── 品牌流量在总转化中的占比异常偏高
└── 竞争对手在品牌词上竞价

执行步骤：
1. 在 PMax 中设置品牌排除
   ├── PMax → 设置 → 品牌排除列表
   ├── 添加自有品牌名称及常见拼写变体
   └── 注意：品牌排除仅影响搜索和购物广告位

2. 创建品牌搜索广告系列
   ├── 命名：[品牌]_Search_Brand
   ├── 关键词：品牌名 + 变体（精确匹配 + 词组匹配）
   ├── 出价策略：手动 CPC 或 eCPC
   ├── CPC 上限：按品牌词竞争强度和历史区间设置
   └── 广告文案：品牌名 + 核心卖点 + 全部附加信息

3. 创建品牌购物广告系列（可选）
   ├── 命名：[品牌]_Shopping_Brand
   ├── 出价策略：手动 CPC
   ├── CPC 上限：按品牌购物位竞争强度设置
   └── 优先级：高（确保品牌搜索优先匹配此系列）

4. 效果监控
   ├── 对比：品牌 CPC 下降幅度
   ├── 对比：品牌流量的 ROAS 变化
   ├── 目标：在不伤害品牌收口的前提下压低品牌 CPC
   └── 警戒：如果品牌转化明显下滑，应复核排除设置与流量承接
```

#### 策略 3："利润"捍卫者（The Margin Defender）

```
目标：保护高利润产品，避免算法为达成 ROAS 目标而牺牲利润

适用场景：
├── 产品利润率差异显著
├── PMax 倾向于推广高客单价但低利润的产品
├── 需要按利润而非收入优化
└── 有明确的贡献利润目标

执行步骤：
1. 在产品 Feed 中标记利润等级
   ├── 自定义标签 0：利润等级（高/中/低）
   ├── 高利润：按品牌财务口径定义
   ├── 中利润：按品牌财务口径定义
   └── 低利润：按品牌财务口径定义

2. 创建高利润产品标准购物系列
   ├── 命名：[品牌]_Shopping_HighMargin
   ├── 产品组：仅包含"高利润"标签产品
   ├── 出价策略：目标 ROAS（设置比 PMax 更高的 ROAS 目标）
   └── 日预算：按高利润产品的收入占比分配

3. PMax 调整
   ├── 在 PMax 中保留中/低利润产品
   ├── 或：使用转化价值规则，为高利润产品设置更高的转化价值
   └── 效果：引导 PMax 算法更重视高利润转化

4. 进阶：利润导向出价
   ├── 使用 Google Ads 的利润优化功能（如可用）
   ├── 或：通过 Feed 中的 cost_of_goods_sold 属性
   └── 目标：从"最大化收入"转向"最大化贡献利润"
```

#### 策略 4："清仓"甩卖（The Clearance Liquidation）

```
目标：快速清理季节性或生命周期末期的库存

适用场景：
├── 季节切换（冬→春、夏→秋）
├── 产品迭代（旧款清仓为新款让路）
├── 库存积压（占用仓储成本）
└── 限时促销（Black Friday 尾货）

执行步骤：
1. 创建清仓专用标准购物系列
   ├── 命名：[品牌]_Shopping_Clearance_[季节/原因]
   ├── 产品组：仅包含需清仓的 SKU
   ├── 出价策略：最大化转化价值（低 ROAS 目标）
   ├── ROAS 目标：按清仓速度与利润容忍度适度下调
   └── 日预算：确保有足够预算支撑清仓速度

2. 配合促销设置
   ├── 在 Feed 中更新 sale_price
   ├── 添加 Merchant Promotions（优惠券/折扣）
   ├── 使用 Google Ads 促销附加信息
   └── 落地页突出"限时清仓"信息

3. 退出机制
   ├── 库存清零 → 自动暂停
   ├── 清仓周期结束 → 暂停系列
   └── 未清完的库存 → 评估是否降价或捐赠
```

#### 策略 5："数据源"捕获（The Feed-Only Catch-All）

```
目标：以极低成本捕获 PMax 忽略的超长尾流量

适用场景：
├── SKU 数量很多，PMax 难以充分覆盖
├── PMax 无法覆盖所有产品的搜索查询
├── 希望以最低成本获取增量流量
└── 作为"安全网"补充 PMax 的覆盖盲区

执行步骤：
1. 创建 Feed-Only 标准购物系列
   ├── 命名：[品牌]_Shopping_CatchAll
   ├── 产品组：所有产品
   ├── 出价策略：手动 CPC
├── CPC 出价：以低成本探索为目标设置较低出价
├── 优先级：低（尽量不与 PMax 和其他系列竞争）
└── 日预算：按探索意图和覆盖范围设置


2. 关键设置
   ├── 不添加任何受众信号
   ├── 不添加任何创意素材
   ├── 纯粹依赖产品 Feed 数据
   └── 让 Google 自动匹配长尾搜索

3. 数据挖掘
   ├── 定期查看搜索词报告
   ├── 发现高转化的长尾词 → 创建专门的搜索广告系列
   ├── 发现高转化的产品 → 移入 PMax 或高利润系列
   └── 发现无效流量 → 添加否定关键词
```

## 四、核心框架二：Demand Gen 最佳实践框架

> **核心理念**：在用户"探索"而非"搜索"的心态下，通过沉浸式视觉创意激发购买意愿。

### 4.1 受众信号叠加策略

```
第 1 层：核心种子受众（优先）
├── 高 LTV 客户列表（按品牌复购定义的老客户）
├── 高价值转化用户（相对品牌平均值明显更高）
└── 创建"类似细分受众群"（Lookalike Segments）

第 2 层：兴趣信号叠加（推荐）
├── 有具体购买意向的细分受众群（In-market）
├── 兴趣相关的亲和力受众（Affinity）
└── 自定义细分受众群（基于竞品 URL + 关键词）

第 3 层：排除规则（优先）
├── 排除已购买用户（按复购周期设置窗口）
├── 排除品牌搜索转化用户
└── 排除低质量流量来源
```

### 4.2 创意三位一体

```
视频为王（核心）：
├── 格式：优先考虑更适配短视频版位的竖版素材
├── 时长：按信息密度与版位节奏决定，前段应尽快建立 Hook
├── 风格：原生、真实、避免硬广感
├── 数量：准备足够覆盖不同角度的素材供系统学习
├── 工具：Google Veo（静态图→动态视频）
└── 刷新频率：按疲劳节奏与产能定期更新

图片补充（重要）：
├── 场景化生活方式图片（Lifestyle Images）
├── 产品使用场景图（非白底产品图）
├── 工具：Product Studio 优化背景
└── 数量：准备足够覆盖不同场景的图片素材

轮播广告（补充）：
├── 适用：多产品展示 / 单产品多特性
├── 每张卡片：独立卖点 + 独立落地页链接
├── 数量：按卖点层次和产品复杂度决定
└── 顺序：优先把最强卖点放前面
```

### 4.3 出价与预算策略

```
出价策略选择：
├── 追求直接转化 → 最大化转化次数 / 目标 CPA
├── 追求品牌考虑 → 最大化点击次数
└── 追求品牌曝光 → 目标 CPM（不推荐 DTC）

预算规则：
├── 日预算应足以支撑当前目标策略稳定学习
├── 学习期：留出完整观察窗口，不频繁做重大调整
├── 扩量：逐步增加，并在每次调整间保留观察间隔
└── 缩量：当效率显著恶化且持续一段时间后再谨慎下调

效果评估（重要）：
├── 不要仅看直接转化 ROAS
├── 应同时看 View-through Conversions（浏览后转化）
├── 应优先用 DDA 2.0 归因评估全链路贡献
├── 核心指标：辅助转化次数 + 辅助转化价值
└── 正确心态：Demand Gen 是"投资"不是"收割"
```

### 4.4 落地页体验一致性

```
核心原则：广告创意 → 落地页 = 无缝过渡

检查清单：
├── 视觉风格一致：广告图片/视频的色调 = 落地页色调
├── 信息一致：广告承诺的卖点 = 落地页首屏内容
├── 优惠一致：广告中的折扣/赠品 = 落地页可见
├── 情感一致：广告的情绪基调 = 落地页的情绪基调
└── 体验升级：落地页应比广告提供更丰富的内容

推荐落地页类型：
├── 冷流量 → 教育型落地页（故事 + 社证 + 产品）
├── 温流量 → 产品集合页（多选择 + 对比）
├── 热流量 → 产品详情页（直接购买）
└── 高 AOV → 互动式落地页（测评 + 推荐）
```

## 五、核心框架三：DDA 2.0 数据驱动归因

> **核心理念**：从"最终点击归功"到"全链路价值评估"，让每个触点获得公平的功劳分配。

### 5.1 DDA 2.0 核心原理

```
传统归因（最终点击）：
用户旅程：YouTube 视频 → 搜索广告 → 品牌搜索 → 购买
功劳分配：    几乎全部归功于最后一触点       ←  不公平！

DDA 2.0 归因：
用户旅程：YouTube 视频 → 搜索广告 → 品牌搜索 → 购买
功劳分配：   按模型动态分配，不再用固定比例归功       ←  更公平！

核心问题："如果缺少这个触点，用户转化的概率会降低多少？"
```

### 5.2 跨渠道整合能力

```
DDA 2.0 整合的数据源：
├── 搜索广告点击
├── 购物广告点击
├── YouTube 视频观看（即使未点击）
├── Demand Gen 展示互动
├── Display 展示和点击
├── GA4 事件数据（网站行为）
├── 增强型转化数据（第一方匹配）
└── 离线转化数据（如有上传）

关键升级：
├── YouTube 视频观看纳入归因（不再是"看不见的贡献"）
├── 跨设备追踪（手机看视频 → 电脑购买）
├── 与 GA4 深度融合（网站行为 + 广告数据统一分析）
└── 向 LTV 倾斜（高价值客户的触点获得更高权重）
```

### 5.3 基于 DDA 2.0 的预算分配

```
步骤 1：查看 DDA 归因报告
├── Google Ads → 工具 → 归因 → 转化路径
├── GA4 → 广告 → 归因路径
└── 关注"辅助转化"和"路径长度"

步骤 2：识别被低估的渠道
├── 如果 Demand Gen 的辅助转化显著高于直接转化 → 可能被低估
├── 如果 YouTube 的辅助转化价值高于直接转化价值 → 可能被低估
└── 如果品牌搜索的直接转化占比异常偏高 → 其他渠道贡献可能被忽视

步骤 3：调整预算分配
├── 增加被低估渠道的预算（小步调整）
├── 减少过度依赖品牌搜索的预算
├── 设置完整观察期
└── 用"增量测试"验证调整效果

步骤 4：向 LTV 导向转变
├── 识别高 LTV 客户的获取路径
├── 为这些路径上的触点分配更多预算
├── 接受更高的初始 CPA（如果 LTV 证明值得）
└── 长期目标：最大化客户资产而非单次 ROAS
```

## 六、核心框架四：生成式引擎优化（GEO）

> **核心理念**：从"为关键词排名优化"到"成为 AI 生成答案的权威信息源"。

### 6.1 GEO 与传统 SEO 的区别

```
传统 SEO：                              GEO：
为搜索爬虫优化                    →     为 AI 的理解和采纳而优化
目标：链接列表中的高排名           →     目标：成为 AI 答案的信息源
关键词密度和反向链接               →     内容权威性和结构化数据
页面级优化                         →     品牌内容生态系统级优化
```

### 6.2 GEO 五步实施框架

```
步骤 1：建立权威内容中心
├── 围绕品牌核心领域创建"知识库"
├── 内容类型：深度指南 + 白皮书 + 视频教程 + FAQ
├── 质量标准：E-E-A-T（经验、专业、权威、可信）
├── 更新频率：按内容时效性与竞争变化定期更新
└── 目标：成为该领域可信的信息源

步骤 2：结构化数据全面标记
├── Product Schema：产品属性、价格、库存、评分
├── FAQ Schema：常见问题和详细回答
├── HowTo Schema：使用教程和操作指南
├── Review Schema：用户评价和评分
├── Article Schema：博客文章和指南
└── Organization Schema：品牌信息和联系方式

步骤 3：多模态内容供给
├── 原创高质量图片（非库存图）
├── 产品使用教程视频
├── 信息图表和数据可视化
├── 播客和音频内容（如适用）
└── 目标：为 AI 提供多种格式的"养料"

步骤 4：第一方评论与 UGC 生态
├── 积极收集真实用户评论
├── 鼓励用户上传使用照片/视频
├── 创建 Q&A 社区（产品页面内）
├── 展示专家/KOL 的推荐和评测
└── 目标：为 AI 提供"社会证明"信号

步骤 5：跨平台信号一致性
├── 官网、社交媒体、行业论坛信息一致
├── Google Business Profile 完整准确
├── 第三方评论平台（Trustpilot 等）积极管理
├── 合作伙伴网站的品牌信息准确
└── 目标：让 AI 从多个来源验证品牌权威性
```

### 6.3 GEO 与 Google Ads 的协同

```
GEO 为 Google Ads 带来的增益：
├── AI Overviews 中的品牌曝光 → 提升品牌搜索量 → 降低品牌 CPC
├── 结构化数据 → 更丰富的购物广告展示 → 提升 CTR
├── 高质量内容 → 更好的着陆页体验 → 提升质量得分 → 降低 CPC
├── 用户评论 → 卖家评分 → 购物广告星级展示 → 提升 CTR
└── 品牌权威性 → 更高的自然排名 → 减少对付费流量的依赖

Google Ads 为 GEO 带来的增益：
├── 搜索词报告 → 发现用户的真实问题 → 创建对应内容
├── 广告测试数据 → 验证哪些卖点最有效 → 优化内容方向
├── Demand Gen → 推广内容中心 → 加速内容被 AI 索引
└── YouTube 广告 → 视频内容分发 → 多模态信号积累
```
