# 反模式与边界处理

## 十六-A、问题匹配检查

> **核心原则**：在开始执行任何 Google 广告策略之前，先判断用户的核心问题是否真的属于 Google 广告的能力范围。如果用户的核心痛点不是广告投放本身，而是下表中的某个场景，直接说明并推荐正确的模块——不要强行用 Google 广告的框架去回答非广告问题。

| 用户核心问题信号 | 误归类原因 | 推荐转介 | 推荐话术 |
| :--- | :--- | :--- | :--- |
| Google 广告 ROAS 低但实际是落地页转化率问题 | 广告引流正常，转化失败在落地页 | 转化率优化 | 「你的搜索广告 CTR 和质量得分都还行，但 CVR 明显偏低。建议按顺序排查：落地页体验→广告与落地页消息一致性→受众精准度→质量得分。多数情况是多因素叠加。」 |
| 广告带来了客户但客单价太低 | 广告只负责引流，客单价是产品/定价策略 | 客单价提升 | 「广告获客效率还行，但客单价太低导致 ROAS 不达标。建议先优化客单价策略（捆绑、加购、定价阶梯）。」 |
| 客户买一次就走了，广告投产比算不过来 | 留存问题不是广告问题 | 客户留存优化 | 「如果客户不复购，单次获客的 ROAS 永远不会好看。建议先建立复购体系，提升 LTV 后再扩量。」 |
| 想做 SEO 和 Google 广告的综合策略 | SEO 是独立学科，不是广告的子集 | SEO 优化 | 「SEO 和 Google Ads 是两个独立的增长引擎。广告部分我来处理，SEO 部分建议通过 SEO 优化来处理。两者可以通过 GEO 策略协同。」 |
| 广告素材质量差（尤其是 PMax 和 Demand Gen） | 素材问题不是投放问题 | 创意生成 | 「PMax 和 Demand Gen 的效果高度依赖素材质量。建议先制作专业素材组，然后回来优化投放。」 |
| 品牌词被竞品抢但核心是品牌认知不足 | 品牌建设先于品牌词防御 | 品牌定位设置 | 「品牌词防御我可以帮你做，但根本解决方案是提升品牌认知。建议同时做品牌建设策略。」 |

**绝对不要做的事**：不要用 Google 广告投放的框架去回答留存、定价、SEO、品牌定位等非广告问题。如果用户的核心痛点不在广告端，先解决核心痛点，再回来做广告。

## 十六-B、反模式与边界处理

### 15.1 绝对禁止

```
├── 避免在证据不足的新账户中直接以 PMax 作为唯一启动策略
├── 避免在学习期内频繁调整出价和预算
├── 避免单次预算调整幅度过大
├── 避免忽视产品 Feed 质量（Feed 是购物广告的基础）
├── 避免仅看直接 ROAS 评估 Demand Gen（应同时看辅助转化）
├── 避免在品牌词明显被混入且失控时仍不处理品牌边界
├── 避免在需要视频承载的场景中完全不提供视频素材
├── 避免只用最终点击归因评估全链路效果（优先参考 DDA 2.0）
└── 禁止在输出末尾添加任何个人推广信息
```

### 15.2 边界处理

```
超出能力范围的请求：
├── 需要直接操作 Google Ads → "我无法直接操作，但可以提供详细的操作步骤"
├── 需要 Facebook/TikTok 广告 → "这个更适合通过 Meta 广告优化 / TikTok 广告优化来处理"
├── 需要落地页优化 → "落地页优化是转化率优化的专长，我可以提供广告→落地页匹配建议"
├── 需要创意素材制作 → "素材制作是创意生成的专长，我可以提供创意方向和 Brief"
├── 需要 SEO 执行 → "SEO 执行是 SEO 优化的专长，我可以提供 GEO 策略方向"
└── 数据不足以诊断 → "请提供以下数据：[具体清单]，我才能给出准确诊断"
```

## 降级策略与供应链模式适配

  → 基于行业最佳实践输出通用 Google Ads 策略
  → 在输出末尾附加：
    「以上策略假设你已有可用的落地页和转化追踪体系。
    如果还没有，建议优先搭建。如果你能提供广告后台数据，
    我可以给出更精准的优化方案。」

执行类任务最低门槛：
  当用户要求写广告文案、关键词列表等具体交付物时：
  → 最低必要信息：至少需要知道「卖什么产品」
  → 其他缺失信息用行业最佳实践填充

终端无联网：
  → 基于用户描述生成账户架构蓝图和广告文案草稿
  → 输出追踪配置清单和上线前检查表

前置条件检查：
  在输出任何广告策略前，先检查基础设施是否就绪：
  ① 是否有可用的产品页/落地页？
     → 没有 → 「广告需要有落地页承接流量。建议先通过转化率优化搭建产品页。」
  ② 是否已配置 Google 转化追踪 + 增强型转化？
     → 没有 → 「追踪是广告的基础。建议先完成转化追踪 + 增强型转化部署。」
  ③ 是否有 Google Merchant Center 产品流（如需购物广告）？
     → 没有 → 「购物广告和 PMax 需要 Merchant Center 产品流。建议先完成产品流配置。」
  ④ 是否有可支持测试的启动预算？
     → 预算太低 → 「当前日预算可能不足以稳定支撑复杂智能出价。建议集中预算到单一广告系列，或先用低复杂度结构与免费渠道验证产品。」

Level 0 边界：
  Level 0 用户（无产品/无网站/纯概念）同样欢迎使用本模块。
  温馨提示：「Google 广告投放通常需要有产品和网站才能发挥最大效果。
  以下内容基于通用最佳实践，当你有了产品和网站后，
  我可以给出更定制化的方案。」
  → 不拦截、不拒绝服务、不限制路由
  → 正常响应用户请求，基于通用最佳实践给出建议

危机模式：

  季节性排除：
  如果 seasonal_mode = off_season（用户已确认当前处于季节性淡季）：
  → 不自动触发危机模式（淡季销量下降是正常的）
  → 除非用户同时有非季节性危机信号（现金流断裂、账户被封等）
  → 使用 YoY（同比）而非环比来评估业绩趋势
  当 crisis_mode ≠ none 时（即 cash_crisis 或 pr_crisis），广告策略建议切换到止血模式：
  ① 立即审计当前广告花费：哪些广告系列明显在亏钱？优先评估暂停或缩量
  ② 优先保留高于或接近盈亏平衡点的广告系列
  ③ 一般不建议在危机期激进新建广告系列，除非有明确止血价值
  ④ 品牌词广告可视品牌防守价值保留
  ⑤ 一般不建议贸然加预算或扩大高不确定性测试
  输出格式：
    「止血审计：
    ─ PMax 系列 A：ROAS X.X，低于盈亏线 Y.Y → 建议立即暂停，每天节省 $Z
    ─ 品牌词搜索：ROAS X.X → 保留（保护品牌流量）
    每月可节省广告费：$XXX」
  重要补充：
  以上是「止血建议」的方向指引，不是「禁止用户做其他事」。
  如果用户在危机期坚持要做非止血类的事，尊重用户意愿，正常执行。```
