# Google Ads 诊断规则库

> **层级声明**：本文件默认作为 **internal-only** 的内部诊断规则参考使用，不得将诊断树、阈值、窗口期、预算比例或行动指令整段直接粘贴给用户。
> **用途**：本文件为 Google 广告模块的内部参考文件，用于诊断账户问题，覆盖搜索、购物、PMax、Demand Gen 四大广告类型。
> **渲染规则**：本文件中的低/高判定、质量得分、页面速度、平均位置、展示份额、预算倍数、学习期长度、品牌词占比与差异百分比，只能作为排查起点或待验证假设；不得写成跨品牌统一红线或唯一根因。

## 一、搜索广告诊断树

### 1.1 CTR 低（相对品类参考或账户基线偏弱）

```
CTR 低
├── 质量得分是否明显偏弱？
│   ├── 是 → 检查三个维度
│   │   ├── 预期点击率"低于平均" → 重写广告文案
│   │   │   ├── 标题是否包含关键词？
│   │   │   ├── 是否有明确的价值主张？
│   │   │   ├── 是否有行动号召？
│   │   │   └── 是否充分利用可用标题位？
│   │   ├── 广告相关性"低于平均" → 检查关键词-广告匹配
│   │   │   ├── 广告组关键词是否过于宽泛？
│   │   │   ├── 关键词是否出现在标题中？
│   │   │   └── 是否需要拆分广告组？
│   │   └── 着陆页体验"低于平均" → 优化着陆页
│   │       ├── 着陆页是否与搜索意图匹配？
│   │       ├── 页面加载速度是否明显拖累体验？
│   │       ├── 移动端体验是否良好？
│   │       └── 是否有清晰的 CTA？
│   └── 否 → 检查其他因素
│       ├── 广告排名是否偏弱？
│       │   ├── 提高出价或预算
│       │   └── 提高质量得分
│       ├── 广告扩展不完整？
│       │   ├── 确保使用所有相关扩展
│       │   └── 站点链接、摘要、促销、电话等
│       ├── 关键词匹配类型过宽？
│       │   ├── 检查搜索词报告
│       │   ├── 添加否定关键词
│       │   └── 考虑收紧匹配类型
│       └── 竞争加剧？
│           ├── 检查竞争对手报告
│           └── 差异化广告文案
```

### 1.2 CPC 高（相对品类参考或账户基线明显偏高）

```
CPC 高
├── 质量得分是否偏弱？
│   ├── 是 → 提高质量得分（见 1.1）
│   └── 否 → 检查其他因素
│       ├── 竞争激烈？
│       │   ├── 检查竞争对手数量和出价
│       │   ├── 考虑长尾关键词策略
│       │   └── 考虑非高峰时段投放
│       ├── 关键词过于宽泛？
│       │   ├── 添加否定关键词
│       │   ├── 使用更精准的匹配类型
│       │   └── 拆分高 CPC 关键词到独立广告组
│       ├── 出价策略不当？
│       │   ├── 目标 ROAS 设置过高？
│       │   ├── 目标 CPA 设置过低？
│       │   └── 考虑手动 CPC 测试基准
│       └── 着陆页体验差？
│           ├── 着陆页不相关 → 创建专用着陆页
│           └── 着陆页速度慢 → 优化页面速度
```

### 1.3 CVR 低（相对品类参考或账户基线偏弱）

```
CVR 低
├── 搜索意图匹配吗？
│   ├── 否 → 检查搜索词报告
│   │   ├── 大量信息型搜索词？ → 添加否定关键词
│   │   ├── 大量无关搜索词？ → 添加否定关键词
│   │   └── 搜索词与产品不匹配？ → 调整关键词
│   └── 是 → 检查着陆页
│       ├── 着陆页与广告承诺一致吗？
│       │   ├── 广告说"免费送货"但着陆页没有？
│       │   ├── 广告承诺明确折扣但着陆页未体现？
│       │   └── 修复不一致 → 有助于改善 CVR
│       ├── 着陆页 UX 问题？
│       │   ├── 移动端体验差？
│       │   ├── CTA 不明显？
│       │   ├── 页面加载慢？
│       │   ├── 信任信号不足？
│       │   └── → 优先优化着陆页体验、信任信号与转化路径
│       ├── 价格竞争力？
│       │   ├── 竞品价格更低？
│       │   ├── 价值感不足？
│       │   └── → 考虑价格策略或增强价值展示
│       └── 产品页面问题？
│           ├── 产品描述不充分？
│           ├── 产品图片质量差？
│           ├── 缺少评论/社会证明？
│           └── → 优先补强产品页说服力与转化承接
```

## 二、购物广告诊断树

### 2.1 展示量低

```
展示量低
├── 产品被拒登？
│   ├── 检查 Merchant Center → 诊断
│   ├── 修复所有被拒产品
│   └── 常见原因：价格不匹配、图片不符合要求、缺少标识符
│
├── Feed 质量差？
│   ├── 标题不含关键词？ → 优化标题
│   ├── 描述过短？ → 充实描述
│   ├── 图片质量低？ → 更新图片
│   ├── 属性不完整？ → 填写所有属性
│   └── → 参考 feed-optimization.md
│
├── 出价过低？
│   ├── 检查展示份额
│   ├── 如果展示份额长期偏低 → 结合利润与目标提高出价/预算
│   └── 考虑按产品利润分层出价
│
├── 否定关键词过多？
│   ├── 检查是否误排除了有效搜索词
│   └── 清理过度否定
│
└── 产品竞争力？
    ├── 价格远高于竞品？
    ├── 缺少促销标注？
    └── 缺少评论星级？
```

### 2.2 CTR 低（相对品类参考或账户基线偏弱）

```
CTR 低
├── 产品图片问题？
│   ├── 图片模糊/低质量？ → 更新高清图
│   ├── 图片不吸引人？ → 测试不同角度/风格
│   └── 竞品图片更好？ → 对标优化
│
├── 标题问题？
│   ├── 标题不包含搜索关键词？ → 优化标题
│   ├── 标题过于通用？ → 添加差异化属性
│   └── 标题过长被截断？ → 关键信息前置
│
├── 价格竞争力？
│   ├── 价格明显高于竞品？ → 考虑价格策略
│   ├── 缺少促销标注？ → 设置 Merchant Promotions
│   └── 缺少免费送货标注？ → 添加送货信息
│
├── 评论星级？
│   ├── 缺少评论？ → 接入 Google 评论合作伙伴
│   ├── 评分低？ → 改善产品/服务质量
│   └── 竞品评论更多/更好？ → 加强评论收集
│
└── 广告位问题？
    ├── 展示在低可见位置？ → 提高出价
    └── 主要展示在 Display 而非搜索？ → 检查系列设置
```

## 三、PMax 诊断树

### 3.1 ROAS 低于目标

```
ROAS 低
├── 品牌词蚕食？
│   ├── 检查搜索词洞察
│   ├── 品牌词占比是否异常偏高？
│   │   ├── 是 → 设置品牌排除（见 pmax-playbook.md）
│   │   └── 否 → 继续排查
│   └── 排除品牌词后重新计算 ROAS
│
├── 素材质量差？
│   ├── 检查素材评分
│   ├── 有"低"评分素材？ → 结合实际表现优先替换
│   ├── 缺少视频？ → 视版位与策略补充视频素材
│   └── 素材数量不足？ → 按投放规模与疲劳节奏补充
│
├── 受众信号不精准？
│   ├── 客户匹配列表是否最新？
│   ├── 自定义细分是否准确？
│   ├── 搜索主题是否相关？
│   └── 调整受众信号并留出足够观察窗口
│
├── 转化追踪问题？
│   ├── 增强型转化是否开启？
│   ├── 转化值是否正确传递？
│   ├── 是否有重复计数？
│   ├── GA4 vs Google Ads 差异是否异常偏大？
│
├── 预算问题？
│   ├── 日预算是否不足以支撑当前目标策略？ → 评估增加预算
│   ├── 预算利用率是否长期偏低？ → 可能受众过窄或出价过低
│   └── 频繁受限？ → 增加预算或降低出价目标
│
└── 产品问题？
    ├── 高利润产品获得足够展示吗？
    ├── 低利润产品消耗过多预算？
    └── → 使用自定义标签分层管理
```

### 3.2 学习期过长（长时间无法退出）

```
学习期过长
├── 转化量不足？
│   ├── 月转化是否不足以支撑稳定学习？
│   │   ├── 增加预算
│   │   ├── 适度放宽出价目标
│   │   ├── 使用"最大化转化次数"替代"目标 ROAS"
│   │   └── 如果仍不够 → 先用搜索/购物积累数据
│   └── 转化量已相对充足但仍在学习？
│       ├── 是否频繁调整设置？ → 停止调整，并留出完整观察窗口
│       └── 转化追踪是否稳定？ → 检查追踪代码
│
├── 频繁调整？
│   ├── 每次重大调整都会重置学习期
│   ├── "重大调整"包括：
│   │   ├── 预算变化幅度过大
│   │   ├── 出价目标变化幅度过大
│   │   ├── 素材组大幅更换
│   │   └── 受众信号大幅调整
│   └── 解决：制定稳定观察期，只做小幅优化
│
└── 数据质量问题？
    ├── 转化延迟过长？ → 检查转化窗口设置
    ├── 转化值波动大？ → 检查价值传递逻辑
    └── 有大量微转化干扰？ → 清理转化操作列表
```

## 四、Demand Gen 诊断树

### 4.1 CTR 低（相对格式参考或账户基线偏弱）

```
CTR 低
├── 创意吸引力不足？
│   ├── 图片/视频是否有强 Hook？
│   ├── 是否使用了 UGC 风格内容？
│   ├── 是否测试了不同创意方向？
│   └── → 参考创意优化与素材迭代规范
│
├── 受众不精准？
│   ├── 受众范围是否过宽？
│   ├── 是否使用了客户匹配列表？
│   ├── 相似受众扩展是否过度？
│   └── 结合覆盖质量与量级平衡收紧受众定位
│
└── 广告格式不匹配？
    ├── YouTube 广告位用了静态图片？ → 改用视频
    ├── Discover 广告位用了横版图片？ → 改用方形/竖版
    └── 测试不同格式在不同广告位的表现
```

### 4.2 转化归因困难

```
转化归因困难
├── 理解 Demand Gen 的角色
│   ├── 上层漏斗 → 直接转化少是正常的
│   ├── 重点看辅助转化和 View-through
│   └── 使用数据驱动归因（DDA）而非末次点击
│
├── 检查辅助转化
│   ├── GA4 → 转化路径 → 辅助转化
│   ├── Demand Gen 的辅助转化倍数需结合归因口径与品牌成熟度理解，不写固定倍数
│   └── 如果辅助转化也很少 → 可能受众不对
│
├── 检查 View-through 转化
│   ├── Google Ads → 列 → 修改列 → 浏览型转化
│   ├── 添加浏览型转化列
│   └── 评估浏览型转化的价值
│
└── 增量测试
    ├── 使用 Google 的转化提升测试
    ├── 或使用地理测试（A/B 地区对比）
    └── 评估 Demand Gen 的真实增量贡献
```

## 五、跨系列诊断

> 合规说明：凡涉及 Cookie、受众信号、增强型转化、再营销或服务器端追踪的能力，均应先确认目标市场的合法性基础、同意状态与平台政策符合性；合规未闭环时，不得将“数据缺失”简单归因为技术问题并继续放量。


### 5.1 预算分配诊断

```
预算分配检查：
├── 品牌词系列占总预算是否异常偏高？
│   └── 问题：品牌词预算可能过高，挤压增长预算
│       └── 结合品牌成熟度降低品牌词出价或释放预算给非品牌
│
├── PMax 占总预算是否异常偏高？
│   └── 问题：可能过度依赖 PMax，缺乏控制
│       └── 结合控制需求分配部分预算给标准搜索/购物
│
├── 单一系列消耗是否过度集中？
│   └── 问题：集中风险过高
│       └── 视结构分散到多个系列
│
├── 多个系列竞争同一关键词？
│   └── 问题：内部竞争推高 CPC
│       └── 使用否定关键词划分边界
│
└── 学习期系列占比是否过高？
    └── 问题：调整可能过于频繁
        └── 减少调整频率，并留出稳定观察期
```

### 5.2 转化追踪健康检查

```
追踪健康检查：
├── □ 增强型转化已开启？
├── □ 转化值正确传递？（检查几笔最近订单）
├── □ 无重复计数？（GA4 vs Google Ads 差异是否处于可解释范围）
├── □ 转化窗口设置合理？（按渠道角色与购买周期校准）
├── □ 微转化未混入主要转化？
├── □ 服务器端追踪是否按资源与合规条件部署？
├── □ 同意模式已按目标市场的适用法规与平台要求正确配置，并完成实测验证？如未确认，应按更严格基线处理并暂停依赖相关追踪数据做投放判断
└── □ 转化延迟已考虑？（等待完整归因窗口后再评估）
```
