# Google 产品 Feed 优化手册

> **层级声明**：本文件默认作为 **internal-only** 的内部 Feed 优化参考使用，不得将字段公式、字数门槛、图片比例、标签分箱或更新频率整段直接粘贴给用户。
> **用途**：本文件为 Google 广告模块的内部参考文件，用于优化购物广告和 PMax 的 Feed 质量。产品 Feed 是购物广告和 PMax 的基础，Feed 质量直接决定广告表现。
> **渲染规则**：本文件中的字符长度、图片尺寸、面积占比、附加图数量、利润率分箱、价格分层与更新频率，只能作为说明性样例或排查起点；不得写成跨品牌统一配方或固定红线。

## 一、产品标题优化

### 1.1 标题公式（按品类）

```
通用公式：[品牌] + [产品类型] + [关键属性] + [尺寸/颜色/材质]

美妆护肤：
├── [品牌] + [产品类型] + [核心成分] + [功效] + [容量]
├── 示例："CeraVe Moisturizing Cream with Ceramides for Dry Skin 16oz"
└── 关键：成分和功效是搜索高频词

服饰鞋帽：
├── [品牌] + [产品类型] + [材质] + [风格] + [颜色] + [尺码范围]
├── 示例："Allbirds Tree Runner Lightweight Running Shoes Mint Green Women's US5-11"
└── 关键：材质、风格、颜色是关键筛选条件

健康保健：
├── [品牌] + [产品类型] + [核心成分] + [剂量] + [数量]
├── 示例："Athletic Greens AG1 Daily Greens Powder 30 Servings Pouch"
└── 关键：成分和剂量是专业用户的搜索词

食品饮料：
├── [品牌] + [产品类型] + [口味] + [特性] + [包装规格]
├── 示例："OLIPOP Prebiotic Soda Vintage Cola Low Sugar 12-Pack 12oz Cans"
└── 关键：口味和特性（低糖、有机等）是高频词

家居生活：
├── [品牌] + [产品类型] + [材质] + [尺寸] + [颜色/风格]
├── 示例："Brooklinen Luxe Sateen Sheet Set 100% Cotton King Size Cream"
└── 关键：材质、尺寸通常是重要属性

宠物用品：
├── [品牌] + [产品类型] + [适用宠物] + [特性] + [规格]
├── 示例："The Farmer's Dog Fresh Dog Food Chicken Recipe Small Breed 2-Week Supply"
└── 关键：适用宠物和特性是关键筛选条件
```

### 1.2 标题优化清单

```
优先包含：
├── □ 品牌名（通常放在前面）
├── □ 产品类型（用户搜索的核心词）
├── □ 关键属性（材质/成分/功效等）
├── □ 规格属性（尺寸/颜色/容量等）
└── □ 在可读前提下充分利用可用标题空间

避免：
├── □ 全大写字母
├── □ 促销信息（如折扣口号、免邮口号等）
├── □ 特殊符号（★、♥、→）
├── □ 过度关键词堆砌
└── □ 与描述完全重复
```

## 二、产品描述优化

### 2.1 描述结构模板

```
段落 1（核心卖点，前段最重要）：
[产品名称] 是 [品牌] 的 [核心定位]。[1-2 句核心卖点，
包含目标关键词]。

段落 2（产品特性）：
主要特性包括：[特性 1]、[特性 2]、[特性 3]。
[每个特性用 1 句话展开说明]。

段落 3（使用场景）：
适合 [目标人群] 在 [使用场景] 中使用。
[1-2 句场景化描述]。

段落 4（规格参数）：
规格：[尺寸/重量/容量/成分]。
包装内含：[清单]。
```

### 2.2 描述优化清单

```
优先包含：
├── □ 核心关键词（自然融入，不堆砌）
├── □ 产品核心卖点（优先放在前段）
├── □ 产品特性和规格
├── □ 使用场景描述
└── □ 在可读前提下充分利用可用描述空间

避免：
├── □ HTML 标签
├── □ 外部链接
├── □ 促销信息和价格
├── □ 与竞品的直接对比
└── □ 过度营销语言
```

## 三、产品图片优化

### 3.1 图片要求

```
主图（优先遵循平台要求）：
├── 白底产品图（纯白背景通常更稳妥）
├── 分辨率：达到平台要求并兼顾清晰度
├── 产品占图片面积：以主体清晰可辨且不过度留白为宜
├── 无文字覆盖、无水印、无边框
├── 格式：JPEG 或 PNG
└── 文件大小：符合平台上传限制

附加图（强烈推荐，数量按素材复杂度决定）：
├── 图 2：产品不同角度
├── 图 3：产品使用场景（生活方式图）
├── 图 4：产品细节特写
├── 图 5：产品包装/配件
├── 图 6：尺寸对比参考
├── 图 7：成分/材质特写
├── 图 8-10：更多场景/角度
└── 如包含文字，应避免明显影响审核与展示效果
```

### 3.2 Product Studio 优化

```
Google Product Studio 功能：
├── 背景替换：将白底图替换为场景化背景
├── 图片增强：提升图片质量和清晰度
├── 场景生成：AI 生成产品使用场景
└── 适用：Merchant Center 中直接使用

使用建议：
├── 为每个产品准备足够的场景化背景备选
├── 选择与目标受众生活方式匹配的场景
├── 保持品牌视觉一致性
└── A/B 测试不同场景的 CTR 差异
```

## 四、自定义标签策略

### 4.1 五个自定义标签的推荐用法

```
custom_label_0：利润等级
├── "high_margin"：按品牌财务口径定义高利润
├── "medium_margin"：按品牌财务口径定义中利润
├── "low_margin"：按品牌财务口径定义低利润
└── 用途：创建利润导向的购物系列

custom_label_1：产品生命周期
├── "new_arrival"：按品牌新品观察窗口定义
├── "core"：常规在售产品
├── "seasonal"：季节性产品
├── "clearance"：清仓产品
└── 用途：僵尸复活 + 清仓系列

custom_label_2：畅销度
├── "bestseller"：按销量表现定义头部产品
├── "steady"：按销量表现定义稳定产品
├── "longtail"：按销量表现定义长尾产品
└── 用途：预算分配优化

custom_label_3：价格区间
├── "premium"：按品牌价格体系定义高价带
├── "mid_range"：按品牌价格体系定义中价带
├── "entry"：按品牌价格体系定义入门价带
└── 用途：按价格区间设置不同 ROAS 目标

custom_label_4：促销状态
├── "full_price"：正价
├── "on_sale"：促销中
├── "bundle"：捆绑销售
└── 用途：促销系列管理
```

## 五、Merchant Center 健康检查

### 5.1 常见问题与修复

```
产品被拒登：
├── 价格不匹配 → 确保 Feed 价格 = 网站价格（含货币）
├── 图片不符合要求 → 主图应优先保持白底、无文字、无水印
├── 缺少 GTIN/MPN → 优先提供可用的产品标识符
├── 着陆页 404 → 检查产品 URL 是否有效
├── 违反政策 → 检查产品描述是否包含禁止内容
└── 库存不匹配 → 确保 Feed 库存状态 = 实际库存

产品表现差：
├── 标题不含关键词 → 按品类公式优化标题
├── 图片质量低 → 更新为高分辨率产品图
├── 描述过短 → 充实描述到足以支持搜索匹配与转化理解
├── 缺少附加图 → 按产品复杂度补充场景图
├── 缺少评论 → 接入 Google 评论合作伙伴
└── 属性不完整 → 填写所有可选属性

Feed 更新频率：
├── 价格/库存：保持足够高的同步频率，避免明显滞后
├── 产品信息：有变更时尽快更新
├── 新品：上架后尽快同步到 Feed
└── 推荐：如条件允许，使用更实时的同步方式
```

## 六、结构化数据标记

### 6.1 Product Schema 模板

```json
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "[产品标题]",
  "image": ["[主图URL]", "[附加图URL]"],
  "description": "[产品描述]",
  "brand": {
    "@type": "Brand",
    "name": "[品牌名]"
  },
  "sku": "[SKU]",
  "gtin13": "[GTIN]",
  "offers": {
    "@type": "Offer",
    "url": "[产品页URL]",
    "priceCurrency": "USD",
    "price": "[价格]",
    "availability": "https://schema.org/InStock",
    "seller": {
      "@type": "Organization",
      "name": "[品牌名]"
    }
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "[评分]",
    "reviewCount": "[评论数]"
  }
}
```
