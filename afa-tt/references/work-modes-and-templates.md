## 十、衡量与 KPI

```text
TikTok 广告核心 KPI 体系：

  广告投放指标：
  ├── CPM：结合品类、地区、竞价环境和投放阶段与账户历史数据一起判断，不把单一平台均值当作硬标准
  ├── CTR：结合创意形态、版位、受众成熟度和历史波动区间判断相对健康度
  ├── CVR：优先与站内真实漏斗表现、产品页体验和同账户历史区间对比，不写统一健康线
  ├── CPA：以品牌可承受获客成本、毛利结构和回收周期为主，不预设统一上限
  └── ROAS：结合盈亏平衡点、新客占比、复购结构和投放目标综合判断

  创意指标：
  ├── Hook Rate（前段留存）：根据视频长度、叙事方式与历史优胜素材建立相对比较，不固化单一秒数或阈值
  ├── Hold Rate（完播率）：结合内容长度、内容密度和平台原生程度判断，不把单一完播目标写死
  ├── 创意疲劳周期：观察 CTR、CPM、频次、互动质量和转化效率的连续变化后再判断，不设统一天数和跌幅
  └── 创意胜率：以同批测试素材中的相对超越幅度为准，不写固定胜率门槛

  TikTok Shop 指标：
  ├── GMV Max ROAS：结合商品毛利、退货率、补贴和目标阶段判断，不写统一目标值
  ├── Affiliate 转化率：按品类、达人类型、内容形态和报价结构综合判断
  ├── 视频购物 CVR：与店铺页面质量、定价策略和内容匹配度联动分析
  └── 直播购物 CVR：结合流量质量、主播能力、产品结构和直播目标判断

  账户健康指标：
  ├── Pixel 事件匹配率：优先看事件完整性、关键事件可用性和同账户趋势，不固化单一百分比
  ├── 广告组学习期通过率：结合预算规模、转化密度和账户阶段判断，不写统一通过率
  └── 素材储备：根据预算体量、素材疲劳速度和产能决定更新节奏，不设固定周产量
```

## 十一、工作模式

### 模式 1：TikTok Ads 架构设计

**触发**：新品牌/新产品启动 TikTok 广告

**流程**：
1. 收集品牌信息（产品、受众、预算、目标）
2. 设计账户架构（Campaign → Ad Group 结构）
3. 制定受众策略（Broad + Interest + Lookalike）
4. 规划创意需求（格式 × 测试覆盖范围 × 节奏，数量按预算、素材缺口与产能确定）
5. 设置追踪方案（Pixel + Events API + UTM）
6. 制定预算分配方案
7. 输出完整的启动计划文档

### 模式 2：创意策略制定

**触发**：需要新的创意方向/Hook/脚本

**流程**：
1. 分析当前创意表现数据
2. 识别优胜创意的成功要素
3. 研究竞品/行业最新创意趋势
4. 生成新的 Hook 矩阵
5. 按当前测试需求撰写足够支持决策的广告脚本，数量由预算、素材缺口和测试范围决定
6. 制定创意测试计划
7. 输出 Creative Brief（可交给创作者/团队）

### 模式 3：诊断与优化

**触发**：广告表现下降/未达目标

**流程**：
1. 收集关键数据（CPM、CTR、CVR、ROAS、频次）
2. 运行 CPM-CTR-CVR 诊断链
3. 检查创意疲劳信号
4. 检查账户健康状态
5. 生成诊断报告 + 优化行动方案
6. 如需跨模块协同 → 路由到对应 Skill

### 模式 4：TikTok Shop 运营

**触发**：TikTok Shop 设置/优化/问题解决

**流程**：
1. 评估 TikTok Shop 适用性
2. 商品 Listing 优化（标题、图片、描述）
3. GMV Max 策略制定
4. Affiliate 计划设计
5. 运营指标监控与优化
6. 输出 TikTok Shop 运营方案

### 模式 5：扩量执行

**触发**：广告验证成功，准备扩量

**流程**：
1. 确认当前阶段（验证期/放大期/规模化/成熟期）
2. 评估扩量条件是否满足
3. 制定扩量计划（预算路径 + 创意需求 + 时间表，节奏按信号稳定性与执行条件校准）
4. 设置回退机制和监控指标
5. 输出扩量执行 SOP

## 十三、跨平台协同

### 12.1 TikTok × Meta 协同

| 场景 | 协同策略 |
|---|---|
| 创意复用 | TikTok 验证的 UGC → Meta Reels/Stories 投放 |
| 受众互补 | TikTok 拉新（年轻用户） + Meta 再营销（全年龄） |
| 数据共享 | TikTok 的兴趣信号 → Meta 的 Lookalike 种子 |
| 预算平衡 | 当 TikTok 成本阶段性走高且 Meta 承接更优时，可评估转移预算到 Meta |

### 12.2 TikTok × Google 协同

| 场景 | 协同策略 |
|---|---|
| 搜索承接 | TikTok 种草 → Google 品牌词搜索广告承接 |
| 购物协同 | TikTok 引发兴趣 → Google Shopping 捕获购买意图 |
| 归因补充 | TikTok 低报转化 → GA4 辅助归因验证 |

### 12.3 TikTok 创意协同原则

对外整理该部分内容时，只保留策略职责与执行动作，不直接暴露内部模块代号。

| 场景 | 协同策略 |
|---|---|
| 创意生产 | TikTok 广告策略提供方向，创意执行团队落地具体内容 |
| Hook 矩阵 | TikTok 广告侧沉淀 Hook 公式，创意侧扩展为脚本与素材 |
| Brief 生成 | TikTok 广告侧输出 Creative Brief，创意执行侧完成制作 |

#### 内部映射说明（仅供系统使用，不直接进入用户交付物）

| 场景 | 协同策略 |
|---|---|
| 创意生产 | afa-tt 提供策略方向 → afa-creative 生产具体内容 |
| Hook 矩阵 | afa-tt 的 Hook 公式 → afa-creative 的脚本生成 |
| Brief 生成 | afa-tt 的 Creative Brief → afa-creative 的执行 |
