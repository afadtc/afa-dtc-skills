# AFA-TT Account Setup SOP — TikTok 广告账户完整搭建手册

> **版本**: 1.0 | **适用**: TikTok Ads + TikTok Shop + Shopify 集成

---

## 一、TikTok Business Center 设置

### 1.1 创建 Business Center

```
Step 1: 访问 business.tiktok.com
Step 2: 使用企业邮箱注册（避免个人邮箱）
Step 3: 填写企业信息
  ├── 企业名称（与营业执照一致）
  ├── 行业类别
  ├── 企业规模
  └── 国家/地区
Step 4: 验证邮箱
Step 5: 完成 Business Center 创建
```

### 1.2 Business Center 架构

```
Business Center（企业中心）
├── Ad Accounts（广告账户）
│   ├── 主广告账户
│   └── 备用广告账户（建议至少 2 个）
├── TikTok Accounts（TikTok 账号）
│   └── 绑定品牌 TikTok 账号
├── Catalogs（产品目录）
│   └── 连接 Shopify 产品
├── Pixels（像素）
│   └── 网站追踪像素
├── Members（成员）
│   ├── Admin（管理员）
│   ├── Advertiser（广告主）
│   └── Analyst（分析师）
└── Assets（资产）
    ├── Creative Library（创意库）
    └── Audiences（受众库）
```

### 1.3 成员权限设置建议

| 角色 | 广告账户 | 数据查看 | 财务 | 成员管理 |
|:---|:---:|:---:|:---:|:---:|
| 创始人/CEO | 完全 | 完全 | 完全 | 完全 |
| 广告运营 | 操作 | 完全 | 只读 | 无 |
| 创意团队 | 只读 | 有限 | 无 | 无 |
| 外部代理 | 操作 | 完全 | 只读 | 无 |

## 二、TikTok Pixel 安装（Shopify）

### 2.1 方法选择

| 方法 | 难度 | 推荐度 | 说明 |
|:---|:---:|:---:|:---|
| TikTok Shopify App | ⭐ | ⭐⭐⭐⭐⭐ | 最简单，一键安装 |
| 手动代码安装 | ⭐⭐⭐ | ⭐⭐⭐ | 更灵活，需要技术能力 |
| Google Tag Manager | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 最灵活，适合多平台追踪 |

### 2.2 TikTok Shopify App 安装 SOP

```
Step 1: Shopify 后台 → Apps → 搜索 "TikTok"
Step 2: 安装 "TikTok" 官方 App
Step 3: 连接 TikTok Business Center
  ├── 登录 TikTok 账号
  ├── 选择 Business Center
  └── 授权连接
Step 4: 设置数据共享级别
  ├── Standard（标准）— 基础事件
  ├── Enhanced（增强）— 推荐 ✅
  └── Maximum（最大化）— 最全面，推荐 ✅
Step 5: 选择或创建 Pixel
Step 6: 设置产品目录同步
Step 7: 完成安装

验证安装:
├── 使用 TikTok Pixel Helper 浏览器扩展
├── 访问网站各页面，检查事件触发
├── 在 TikTok Ads Manager → Events → 查看事件状态
└── 等待 24 小时确认数据回传
```

### 2.3 关键事件配置

| 事件 | 类型 | 触发位置 | 重要性 |
|:---|:---:|:---|:---:|
| PageView | 标准 | 所有页面 | ⭐⭐⭐ |
| ViewContent | 标准 | 产品页 | ⭐⭐⭐⭐⭐ |
| AddToCart | 标准 | 加购按钮 | ⭐⭐⭐⭐⭐ |
| InitiateCheckout | 标准 | 结账页 | ⭐⭐⭐⭐ |
| CompletePayment | 标准 | 感谢页 | ⭐⭐⭐⭐⭐ |
| Search | 标准 | 搜索结果页 | ⭐⭐⭐ |
| Subscribe | 自定义 | 邮件订阅 | ⭐⭐⭐ |

### 2.4 Events API (Server-Side) 设置

```
为什么需要 Events API:
├── 浏览器端追踪受 iOS 14+ 和广告拦截器影响
├── Events API 从服务器端发送数据，不受浏览器限制
├── 双重追踪（Pixel + Events API）可提高数据准确性 30-50%
└── TikTok 会自动去重，不会重复计算

通过 Shopify App 启用:
├── Shopify 后台 → TikTok App → 设置
├── 数据共享级别选择 "Maximum"
├── 这会自动启用 Events API
└── 无需额外代码

验证:
├── TikTok Ads Manager → Events → Web Events
├── 检查每个事件的 "Match Quality Score"
├── 目标: 每个事件的匹配质量 > 6/10
└── 如果低于 6，检查是否传递了足够的用户参数
```

## 三、TikTok Shop 与 Shopify 集成

### 3.1 集成方案选择

| 方案 | 适用场景 | 优点 | 缺点 |
|:---|:---|:---|:---|
| **双轨独立运营** | 两个渠道独立 | 简单，无冲突 | 库存需手动同步 |
| **Shopify 为主，TikTok Shop 为辅** | Shopify 是主站 | 统一库存管理 | TikTok Shop 功能受限 |
| **TikTok Shop 为主** | TikTok 是主渠道 | 最大化 TikTok 生态 | 品牌建设受限 |

### 3.2 库存同步设置

```
方案 A: 使用第三方工具同步
├── 推荐工具: Syncio, Stock Sync, Sellbrite
├── 设置: 双向同步，实时更新
├── 注意: 设置安全库存缓冲（实际库存 - 10%）
└── 成本: $20-$100/月

方案 B: 手动管理
├── 适用: 产品 SKU < 50
├── 频率: 每日检查一次
├── 工具: Excel/Google Sheets 追踪
└── 风险: 可能超卖
```

### 3.3 订单管理流程

```
TikTok Shop 订单:
├── 订单在 TikTok Shop Seller Center 管理
├── 需在承诺时效内发货（通常 3-5 天）
├── 使用 TikTok Shop 支持的物流商
└── 退货在 TikTok Shop 平台处理

Shopify 订单:
├── 订单在 Shopify 后台管理
├── 使用 Shopify Shipping 或第三方物流
└── 退货按 Shopify 设置处理

统一管理建议:
├── 使用 ERP 系统统一管理（如 ShipStation）
├── 设置统一的发货流程
├── 统一客服响应标准
└── 统一退货政策（尽量一致）
```

## 四、受众创建 SOP

### 4.1 自定义受众

**网站访客受众**:
```
TikTok Ads Manager → Assets → Audiences → Create Audience → Custom Audience

推荐创建的受众:
├── 所有网站访客（180 天）— 最大再营销池
├── 产品页访客（30 天）— 高意向
├── 加购未购买（14 天）— 最高意向
├── 购买客户（180 天）— 排除/相似受众种子
├── 高价值客户（LTV Top 20%）— 最佳相似种子
└── 邮件订阅者 — 中等意向
```

**客户文件受众**:
```
上传方式: CSV 文件
必需字段: Email 或 Phone（至少一个）
推荐字段: Email + Phone + Name（提高匹配率）
最低数量: 1,000 条记录
匹配率预期: 30-60%（取决于数据质量）

推荐创建:
├── 全部客户列表
├── 高价值客户（消费 Top 20%）
├── 近期活跃客户（90 天内购买）
├── 流失客户（180 天+ 未购买）
└── 邮件订阅但未购买
```

**互动受众**:
```
基于 TikTok 平台互动创建:
├── 视频观看者（完播/部分观看）
├── 主页访问者
├── 关注者
└── 广告互动者（点击/点赞/评论/分享）

推荐创建:
├── 视频完播 75%+（7 天）— 高兴趣
├── 视频完播 25%+（30 天）— 中等兴趣
├── 广告点击者（30 天）— 高意向
└── 主页访问者（30 天）— 品牌兴趣
```

### 4.2 相似受众

```
创建优先级:
1. 基于购买客户 → 1% 相似（最精准）
2. 基于高价值客户 → 1% 相似（最高质量）
3. 基于加购客户 → 2% 相似（扩大范围）
4. 基于网站访客 → 3-5% 相似（最大覆盖）

设置建议:
├── 种子受众最低 100 人（推荐 1,000+）
├── 相似度: 1%（精准）→ 5%（宽泛）→ 10%（最宽）
├── 初始测试用 1-2%，扩量时逐步放宽
└── 定期更新种子受众（每月一次）
```

## 五、广告账户结构 SOP

### 5.1 推荐账户结构

```
广告账户
├── 🧪 测试系列（Testing Campaign）
│   ├── 目标: 转化 / 网站访问
│   ├── 预算: 日预算的 30-40%
│   ├── 广告组 1: 宽泛受众 + 创意 A
│   ├── 广告组 2: 兴趣受众 + 创意 B
│   ├── 广告组 3: 相似受众 + 创意 C
│   └── 规则: 每组 $20-$50/天，3 天后评估
│
├── 📈 扩量系列（Scaling Campaign）
│   ├── 目标: 转化
│   ├── 预算: 日预算的 40-50%
│   ├── 广告组: 测试胜出的受众+创意组合
│   └── 规则: CBO，逐步增加预算
│
├── 🔄 再营销系列（Retargeting Campaign）
│   ├── 目标: 转化
│   ├── 预算: 日预算的 10-20%
│   ├── 广告组 1: 网站访客（7 天）
│   ├── 广告组 2: 加购未购买（14 天）
│   └── 广告组 3: 视频互动者（30 天）
│
└── 🛒 TikTok Shop 系列（如适用）
    ├── GMV Max 系列
    └── Product Shopping Ads 系列
```

### 5.2 命名规范

```
系列命名: [目标]_[受众类型]_[日期]
示例: CONV_Prospecting_2026Q1

广告组命名: [受众]_[出价策略]_[预算]
示例: LAL1%_Purchasers_AutoBid_$50

广告命名: [创意类型]_[Hook类型]_[创作者]_[日期]
示例: UGC_PainPoint_Sarah_0315
```

## 六、搭建完成检查清单

```
□ Business Center
  ├── □ 企业信息完整
  ├── □ 成员权限设置正确
  ├── □ 绑定 TikTok 品牌账号
  └── □ 支付方式设置（信用卡/预充值）

□ Pixel & 追踪
  ├── □ Pixel 已安装并验证
  ├── □ 所有关键事件正常触发
  ├── □ Events API 已启用
  ├── □ Match Quality Score > 6/10
  └── □ TikTok Pixel Helper 验证通过

□ 产品目录
  ├── □ Shopify 产品已同步
  ├── □ 产品信息完整（标题/图片/价格/描述）
  └── □ 产品状态为"已审核通过"

□ 受众
  ├── □ 自定义受众已创建（至少 3 个）
  ├── □ 相似受众已创建（至少 2 个）
  └── □ 客户文件已上传（如有）

□ 广告账户
  ├── □ 账户结构已搭建
  ├── □ 命名规范已统一
  ├── □ 出价策略已确定
  └── □ 预算分配已规划

□ TikTok Shop（如适用）
  ├── □ 店铺已开通并审核通过
  ├── □ 产品已上架并优化
  ├── □ 物流和退货政策已设置
  ├── □ GMV Max 已配置
  └── □ Affiliate 计划已开启

□ 平台政策
  ├── □ 广告内容符合 TikTok 政策
  ├── □ 落地页符合要求（隐私政策/退款政策）
  └── □ 品类限制已确认（如适用）
```
