# 反模式与边界处理

## 十四-A、问题匹配检查

> **核心原则**：在开始执行任何 TikTok 广告/商业化策略之前，先判断用户的核心问题是否真的属于 TikTok 广告的能力范围。如果用户的核心痛点不是 TikTok 广告投放本身，而是下表中的某个场景，直接说明并推荐正确的模块。

| 用户核心问题信号 | 误归类原因 | 推荐转介 | 推荐话术 |
| :--- | :--- | :--- | :--- |
| TikTok 视频没流量但实际是有机内容问题而非广告问题 | 有机运营和付费投放是两个不同的学科 | 社交媒体运营 | 「你的问题是 TikTok 有机内容运营，不是广告投放。建议先制定内容策略，等有机内容跑通后再用广告放大。」 |
| TikTok 广告带来了客户但他们不复购 | 留存问题不是广告问题 | 客户留存优化 | 「TikTok 广告的职责是获客，复购需要留存体系支撑。建议通过客户留存优化做生命周期管理。」 |
| TikTok Shop 产品定价不知道怎么设 | 定价策略不是广告策略 | 客单价提升 | 「TikTok Shop 的定价策略需要综合考虑利润率、竞品和平台佣金。建议先梳理定价策略。」 |
| 想找 TikTok 网红合作（非广告投放） | 网红合作和广告投放是两个不同的学科 | 网红合作 | 「网红发现、外联、谈判和管理是网红合作的专长。如果你想把网红内容做成 Spark Ads 投放，我可以帮你处理投放部分。」 |
| TikTok 广告素材不行/没有好素材 | 素材问题不是投放问题 | 创意生成 | 「TikTok 广告效果 80% 取决于素材。建议先制作原生风格的 TikTok 素材，然后回来做投放策略。」 |
| 品牌定位不清晰，不知道 TikTok 上该传递什么 | 品牌策略先于广告策略 | 品牌定位设置 | 「建议先确定品牌定位和核心信息，然后回来制定 TikTok 广告策略。」 |

**绝对不要做的事**：不要用 TikTok 广告投放的框架去回答有机内容运营、留存、定价、网红合作等非广告问题。如果用户的核心痛点不在广告端，先解决核心痛点，再回来做广告。

## 十四-B、反模式与边界

### 13.1 反模式（Anti-Patterns）

| 反模式 | 正确做法 |
|---|---|
| ❌ 用 Meta 的创意直接投 TikTok | ✅ 为 TikTok 制作原生内容 |
| ❌ 高制作质量的品牌广告 | ✅ 真实感 > 制作质量 |
| ❌ 频繁调整预算（每天改） | ✅ 至少 48 小时观察期 |
| ❌ 只投一种创意格式 | ✅ 7-10 种格式同时测试 |
| ❌ 忽视 TikTok Shop | ✅ Ads + Shop 双轨并行 |
| ❌ 不做 Affiliate | ✅ Affiliate 是最强增长杠杆 |
| ❌ 只看 TikTok 后台归因 | ✅ 结合 GA4 + UTM 综合归因 |
| ❌ 创意疲劳后继续投放 | ✅ 提前准备替换创意 |
| ❌ 一次性大幅提升预算 | ✅ 每次 ≤ 30%，间隔 48 小时 |

### 13.2 边界声明

| 场景 | 处理方式 |
|---|---|
| 用户需要落地页优化 | → 建议通过转化率优化解决 |
| 用户需要邮件/SMS 跟进 | → 建议通过邮件营销优化 / 短信营销优化解决 |
| 用户需要 Meta 广告策略 | → 建议通过 Meta 广告优化解决 |
| 用户需要 Google 广告策略 | → 建议通过 Google 广告优化解决 |
| 用户需要深度创意生产 | → 建议通过创意生成解决 |
| 用户需要品牌策略 | → 建议通过品牌定位设置解决 |
| 用户需要全局诊断 | → 建议通过全局诊断解决 |
| 用户需要数据看板 | → 建议通过数据分析解决 |

## 降级策略与供应链模式适配

当 `supply_chain_mode = dropshipping` 时：

    提升优先级：
      │
      ├── TikTok Shop 带货（最高优先级）：
      │   • TikTok Shop 是 Dropshipping 的重要销售渠道
      │   • 短视频挂车 + 直播带货组合拳
      │   • 联盟达人带货（低风险，按成交付费）
      │
      ├── 产品测试广告（最高优先级）：
      │   • 低预算快速测试多个产品
      │   • 素材测试优先于受众测试
      │   • UGC 风格素材在 TikTok 上表现最好
      │
      └── 有机内容作为测试场（高优先级）：
          • 先发有机内容测试产品反应
          • 有机表现好的内容转为 Spark Ads
          • 降低广告测试成本

    降低优先级（但仍然提供）：
      • 复杂的再营销漏斗（复购基数不足）
      • 大规模扩量框架（利润率不支持激进扩量）
      • 品牌建设类广告（先跑通直接转化）

当 `supply_chain_mode = wholesale` 时：

    提升优先级：
      │
      ├── B2B 展示型内容（最高优先级）：
      │   • 产品目录展示、供应链能力展示、工厂实拍视频
      │   • 行业展会精彩回顾、客户合作案例视频
      │   • TikTok 作为 B2B 品牌展示窗口而非直接销售渠道
      │
      └── 行业洞察内容（高优先级）：
          • 行业趋势解读、供应链知识分享
          • 建立行业专家形象

    降低优先级（但仍然提供）：
      • TikTok Shop 带货（B2B 客户不通过 TikTok Shop 采购）
      • 冲动购买型广告（B2B 决策周期长）

当 `supply_chain_mode = manufacturing` 时：

    提升优先级：
      │
      ├── 品牌故事型内容（最高优先级）：
      │   • 工厂幕后、生产过程、质量控制视频
      │   • 制造商有最强的原创内容素材，充分利用
      │   • “Made by Us” 类型的品牌叙事
      │
      ├── 品牌建设类广告（高优先级）：
      │   • 制造商有品牌基础，可以做品牌建设型广告
      │   • 强调质量、工艺、传承等差异化价值
      │
      └── 产品教育内容（高优先级）：
          • 产品使用教程、工艺解读、材质科普
          • 通过教育内容提升产品感知价值

    降低优先级（但仍然提供）：
      • 低价促销型广告（制造商品牌应保护价格形象）
      • 快速测品型广告（产品线稳定）

  当 `supply_chain_mode = dtc` 时：
    → 保持现有所有广告策略和优先级不变（DTC 是系统的默认设计目标）

用户拒绝提供信息时（v2.3.4 收口封版）：
  当用户表达「别问了」「直接给策略」等意图时：
  → 立即停止追加盘问，不把追问当作首答前提
  → 仍然保留最小前置判断，但对缺失项只做「待验证清单」，不自动假设用户已具备落地页、追踪体系或视频素材
  → 只基于用户已明确给出的产品、市场、预算、渠道现状等事实输出「保守可执行版」
  → 若关键资产未知，使用条件化表达：
    「如果你已有落地页/Shop 承接，可直接进入测试；如果尚未具备，先完成该项再上广告。」
  → 在输出末尾附加：
    「以上策略仅基于你已提供的信息制定。未确认的基础设施（如落地页、Pixel、素材）已被列入待验证项，而不是默认视为已完成。如果你补充这些信息，我可以把方案收紧得更精准。」

执行类任务最低门槛（v2.3.4 收口封版）：
  当用户要求写广告文案、Hook 脚本等具体交付物时：
  → 唯一硬性门槛仍是需要知道「卖什么产品」
  → 其他缺失信息只能用「通用结构模板 + 显式待验证项」承接，不得把行业最佳实践写成该品牌已具备的资产、受众反馈或历史表现
  → 可以输出：
    • 通用版 Hook 框架
    • 多情境脚本（如“若已有 TikTok Shop”“若仍走独立站”）
    • 待用户补齐后可二次收紧的占位字段

当终端不支持联网时：
  → 无法抓取竞品 TikTok 内容
  → 无法获取最新 TikTok 政策更新
  → 使用内置基准数据和创意模板
  → 明确告知用户：「当前环境无法联网，竞品创意分析需要你手动提供截图或链接」

前置条件检查（v1.9 新增）：
  在输出任何广告策略前，先检查基础设施是否就绪：
  ① 是否有可用的产品页/落地页或 TikTok Shop？
     → 没有 → 「广告需要有落地页或 TikTok Shop 承接流量。建议先搭建。」
  ② 是否已安装 TikTok Pixel + Events API？
     → 没有 → 「追踪是广告的基础。建议先完成 Pixel + Events API 部署。」
  ③ 是否有最低启动预算（建议 ≥$30/天）？
     → 预算太低 → 「你的日预算低于 $30，TikTok 算法很难积累足够的学习数据。
     建议先用 TikTok 自然流量验证内容方向，再考虑付费投放。」
  ④ 是否有原生风格的视频素材？
     → 没有 → 「TikTok 广告必须是原生风格视频，传统广告图片效果极差。
     建议先通过创意生成制作 3-5 条原生视频素材。」

Level 0 边界（v1.9 新增，v1.9.2-p8 重写）：
  Level 0 用户（无产品/无网站/纯概念）同样欢迎使用本模块。
  温馨提示：「TikTok 广告投放通常需要有产品和网站才能发挥最大效果。
  以下内容基于通用框架与显式假设，当你有了产品和网站后，
  我可以给出更定制化的方案。」
  → 不拦截、不拒绝服务、不限制路由
  → 正常响应用户请求，但不得把缺失资产默认写成已存在

危机模式（v1.9 新增，v1.9.2-p8 重写）：

  季节性排除（v1.9.3 新增）：
  如果 seasonal_mode = off_season（用户已确认当前处于季节性淡季）：
  → 不自动触发危机模式（淡季销量下降是正常的）
  → 除非用户同时有非季节性危机信号（现金流断裂、账户被封等）
  → 使用 YoY（同比）而非环比来评估业绩趋势
  当 crisis_mode ≠ none 时（即 cash_crisis 或 pr_crisis），广告策略建议切换到止血模式：
  ① 立即审计当前广告花费：哪些广告在亏钱？立即暂停
  ② 只保留 ROAS > 盈亏平衡点的广告
  ③ 不建议新建广告活动（危机期不应增加支出）
  ④ 如果有 TikTok Shop，可以保留 Affiliate 带货（零风险渠道）
  ⑤ 不建议加预算、不建议测试新受众
  输出格式：
    「止血审计：
    ─ 广告组 A：ROAS X.X，低于盈亏线 Y.Y → 建议立即暂停，每天节省 $Z
    ─ Affiliate 带货：零风险渠道 → 保留并加强
    每月可节省广告费：$XXX」
  重要补充（v1.9.2-p8）：
  以上是「止血建议」的方向指引，不是「禁止用户做其他事」。
  如果用户在危机期坚持要做非止血类的事，尊重用户意愿，正常执行。```
