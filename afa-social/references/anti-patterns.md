# 反模式与边界规则

> 本文件为社交媒体引擎的内部参考文件，用于沉淀本模块的常见误区、边界规则与异常处理原则。
> 如需整理为用户可见交付物，应删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。

## 1. Dropshipping 适配 (supply_chain_mode = dropshipping)

```
提升优先级：
├── 产品演示内容（最高优先级）：
│   • 产品使用教程 / 开箱视频 / 前后对比
│   • 不需要自有仓库或生产线即可制作
│   • UGC 风格在所有平台表现最好
│
├── 社交证明内容（最高优先级）：
│   • 客户评价截图 / 用户反馈 / 使用照片
│   • 新店铺最需要的信任建设素材
│   • 鼓励客户分享使用体验（提供小激励）
│
└── 问题解决方案内容（高优先级）：
    • "你是否也有这个问题？" → 产品解决方案
    • 痛点驱动的内容在 Dropshipping 中转化率最高
    • 适合 TikTok 和 Instagram Reels 短视频格式

降低优先级（但仍然提供）：
  • 品牌叙事和创始人故事（品牌故事尚未成型）
  • 仓库打包幕后（无自有仓库时不适用）
  • 团队文化展示（团队规模较小时不适用）

当 supply_chain_mode = wholesale 时：
  提升优先级：
    • LinkedIn 内容策略（最高优先级）：B2B 采购决策者活跃在 LinkedIn，发布行业洞察、合作案例、供应链解决方案
    • 行业展会/活动内容（高优先级）：围绕行业展会创建内容，展示专业度
    • 客户证言和案例研究（高优先级）：B2B 客户更看重同行评价
  降低优先级（但仍然提供）：
    • 娱乐性/病毒式内容（B2B 客户更看重专业性）
    • 网红合作（批发客户不受网红影响）

当 supply_chain_mode = manufacturing 时：
  提升优先级：
    • 工厂幕后内容（最高优先级）：生产线实拍、质检过程、工匠故事——这是制造商独有的内容优势
    • 品牌叙事和创始人故事（高优先级）：制造商有真实的品牌故事可讲
    • 质量对比内容（高优先级）：展示自有制造 vs 代工的质量差异
  降低优先级（但仍然提供）：
    • 快速测品型内容（制造商产品线稳定，不需要频繁测品）
    • 低价促销内容（制造商品牌应保护价格形象）

当 supply_chain_mode = dtc 时：
  → 保持现有所有社媒策略和优先级不变（DTC 是系统的默认设计目标）
```

## 2. 危机模式 (crisis_mode ≠ none)

```
季节性排除：
  如果 seasonal_mode = off_season（用户已确认当前处于季节性淡季）：
  → 不自动触发危机模式（淡季销量下降是正常的）
  → 除非用户同时有非季节性危机信号（现金流断裂、账户被封等）
  → 使用 YoY（同比）而非环比来评估业绩趋势

当 crisis_mode ≠ none 时（即 cash_crisis 或 pr_crisis），社交策略应聚焦在「零成本流量」：
  ① 社交媒体是危机期最重要的免费流量渠道
  ② 加大发布频率，聚焦「可以直接带货」的内容
  ③ 利用存量粉丝做限时促销发布
  ④ 如果有 TikTok Shop / Instagram Shop，优先通过社交商店卖货
  ⑤ 不建议大规模品牌建设类内容（危机期需要直接带货）

重要补充：
  以上是「止血建议」的方向指引，不是「禁止用户做其他事」。
  如果用户在危机期坚持要做非止血类的事，尊重用户意愿，正常执行。
```

## 3. 严格禁止的操作

1. **禁止建议购买假粉或参与度群组 (Engagement Pods)**：这会破坏账号的算法标签，导致永久性限流。
2. **禁止过度依赖"品牌美学"**：在 TikTok/Reels 上，过度精美的广告级视频会被划走。原生、粗糙 (Lo-Fi) 的内容转化更好。
3. **禁止无 CTA 的内容**：电商内容通常应有明确的下一步指示（即使是软性的“关注看更新”）。
4. **禁止忽视 FTC 披露**：所有付费或赠送的 UGC 通常应包含适当的赞助披露（#ad, #sponsored）。

## 4. 常见错误纠正

- **错误**：将横屏 YouTube 视频裁剪为 9:16 发到 TikTok。
  - **纠正**：优先使用原生竖屏拍摄，遵循平台的安全区（Safe Zones），避免文字被 UI 遮挡。
- **错误**：给创作者逐字稿并要求完美背诵。
  - **纠正**：提供结构和关键信息点 (Talking points)，让创作者用自己的话表达。
- **错误**：只看播放量 (Vanity Metrics)。
  - **纠正**：引导用户关注保存率、分享率和主页访问率等高意图指标。

## 5. 边界规则

- 本 Skill 专注于**物理产品电商 (DTC)** 的社交策略。如果用户询问 B2B SaaS、本地服务或个人 IP 变现，请指出本 Skill 的专长是电商，并尽量将策略调整为适用产品销售。
- 不提供具体的规避平台审核的黑帽 (Blackhat) 技巧。
- **绝对不要做的事**：不要用有机社交内容的框架去回答付费广告、网红合作、留存策略、品牌定位、转化优化等非有机内容问题。社交媒体引擎是“内容引擎”，它的能力边界是有机内容策略和 UGC 体系。

## 6. 降级策略 (Degradation Protocol)

```
Level 1（完整数据）：社交平台数据 + 销售数据 + 内容素材库
  → 执行全维度社交电商诊断，输出完整内容策略 + 社交商店优化方案

Level 2（部分数据）：有社交账号但无销售归因数据
  → 基于社交平台公开数据进行内容和互动诊断
  → 标注"需接入社交商店数据后评估转化效果"

Level 3（最少数据）：仅有品牌和产品信息，无社交账号数据
  → 输出社交电商启动蓝图 + 平台选择建议
  → 提供内容日历模板和社交商店设置指南

终端无联网：
  → 基于用户描述生成社交内容策略框架
  → 输出离线可用的内容创作模板和发布节奏建议
```

## 7. 问题匹配检查 (Problem-Matching)

| 用户描述 | 真正问题 | 路由目标 | 建议回复 |
|---------|---------|---------|---------|
| 社交媒体没有流量 | 可能是 SEO/广告问题 | SEO 引擎 / 付费获客引擎 | 「社交媒体流量低有时不是内容问题，而是发现渠道问题。建议同时检查 SEO 和付费广告策略。」 |
| 帖子互动高但没有销售 | 内容与转化路径断裂 | 转化率优化引擎 | 「互动高但不转化，通常是从社交到购买的路径有断裂。建议用转化率优化引擎优化社交商店和落地页。」 |
| 社交媒体粉丝很多但不买东西 | 粉丝转化是转化路径问题 | 转化率优化引擎 | 「粉丝多但不转化，通常是转化路径问题。建议用转化率优化引擎优化从社交媒体到购买的全路径。」 |
| 不知道问题在哪，只是觉得社交媒体效果不好 | 需要全局诊断而非单渠道优化 | 全局诊断引擎 | 「如果不确定问题在哪，建议先用全局诊断引擎做全局诊断。」 |
