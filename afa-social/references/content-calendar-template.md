# 内容日历与运营板

> 本文件为社交增长模块的内部参考文件，用于定义 4 周滚动内容运营板、周配额校验、Organic-to-Paid 晋级标记与每周复盘区的标准结构。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。

---

## 1. 核心原则 (Core Principles)

- **一致性胜过完美**：保持稳定发布节奏通常比偶尔发布一条“完美内容”更重要。
- **内容支柱平衡**：不要只发销售内容。平衡教育、娱乐、灵感和转化内容。
- **批量生产 (Batching)**：集中时间拍摄和剪辑，提高效率。
- **敏捷响应**：留出一部分空间用于响应突发趋势和热点。
- **运营板思维**：日历不是一次性输出物，而是一个持续滚动、每周复盘迭代的运营系统。

## 2. 内容支柱 (Content Pillars)

将内容分为 4-5 个核心支柱，确保内容的多样性。

| 支柱 | 目标 | 内容类型示例 | 占比建议 |
|:---|:---|:---|:---|
| **教育 (Educate)** | 建立权威，解决痛点 | 教程、打破迷思、成分解析、"如何使用" | 参考起点：较高占比 |
| **娱乐 (Entertain)** | 提高参与度，扩大触达 | 趋势劫持、幽默短剧、幕后花絮、POV | 参考起点：较高占比 |
| **灵感 (Inspire)** | 建立情感连接，展示愿景 | 客户故事、创始人故事、生活方式展示 | 参考起点：中等占比 |
| **转化 (Convert)** | 驱动销售，促成购买 | 产品发布、限时促销、开箱、对比评测 | 参考起点：中等占比 |

> **品牌阶段调整规则**：以上比例适用于成长期品牌。对于不同阶段的品牌，按以下规则调整：
>
> | 品牌阶段 | 教育 | 娱乐 | 灵感 | 转化 |
> |:---|:---|:---|:---|:---|
> | **启动期** | 偏低到中等 | 偏高 | 偏低 | 偏高 |
> | **成长期** | 较均衡 | 较均衡 | 中等 | 中等 |
> | **成熟期** | 更趋均衡 | 更趋均衡 | 更趋均衡 | 更趋均衡 |
>
> 启动期侧重娱乐（算法友好，扩大触达）和转化（快速验证产品市场匹配）。成熟期则四柱均衡。

## 3. 品牌阶段发帖速度表 (Stage-Based Posting Velocity)

不同阶段的品牌应采用不同的发帖频率。过高的频率会导致质量下降，过低的频率会导致算法权重衰减。

| 品牌阶段 | TikTok | IG Reels | IG Stories | Facebook | Pinterest |
|:---|:---|:---|:---|:---|:---|
| **启动期** | 高频测试为主 | 中高频起步 | 稳定日更或接近日更 | 低到中频 | 稳定铺量 |
| **成长期** | 高频且更有节奏 | 中高频 | 稳定日更 | 中频 | 更高频铺量 |
| **成熟期** | 稳定高质量输出 | 稳定高质量输出 | 高频陪伴 | 中频 | 持续铺量 |

> **关键原则**：
> - 启动期追求量（测试什么有效），成熟期追求质（品牌一致性）。
> - 永远不要完全停止发布（算法惩罚不活跃账号）。
> - TikTok 和 IG Reels 是 DTC 品牌的核心战场，Pinterest 是 SEO 长尾资产。

## 4. 四周滚动运营板 (4-Week Rolling Operating Board)

这是 AFA 输出内容日历时的**标准格式**。它不是一个一次性的排期表，而是一个包含配额校验、晋级标记和复盘区的运营系统。

### 4.1 运营板结构

每周运营板包含三个区域：**排期区 (Schedule)**、**配额校验区 (Quota Check)**、**复盘区 (Weekly Review)**。

#### A. 排期区 (Schedule)

每条内容必须包含以下字段：

| 字段 | 说明 | 示例 |
|:---|:---|:---|
| **日期** | 计划发布日期 | 2026-04-13 (Mon) |
| **平台** | 发布平台 | TikTok |
| **支柱** | 内容支柱分类 | 教育 |
| **原型** | 内容原型 | 教程 (Tutorial) |
| **Hook** | 前 3 秒文案或视觉 Hook | "Stop doing X if you want Y" |
| **核心信息** | 视频主体传达的核心信息 | 产品 A 的 3 步使用方法 |
| **CTA** | 行动号召 | "Link in bio" |
| **音频** | 原声 / 热门音乐 | 原声 (Voiceover) |
| **状态** | 当前进度 | 🟡 拍摄中 |
| **负责人** | 谁负责 | @Sarah |
| **🏷️ O2P Flag** | Organic-to-Paid 晋级标记 | — (发布后填写) |

**状态标记规范**：
- ⚪ 构思中 (Ideation)
- 🟡 拍摄中 (Shooting)
- 🔵 剪辑中 (Editing)
- 🟢 待发布 (Scheduled)
- ✅ 已发布 (Published)
- 🔴 已取消 (Cancelled)

**🏷️ O2P Flag 标记规范**（发布 48 小时后填写）：

| 标记 | 含义 | 触发条件 |
|:---|:---|:---|
| 🚀 **BOOST** | 优先纳入 Spark Ads / Partnership Ads 候选池 | 保存率、分享率或评论购买意图相对同批内容显著更强 |
| 🧪 **TEST** | 值得做 Hook 变体测试后再投放 | 完播率与主页访问率有积极信号，但保存率尚未形成明确胜出 |
| ♻️ **REMIX** | 核心信息有效，需要换 Hook 或格式重新发布 | 评论情绪正向，但当前播放或承接信号仍偏弱 |
| — | 不晋级 | 不满足以上任何条件 |

#### B. 配额校验区 (Quota Check)

```

│ 总帖数：[X] 条  目标：[Y] 条/周     [✅ 达标 / ⚠️ 不足]  │
│                                                         │
│ 支柱分布：                                               │
│   教育：[X] 条 ([X%])  目标：30%    [✅ / ⚠️ 偏差 >10%]  │
│   娱乐：[X] 条 ([X%])  目标：30%    [✅ / ⚠️ 偏差 >10%]  │
│   灵感：[X] 条 ([X%])  目标：20%    [✅ / ⚠️ 偏差 >10%]  │
│   转化：[X] 条 ([X%])  目标：20%    [✅ / ⚠️ 偏差 >10%]  │
│                                                         │
│ 平台覆盖：                                               │
│   TikTok：[X] 条      目标：[Y] 条  [✅ / ⚠️]            │
│   IG Reels：[X] 条    目标：[Y] 条  [✅ / ⚠️]            │
│                                                         │
│ 敏捷预留：[X] 条 ([X%])  目标：≥20% [✅ / ⚠️]            │
└─────────────────────────────────────────────────────────┘
```

**校验规则**：
- 当任何支柱占比偏差持续偏大时，AFA 应主动提醒用户并建议调整。
- 当敏捷预留明显不足时，AFA 应提醒“趋势响应空间不足”。
- 当总帖数明显低于当前品牌阶段所需节奏时，AFA 应提示“发帖频率可能低于建议水平”。

## 5. 批量生产工作流 (Batch Production Workflow)

为了保持稳定发布，通常更适合采用批量生产模式。

### 步骤 1：构思与脚本 (Ideation & Scripting)
- 每周预留一段集中时间，批量构思一组新创意。
- 编写脚本，确定 Hook、视觉和 CTA。
- 从 `content-archetypes-library.md` 中选择适合的原型。

### 步骤 2：拍摄 (Shooting)
- 每周安排半天时间集中拍摄。
- 准备好所有产品、道具和服装。
- 按照脚本清单逐一拍摄，并为关键 Hook 预留多个变体（为后续 A/B 测试储备素材）。

### 步骤 3：剪辑与排期 (Editing & Scheduling)
- 集中剪辑视频，添加字幕和特效。
- 使用排期工具（如 Later, Buffer, 或平台原生工具）安排发布。
- 在运营板中更新状态标记。

### 步骤 4：发布后 48 小时标记 (Post-Publish Tagging)
- 每条内容发布 48 小时后，根据 O2P Flag 标记规范填写晋级标记。
- 将 🚀 BOOST 内容同步给对应的付费社媒执行链路，用于后续放量测试与素材放大。

## 6. 敏捷响应机制 (Agile Response)

- **趋势监控**：保持高频浏览 TikTok FYP 和 IG Reels，寻找相关趋势。
- **快速执行**：发现趋势后，在较短反应窗口内尽快拍摄并发布（使用运营板中的敏捷预留槽位）。
- **灵活调整**：如果某个有机内容表现极好（触发 🚀 BOOST），立即调整日历，围绕该主题制作更多衍生内容。
- **趋势评估标准**：不是所有趋势都值得跟。只跟满足以下条件的趋势：
  1. 与品牌品类或价值观相关。
  2. 趋势处于上升期（不是已经过了高峰期）。
  3. 可以在 24 小时内完成拍摄和发布。

## 7. 淡季社媒策略 (Off-Season Strategy)

> **激活条件**：当 Hub 传入 `seasonal_mode = off_season` 时，本模块自动激活。淡季的社媒目标从"驱动销售"转向"建设社区和品牌认知"。

### 7.1 淡季社媒内容策略

```
淡季社媒内容比例调整：

旺季：70% 产品/促销 + 30% 品牌/教育
淡季：30% 产品/促销 + 70% 品牌/教育/互动

淡季社媒内容类型（按优先级）：

1. 互动内容（最高优先）
   → 投票、问答、挑战、用户征集
   → 目标：维持互动率，保持算法权重

2. 教育内容
   → 产品使用技巧、品类知识、行业趋势
   → 目标：建立专业信任

3. 品牌故事
   → 创始人故事、团队日常、幕后花絮
   → 目标：建立情感连接

4. UGC 展示
   → 客户评价、使用场景、改造前后
   → 目标：社会认同积累

5. 社区建设
   → 用户故事征集、品牌大使计划、线上活动
   → 目标：从粉丝到社区成员的转化
```

### 7.2 淡季社媒 KPI 调整

| 维度 | 旺季 KPI | 淡季 KPI |
|:---|:---|:---|
| **核心指标** | 社媒驱动的收入、转化率 | 互动率、粉丝增长、UGC 产出量 |
| **内容表现** | 点击率、链接点击 | 保存率、分享率、评论数 |
| **社区** | 社群驱动的 GMV | 社群活跃度、新成员增长 |
| **发布频率** | 高频发布 | 中频但更重质量 |

### 7.3 淡季社媒发布节奏

```
淡季社媒最低维持标准：

Instagram：维持稳定 Feed 与 Story 节奏
TikTok：维持稳定视频输出
Facebook：按社区活跃度保持基础更新
Pinterest：持续铺设具备 SEO 价值的 Pin，为旺季搜索提前布局

关键原则：
  → 永远不要完全停止发布（算法惩罚不活跃账号）
  → 淡季是测试新内容格式的好时机
  → 关注"保存"和"分享"指标（比"点赞"更有价值）
  → 淡季积累的内容资产在旺季可以复用
```
