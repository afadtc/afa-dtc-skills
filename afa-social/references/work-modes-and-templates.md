# 工作模式与模板

> 本文件为社交增长模块的内部参考文件，用于沉淀社交电商策略、UGC 协同、内容日历、Organic-to-Paid 管道与社区增长的标准工作模式及模板。
> **层级声明**：本文件默认作为 **internal-only** 的内部起草模板库使用，不得整段直接粘贴给用户。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。
> **渲染规则**：本文件中的平台配比、内容数量、脚本数量、周频率、阶段节奏和任何获胜标准，默认只可作为参考区间、保守起点或待验证计划；不得写成统一健康线、固定配方或对品牌结果的确定性承诺。

## 1. 工作模式 (Work Modes)

### 模式 1：全盘社交内容策略构建 (Full Strategy Build)
**触发**：用户需要从零开始或全面重构社交策略。
**流程**：
1. **信息收集**：获取品牌、产品、受众、预算和当前状态。
2. **支柱定义**：设定适合当前品牌阶段的内容支柱及建议配比，数量与比例按平台重心和内容产能校准。
3. **平台矩阵**：确定主攻平台和次要平台。
4. **原型匹配**：从原型库中挑选足够覆盖当前目标与测试范围的格式组合。
5. **日历规划**：输出 4 周滚动运营板（格式严格遵循 `content-calendar-template.md` 第 4 章标准，包含排期区、配额校验区和复盘区）。
**输出**：`Social_Strategy_Blueprint.md`（其中第 6 章「初始 4 周运营板」必须包含配额校验和 O2P Flag 列）

### 模式 2：UGC 项目启动与管理 (UGC Program Launch)
**触发**：用户需要获取高质量的 UGC 内容。
**流程**：
1. **需求澄清**：确定所需内容范围、预算和核心产品，数量按投放与运营需求确定。
2. **创作者画像**：定义理想的微型创作者特征。
3. **简报制作**：编写包含 Do's/Don'ts 和足够支持当前拍摄决策的脚本选项的 Creator Brief。
4. **外联策略**：提供 DM/邮件触达模板和跟进序列。
5. **权利与合同**：提供使用权 (Usage Rights) 和白名单 (Whitelisting) 指南。
**输出**：`UGC_Campaign_Kit.md`

### 模式 3：爆款脚本工程 (Viral Script Engineering)
**触发**：用户需要具体的视频拍摄脚本。
**流程**：
1. **产品解构**：提取核心 USP 和痛点。
2. **Hook 构思**：生成足够支持测试决策的不同角度 Hook（如痛点、好奇、对比、利益、争议等）。
3. **结构选择**：选择最适合的脚本框架（PAS、Storytime、对比等）。
4. **脚本编写**：提供包含时间轴、视觉指示、口播词和屏幕文本的完整脚本。
**输出**：`Video_Script_Pack.md`

### 模式 4：有机转付费管道搭建 (Organic-to-Paid Setup)
**触发**：用户希望将社交内容转化为广告素材。
**流程**：
1. **指标设定**：定义识别有机获胜者的判断口径与参考信号，不把单一平台阈值写成通用硬线。
2. **测试矩阵**：设计 Hook 测试和 Body 测试的 A/B 计划。
3. **放大策略**：提供 Spark Ads / Partnership Ads 的设置指南。
4. **迭代循环**：建立从广告数据反哺有机内容创作的反馈机制。
**输出**：`Organic_to_Paid_Pipeline.md`

### 模式 5：大促/季节性内容日历 (Tentpole Campaign Planning)
**触发**：用户需要为 BFCM、情人节、母亲节等规划内容。
**流程**：
1. **时间轴倒推**：从活动日按准备复杂度、库存节奏和内容产能倒推。
2. **阶段划分**：预热期 (Tease) -> 建立列表 (Lead Gen) -> 揭晓 (Reveal) -> 紧迫感 (Urgency)。
3. **内容映射**：为每个阶段分配具体的内容原型和脚本。
**输出**：`Campaign_Content_Calendar.md`

### 模式 6：社区飞轮构建 (Community Flywheel Build)
**触发**：用户希望建立品牌社区或提高客户倡导度。
**流程**：
1. **触点分析**：梳理购后体验和社区互动触点。
2. **激励设计**：设计鼓励客户分享和生成 UGC 的机制。
3. **大使计划**：构建分层的品牌大使/联盟计划（Brand Friend -> Partner -> Ambassador）。
**输出**：`Community_Growth_Playbook.md`

## 6. 初始 4 周运营板
> 格式严格遵循 `content-calendar-template.md` 第 4 章标准。

- **周频率**：[按品牌阶段、内容产能与平台反馈填写]

### 2.2 UGC 创作者简报模板
```markdown
# 创作者简报：[活动/产品名称]

## 品牌与产品概览
- **我们是谁**：[一句话介绍]
- **产品**：[产品名称及核心 USP]
- **目标受众**：[描述]

## 视觉与美学指南
- **光线**：优先选择能稳定呈现产品质感与人物状态的自然或柔和光线。
- **背景**：优先选择干净且不干扰主体表达的环境。
- **着装**：休闲舒适，避免其他品牌大 Logo。

## 必须包含 (Do's)
- [ ] 尽量在开场快速建立产品或场景识别。
- [ ] 在适合该创意的前提下展示产品的实际使用过程。
- [ ] 优先提及 [特定痛点/功能]，但可按脚本自然度调整。

## 绝对禁止 (Don'ts)
- [ ] 不要使用美颜滤镜（我们需要真实的皮肤/效果）。
- [ ] 不要提及任何竞争对手。
- [ ] 不要像读稿子一样说话，保持自然。

## 脚本选项 (选择其一)
### 选项 A：[脚本类型，如 PAS]
- **Hook (0-3s)**：[具体台词]
- **Body (3-20s)**：[具体台词 + 视觉动作]
- **CTA (20-25s)**：[具体台词]

### 选项 B：[脚本类型，如 Storytime]
- [结构同上]

## 交付要求
- **格式**：9:16 竖屏，1080p 或 4K。
- **时长**：按平台、脚本复杂度和信息密度确定。
- **文件**：提供无字幕、无水印的原始素材 (Raw footage) + 剪辑好的版本。
```

## 3. 淡季社媒策略 (Off-Season Strategy)

> **激活条件**：当 Hub 传入 `seasonal_mode = off_season` 时，本模块自动激活。淡季的社媒目标从"驱动销售"转向"建设社区和品牌认知"。

### 3.1 淡季社媒内容策略

```
淡季社媒内容比例调整：

旺季：产品/促销内容占比较高，但按库存承接、活动强度和用户疲劳校准
淡季：品牌/教育/互动内容占比较高，但按阶段目标和内容产能校准

淡季社媒内容类型（按优先级）：

1. 互动内容（常可优先考虑）
   → 投票、问答、挑战、用户征集
   → 目标：维持互动率，保持算法权重

2. 教育内容
   → 产品使用技巧、品类知识、行业趋势
   → 目标：建立专业信任

3. 品牌故事
   → 创始人故事、团队日常、幕后花絮
   → 目标：建立情感连接

4. UGC 展示
   → 客户评价、使用场景、改造前后
   → 目标：社会认同积累

5. 社区建设
   → 用户故事征集、品牌大使计划、线上活动
   → 目标：从粉丝到社区成员的转化
```

### 3.2 淡季社媒 KPI 调整

| 维度 | 旺季 KPI | 淡季 KPI |
|------|---------|---------|
| **核心指标** | 社媒驱动的收入、转化率 | 互动率、粉丝增长、UGC 产出量 |
| **内容表现** | 点击率、链接点击 | 保存率、分享率、评论数 |
| **社区** | 社群驱动的 GMV | 社群活跃度、新成员增长 |
| **发布频率** | 按活动强度、产能和平台节奏安排 | 按质量优先、名单耐受度和平台节奏安排 |

### 3.3 淡季社媒发布节奏

```
淡季社媒维持建议：

Instagram：保持对核心版位的稳定更新，具体频率按内容产能与互动质量安排
TikTok：保持测试所需的稳定更新，具体频率按创作能力与反馈质量安排
Facebook：按社区活跃度和内容回收价值安排
Pinterest：按搜索资产积累需求与素材储备安排

关键原则：
  → 尽量避免长时间完全停更，但是否降频需结合内容产能、账号状态与资源投入判断
  → 淡季是测试新内容格式的好时机
  → 关注“保存”和“分享”等更能反映内容沉淀价值的指标，但需结合平台特性综合判断
  → 淡季积累的内容资产在旺季可以复用
```
