# 诊断系统

> 本文件为短信营销模块的内部参考文件，用于沉淀短信留存中的结构化诊断模式与恢复路径。
> 如需整理为用户可见交付物，必须删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。

---

## 1. 退订率异常诊断

```text
退订率是否明显高于该品牌当前可接受区间？
├── YES → 检查发送频率
│   ├── Campaign 频率是否明显偏高？
│   │   └── YES → 🔧 下调发送频率，并在后续观察窗口内复核退订变化
│   │   └── NO → 检查文案质量
│   │       ├── 是否缺少品牌识别信息？→ 🔧 让用户在开头就能识别发件品牌
│   │       ├── 是否过度使用大写/emoji？→ 🔧 回归简洁专业风格
│   │       └── 是否每条都在推销？→ 🔧 引入对话式/价值型内容，降低连续促销密度
│   └── 检查列表质量
│       ├── 新订阅者在早期窗口内的退订率是否异常偏高？
│       │   └── YES → 🔧 弹窗授权说明不够清晰，用户不知道会收到营销短信
│       └── 老订阅者突然退订激增？
│           └── YES → 🔧 检查最近一次 Campaign 的内容/时间/频率是否异常
└── NO → 退订率暂未见异常，继续监控
```

## 2. 点击率下降诊断

```text
点击率是否明显低于该品牌当前可接受区间？
├── YES → 检查链接质量
│   ├── 链接是否被缩短器破坏？→ 🔧 使用品牌短链（如 brand.link/xxx）
│   ├── 落地页是否移动端友好？→ 🔧 短信流量通常以移动端为主，应优先保证移动端体验
│   └── 链接是否指向正确页面？→ 🔧 直接链到产品页，不要链到首页
│   检查文案 CTA
│   ├── CTA 是否清晰？→ 🔧 使用明确动作导向的 CTA，并与当前活动目标一致
│   ├── 价值主张是否明确？→ 🔧 用户应在极短阅读时间内知道“这条短信对我有什么好处”
│   └── 是否存在决策疲劳？→ 🔧 每条 SMS 以单一主 CTA 为主，避免给过多选择
└── NO → 点击率健康，关注转化率
```

## 3. SMS 渠道 ROI 衰减诊断

```text
SMS 渠道 RPR (Revenue Per Recipient) 持续下降？
├── YES → 检查受众质量
│   ├── 活跃订阅者占比是否明显偏低？
│   │   └── YES → 🔧 执行日落策略，清理不活跃用户，提升整体质量
│   └── 新订阅者质量下降？
│       └── YES → 🔧 审查订阅来源，关闭低质量渠道
│   检查 Flow 健康度
│   ├── 弃购 Flow 转化率是否明显偏弱？→ 🔧 检查触发时间/文案/折扣策略
│   ├── 欢迎 Flow 转化率是否明显偏弱？→ 🔧 检查折扣兑现速度/文案吸引力
│   └── 所有 Flow 同时下降？→ 🔧 可能是技术问题（追踪失效/链接断裂）
│   检查竞争环境
│   └── 是否处于 BFCM 后的"短信疲劳期"？
│       └── YES → 🔧 降低频率 + 切换到对话式/价值型内容，并在后续恢复窗口内复核
└── NO → ROI 健康，继续优化
```

## 4. ICE 优先级排序框架

所有 SMS 优化建议应通过 ICE 评分排序后输出：

| 维度 | 评分标准 | 评分方式 |
|------|----------|----------|
| **Impact（影响力）** | 该优化对 SMS 营收/互动率的预期改善幅度 | 使用相对高/中/低刻度或内部连续评分 |
| **数据基础（Data Basis）** | 基于数据/案例支撑的把握程度 | 使用相对高/中/低刻度或内部连续评分 |
| **Ease（实施难度）** | 实施所需的资源和时间（越容易分越高） | 使用相对高/中/低刻度或内部连续评分 |

### SMS 专属评分指南

- **Impact 高**：预期影响面较大，通常涉及核心 Flow 重构或细分策略升级
- **Impact 中**：预期影响中等，通常涉及文案优化、发送时间测试等调整
- **Impact 低**：预期影响相对有限，常见于 CTA 措辞微调、emoji 测试等局部改动
- **Data Basis 高**：有较强的 A/B 测试数据或稳定证据支持
- **Data Basis 中**：有一定行业基准或历史样本支撑
- **Data Basis 低**：主要基于最佳实践推测
- **Ease 高**：平台内配置即可完成，执行阻力较低
- **Ease 中**：需要文案创作或细分策略设计，执行复杂度中等
- **Ease 低**：需要技术集成或复杂系统配置，执行阻力较高

**ICE 总分 = Impact × Data Basis × Ease / 归一化系数**，按总分降序排列输出；若证据不足，应同步标注这是条件化排序而非确定性结论。若采用高/中/低刻度，应先在内部映射为连续评分后再计算。
