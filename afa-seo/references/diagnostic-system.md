# 诊断系统

> 本文件为 SEO 增长模块的内部参考文件，用于沉淀自然搜索增长中的结构化诊断模式与恢复路径。
> 如需整理为用户可见交付物，应删除内部路由标签、模块代号和系统字段，只保留自然语言、业务角色与行动建议。

---

## 诊断模式 1：自然流量下降

```
触发条件：GSC 或 GA 数据显示自然搜索流量进入明显异常下滑区间（相对近期基线）

决策树：

1. 确认下降的性质
   ├── 突然断崖式下降（在很短观察窗口内出现异常大幅下滑）？
   │   ├── 检查：是否有技术故障（误加 noindex, robots.txt 变更）
   │   ├── 检查：是否被黑客攻击（GSC 安全问题报告）
   │   ├── 检查：是否有 Google 核心算法更新（搜索 "Google algorithm update"）
   │   ├── 检查：是否有服务器宕机（检查服务器日志）
   │   └── 检查：是否有大规模 301 重定向错误
   │
   ├── 逐步缓慢下降（数周/数月）？
   │   ├── 检查：竞争对手是否在崛起（Ahrefs 竞品排名变化）
   │   ├── 检查：搜索需求是否在变化（Google Trends）
   │   ├── 检查：内容是否过时（最后更新时间）
   │   ├── 检查：是否丢失了重要外链（Ahrefs Lost Backlinks）
   │   └── 检查：是否有内部链接结构变化
   │
   └── 周期性波动？
       ├── 对比历史同期数据（YoY）
       ├── 确认是否为季节性趋势
       └── 如果是季节性 → 不是问题，调整预期

2. 定位下降的范围
   ├── 全站下降？ → 技术问题 / 核心算法更新 / 品牌词下降
   ├── 特定目录下降？ → 该目录的内部链接 / 内容质量 / 特定关键词
   ├── 特定页面下降？ → 该页面的排名变化 / 竞品新内容
   └── 特定设备/地区下降？ → 移动端体验 / 本地化设置

3. 检查核心指标（GSC 数据）
   ├── 曝光量 (Impressions) 下降？ → 排名下降或搜索需求减少
   ├── 点击率 (CTR) 下降？ → SERP 展现变化（更多广告/AI 摘要）或 Meta 标签被重写
   ├── 排名 (Positions) 下降？ → 竞品优化更好 / 内容失去相关性 / 外链丢失
   └── 索引页面数下降？ → 爬取问题 / 页面被移除索引

4. 制定恢复计划
   ├── 技术修复（紧急）：修复 404、重定向循环、抓取错误、noindex 误设
   ├── 内容更新（短期执行）：刷新过时内容，增加深度，提升 E-E-A-T
   ├── 链接恢复（中期执行）：联系丢失链接的网站，开展新的外链活动
   └── 长期策略（持续推进）：加强主题权威建设，扩展内容集群

输出：流量下降诊断报告（含时间线、根因分析、恢复行动计划）
```

## 诊断模式 2：关键词排名下滑

```
触发条件：特定关键词排名从原有高可见位置明显下滑

决策树：

1. 确认排名变化的范围
   ├── 单个关键词下滑？ → 该关键词的竞争格局变化
   ├── 一组相关关键词下滑？ → 该主题集群的整体问题
   └── 大量不相关关键词下滑？ → 全站问题（技术/算法）

2. 分析竞争变化
   ├── 新的竞争对手出现？（检查当前排名前 10 的页面）
   ├── 现有竞品更新了内容？（检查竞品页面的更新时间）
   ├── SERP 特征变化？（是否新增了 AI Overview、Featured Snippet）
   └── 搜索意图变化？（Google 是否改变了对该查询的理解）

3. 检查自身页面
   ├── 页面内容是否过时？
   ├── 页面是否有技术问题（加载速度、移动端体验）？
   ├── 内部链接是否减少？
   ├── 是否有新页面与之产生内部竞争（关键词蚕食）？
   └── 页面的外链是否减少？

4. 恢复策略
   ├── 内容刷新：更新数据、添加新段落、改善结构
   ├── 意图对齐：确保页面内容匹配当前的搜索意图
   ├── 内部链接强化：从其他高权重页面添加链接
   ├── 解决蚕食：合并竞争页面或明确区分目标关键词
   └── 外链建设：针对该页面开展定向外链活动

输出：关键词排名恢复方案（含竞品分析、页面诊断、行动清单）
```

## 诊断模式 3：索引问题

```
触发条件：页面未被 Google 索引，或索引数量异常

决策树：

1. 确认索引状态
   ├── GSC → URL 检查工具：检查特定页面的索引状态
   ├── site:example.com 搜索：确认索引页面总数
   └── GSC → 覆盖率报告：查看索引错误和排除原因

2. 常见索引问题
   ├── "已发现 - 尚未编入索引"：
   │   ├── 原因：爬取预算不足，页面质量低
   │   └── 修复：提升内部链接，提交 sitemap，提高内容质量
   │
   ├── "已抓取 - 尚未编入索引"：
   │   ├── 原因：Google 认为页面质量不够高
   │   └── 修复：提升内容深度和独特性，增加 E-E-A-T 信号
   │
   ├── "被 noindex 标记排除"：
   │   ├── 原因：页面有 noindex 标签
   │   └── 修复：移除误设的 noindex（如果页面应该被索引）
   │
   ├── "被 robots.txt 屏蔽"：
   │   ├── 原因：robots.txt 屏蔽了该路径
   │   └── 修复：更新 robots.txt 允许爬取
   │
   └── "重复页面 - 已提交的 URL 未被选为规范网址"：
       ├── 原因：Google 选择了不同的规范版本
       └── 修复：检查 canonical 标签设置，确保指向正确版本

3. 批量索引问题
   ├── 大量页面未索引 → 检查 sitemap 完整性 + 内部链接结构
   ├── 新页面索引缓慢 → 提交 sitemap + 从高权重页面添加链接
   └── 页面被移除索引 → 检查是否有 noindex 变更或内容质量问题

输出：索引问题诊断报告（含问题页面清单、原因分析、修复步骤）
```

## 诊断模式 4：技术 SEO 问题

```
触发条件：网站出现技术性 SEO 问题（速度慢、抓取错误、结构化数据错误）

决策树：

1. 速度问题
   ├── PageSpeed Insights 评分 < 50？
   │   ├── 检查 LCP：首屏大图是否优化？
   │   ├── 检查 INP：JavaScript 执行是否过慢？
   │   ├── 检查 CLS：是否有布局偏移？
   │   └── 检查第三方脚本：是否安装了过多 Shopify App？
   └── 输出：速度优化清单（按影响排序）

2. 抓取错误
   ├── GSC → 抓取统计信息：检查抓取频率和错误率
   ├── 常见错误：404、5xx、重定向循环、DNS 错误
   ├── 修复：设置 301 重定向、修复服务器错误、更新内部链接
   └── 输出：抓取错误修复清单

3. 结构化数据错误
   ├── GSC → 增强功能：检查 Schema 错误和警告
   ├── 常见错误：缺少必填字段、格式错误、数据不一致
   ├── 修复：更新 Schema 标记，确保数据准确
   └── 输出：结构化数据修复清单

4. 移动端问题
   ├── Google Mobile-Friendly Test：检查移动端兼容性
   ├── 常见问题：文字太小、点击目标太近、内容超出屏幕
   ├── 修复：响应式设计调整
   └── 输出：移动端优化清单

输出：技术 SEO 审计报告（含问题清单、优先级排序、修复代码/步骤）
```

## 诊断模式 5：内容表现差

```
触发条件：博客/内容页面发布后未获得预期的流量或排名

决策树：

1. 检查基础问题
   ├── 页面是否已被索引？（URL 检查工具）
   ├── 页面是否有技术问题？（速度、移动端）
   ├── Meta 标签是否优化？（Title、Description）
   └── 内部链接是否充足？（应有来自相关高权重页面的自然支持）

2. 检查内容质量
   ├── 内容是否匹配搜索意图？（对比 SERP 前 10 的内容类型）
   ├── 内容深度是否足够？（字数、覆盖面与竞品对比）
   ├── E-E-A-T 信号是否充分？（经验、专业、权威、可信）
   ├── 内容是否有独特价值？（原创数据、独特视角、第一手经验）
   └── 内容结构是否清晰？（H2/H3 层次、列表、表格）

3. 检查竞争环境
   ├── 目标关键词的竞争度是否过高？
   ├── 是否有更权威的竞品占据了排名？
   ├── SERP 是否被 AI Overview 或其他特征占据？
   └── 是否存在关键词蚕食（多个页面竞争同一关键词）？

4. 优化策略
   ├── 内容升级：增加深度、更新数据、添加多媒体
   ├── 关键词调整：如果竞争过高，转向更长尾的变体
   ├── 内部链接强化：从高流量页面添加链接
   ├── 外链建设：针对该内容开展推广活动
   └── 重新定位：如果意图不匹配，重写以匹配当前 SERP

输出：内容优化方案（含问题诊断、竞品对比、优化行动清单）
```

## 诊断模式 6：链接配置异常

```
触发条件：外链数量异常变化，或怀疑存在有害链接

决策树：

1. 检查链接配置现状
   ├── Ahrefs/GSC：查看总外链数量和趋势
   ├── 新增链接：是否有大量低质量链接突然出现？
   ├── 丢失链接：是否有重要链接最近丢失？
   └── 锚文本分布：是否过度优化（精确匹配关键词占比过高）？

2. 识别有害链接
   ├── 来自链接农场或 PBN 的链接
   ├── 来自不相关行业的大量链接
   ├── 来自被 Google 惩罚的网站
   ├── 使用过度优化锚文本的链接
   └── 来自同一 IP 段的大量链接

3. 处理策略
   ├── 有害链接 → 使用 Google Disavow Tool 否认
   ├── 丢失的高质量链接 → 联系网站管理员恢复
   ├── 锚文本过度优化 → 通过新的自然链接稀释
   └── 链接增长异常 → 监控并准备 Disavow

输出：链接配置诊断报告（含链接质量评估、有害链接清单、修复建议）
```

## ICE 优先级排序框架

> **当诊断出多个 SEO 问题时，应使用 ICE 或同等优先级框架排序，告诉用户先做什么。**

```
每个诊断出的 SEO 问题，都用 ICE 框架评分：

I - Impact（影响力）：修复后对自然搜索流量/收入的影响有多大？
  10 = 直接影响核心流量或收入路径（如修复全站 noindex）
  7-9 = 明显影响关键页面或主题集群（如修复 Core Web Vitals）
  4-6 = 影响局部页面或次级体验（如优化关键页面 Meta）
  1-3 = 边际改善或辅助项（如图片 Alt 标签优化）

C - 数据基础（Data Basis）：我们有多确定这个问题是真实存在的？
  10 = 有 GSC/Ahrefs 数据明确支撑
  7-9 = 有间接数据支撑（如竞品对比）
  4-6 = 基于 SEO 行业最佳实践判断
  1-3 = 纯假设，需要测试验证

E - Ease（容易度）：执行这个方案有多容易？
  10 = 很快可完成（修复 Meta 标签、添加 Schema）
  7-9 = 短期内可完成（内容刷新、内链优化）
  4-6 = 需要中期协作（新内容创作、外链建设）
  1-3 = 需要较长周期（网站重构、域名迁移）

ICE 总分 = I × C × E / 10

优先级分类：
  ICE 高分段 = 🔴 优先执行（先进入当前行动序列）
  ICE 中高分段 = 🟡 规划执行（纳入近期排期）
  ICE 中低分段 = 🟢 条件具备时执行
  ICE 低分段 = ⚪ 暂缓（当前优先级较低）
```
