## 一、KPI 体系

### 1.1 核心 KPI（按重要性排序）

```
Tier 1：商业成果指标
├── 自然搜索收入（Organic Revenue）
├── 自然搜索转化率（Organic Conversion Rate）
├── 自然搜索订单数（Organic Orders）
└── 自然搜索新客获取成本（Organic CAC）

Tier 2：流量与可见性指标
├── 自然搜索会话数（Organic Sessions）
├── 非品牌自然搜索会话数（Non-Brand Organic Sessions）
├── 自然搜索曝光量（Impressions）
├── 自然搜索点击率（CTR）
└── 自然搜索平均排名（Average Position）

Tier 3：技术健康指标
├── Core Web Vitals 评分（LCP, INP, CLS）
├── 索引覆盖率（Indexed Pages / Total Pages）
├── 抓取错误率（Crawl Errors / Total Crawls）
├── 移动端可用性评分
└── 结构化数据有效率

Tier 4：权威与增长指标
├── 域名权威度（Domain Rating / Domain Authority）
├── 引用域名数量（Referring Domains）
├── 主题集群覆盖率（已创建集群内容 / 计划集群内容）
├── 关键词排名分布（Top 3 / Top 10 / Top 20）
└── 内容刷新率（已刷新内容 / 需刷新内容）
```

### 1.2 ROI 计算方法

```
SEO ROI 计算：

公式：SEO ROI = (自然搜索收入 - SEO 投入成本) / SEO 投入成本 × 100%

SEO 投入成本包括：
├── 内容创作成本（写手、编辑、设计）
├── 工具成本（Ahrefs, SEMrush, etc.）
├── 技术优化成本（开发人员时间）
├── 链接建设成本（数字公关、内容推广）
└── 管理成本（SEO 经理/顾问时间）

等效广告价值计算：
├── 公式：等效价值 = Σ (每个关键词的月搜索量 × CTR × CPC)
├── 含义：如果通过 Google Ads 获得相同的流量需要花多少钱
├── 用途：向管理层展示 SEO 的价值
└── 工具：Ahrefs Traffic Value

时间维度：
├── SEO 投资回报通常需要 6-12 个月才能显现
├── 但一旦建立，流量成本趋近于零（复利效应）
├── 对比 PPC：停止投放即停止流量
└── 建议：用 12-24 个月的时间窗口计算 ROI
```

### 1.3 行业基准参考

```
DTC 电商 SEO 基准数据（2025-2026）：

流量指标：
├── 自然搜索占总流量比例：按站点阶段、渠道结构和品牌词依赖度观察参考区间
├── 非品牌搜索占自然流量比例：结合主题权威、品牌成熟度和收录结构判断
├── 自然搜索 CTR（平均）：结合排名位置、SERP 结构和查询意图判断
└── 自然搜索跳出率：结合内容相关性、页面类型和分析口径判断

排名指标：
├── 核心商业词 Top 10 占比：结合站点成熟度、词库结构和竞品格局判断
├── 长尾词 Top 10 占比：结合内容规模、索引结构和意图匹配判断
├── Featured Snippet 获取率：结合 SERP 结构与问答型内容覆盖判断
└── AI Overview 被引用率：行业新兴指标，暂无统一基准

技术指标：
├── LCP / INP / CLS：优先以当前主流 Web Vitals 口径和真实页面分布判断，不把单页理想值直接写成全站结论
├── 索引覆盖率：结合站点结构、可索引策略和页面价值判断
└── 移动端可用性：以关键模板与核心页面的可用性问题为主，不将单一通过率直接写成唯一健康线

转化指标：
├── 自然搜索转化率：结合页面类型、查询意图和品牌阶段判断
├── 自然搜索 AOV：可与付费流量做相对比较，但不写成统一倍数
├── 自然搜索客户 LTV：可与付费流量做相对比较，但不写成统一倍数
└── SEO 等效广告价值：按词库规模、点击率假设与 CPC 结构估算

链接指标：
├── 域名权威度 (DR)：结合站点年龄、品类竞争度和历史积累判断
├── 引用域名增长：按站点阶段与外链策略观察增长质量
├── 锚文本分布：保持自然且与品牌风险控制一致
└── 有害链接比例：以异常增长和风险模式识别为主，不写成统一红线
```

## 二、工作模式

### 模式 1：SEO 全站审计 (Full SEO Audit)

```
触发条件：
├── "帮我检查一下网站的 SEO"
├── "为什么我的网站不收录"
├── "做一个 SEO 体检"
├── 新客户入驻时的初始审计
└── 全链路诊断引擎转发的 SEO 诊断任务

输入要求：
├── 必须：网站 URL
├── 推荐：Google Search Console 访问权限或数据截图
├── 推荐：Google Analytics 数据
├── 可选：竞品网站列表
└── 可选：当前关注的关键词

执行流程：
├── 1. 技术 SEO 审计（参考 references/technical-seo-checklist.md）
│   ├── 爬取与索引检查
│   ├── Core Web Vitals 检查
│   ├── 结构化数据检查
│   ├── 移动端兼容性检查
│   └── 安全性检查（HTTPS, 恶意软件）
│
├── 2. 页面 SEO 审计
│   ├── 首页 Meta 标签和内容
│   ├── 核心分类页优化状态
│   ├── 产品页 SEO 完整性
│   ├── 博客内容质量和结构
│   └── 内部链接架构
│
├── 3. 内容审计
│   ├── 现有内容的主题覆盖度
│   ├── 内容质量评估（E-E-A-T）
│   ├── 内容新鲜度（最后更新时间）
│   ├── 重复内容检查
│   └── 内容差距分析
│
├── 4. 外链审计
│   ├── 域名权威度
│   ├── 引用域名数量和质量
│   ├── 锚文本分布
│   ├── 有害链接检查
│   └── 与竞品的链接差距
│
└── 5. 竞品对标
    ├── 足够代表当前格局的直接竞品 SEO 表现
    ├── 关键词重叠和差距
    ├── 内容策略对比
    └── 链接配置对比

输出格式：
├── SEO 审计报告（按优先级排序的问题清单）
├── 每个问题包含：问题描述 + 影响评估 + 修复方案 + 预期效果
├── 优先级分级：P0（紧急）→ P1（重要）→ P2（改进）→ P3（优化）
└── 阶段性 SEO 行动路线图
```

### 模式 2：关键词研究与主题规划 (Keyword Strategy)

```
触发条件：
├── "我应该写什么内容"
├── "帮我找一些有价值的关键词"
├── "做一个关键词研究"
├── "规划内容策略"
└── 新品类/新市场的内容规划

输入要求：
├── 必须：品牌定位（来自 brand-brain）
├── 必须：核心产品/品类
├── 推荐：目标受众画像
├── 推荐：3-5 个竞品网站
└── 可选：现有内容清单

执行流程：
├── 1. 种子词发现（5 大渠道）
├── 2. 六圈扩展法（系统性扩展）
├── 3. 搜索意图分类（四大意图）
├── 4. 关键词集群化（主题分组 + 意图细分）
├── 5. 优先级矩阵评估（四维评分）
├── 6. 主题集群规划（支柱页面 + 集群内容映射）
└── 7. 内容路线图生成（按优先级排序的内容生产计划）

输出格式：
├── 关键词研究计划（结构化 JSON 或 Markdown 表格）
│   ├── 关键词集群（含搜索量、难度、意图、优先级）
│   ├── 竞品差距分析
│   └── 内容路线图
├── 主题集群地图（支柱页面 + 集群内容的可视化映射）
└── 内容日历（按月/季度的内容生产计划）
```

### 模式 3：页面级内容优化 (On-Page Optimization)

```
触发条件：
├── "帮我优化这个分类页"
├── "为这个产品写 SEO 描述"
├── "优化这个博客文章的 SEO"
├── "写一个 SEO 友好的页面内容"
└── 新页面创建时的 SEO 内容生成

输入要求：
├── 必须：页面类型（分类页/产品页/博客）
├── 必须：目标关键词或产品/分类信息
├── 推荐：页面 URL（如已存在）
├── 推荐：品牌语调指南（来自 brand-brain）
└── 可选：竞品页面 URL

执行流程：
├── 分类页优化：
│   ├── 分析目标关键词的搜索意图
│   ├── 设计 3 层架构内容（Meta + 页面内容 + 内部链接）
│   ├── 撰写 Meta Title 和 Description
│   ├── 生成首屏描述和底部购买指南
│   ├── 编写 FAQ 区域
│   ├── 生成 FAQPage + BreadcrumbList Schema
│   └── 提供内部链接建议
│
├── 产品页优化：
│   ├── 分析交易型长尾关键词
│   ├── 撰写 Meta Title 和 Description
│   ├── 生成 500-1,500 字的产品描述（利益导向）
│   ├── 编写 FAQ 区域（5-8 个问题）
│   ├── 提供图片 Alt 文本建议
│   ├── 生成增强 Product Schema
│   └── 提供内部链接和蚕食检查
│
└── 博客内容优化：
    ├── 分析搜索意图和 SERP 竞品
    ├── 设计文章结构（H2/H3 大纲）
    ├── 撰写 Meta Title 和 Description
    ├── 生成内容简报（参考 content-brief schema）
    ├── 提供内部链接建议
    └── 提供 CTA 放置建议

输出格式：
├── 页面 SEO 优化方案（即用级内容包）
│   ├── Meta 标签（Title + Description + URL Slug）
│   ├── 页面内容（H1 + 首屏描述 + 深度内容 + FAQ）
│   ├── Schema 标记（JSON-LD 代码）
│   └── 内部链接建议
└── 内容简报（如果是博客内容，输出结构化的写作指令）
```

### 模式 4：流量诊断与恢复 (Traffic Diagnosis)

```
触发条件：
├── "我的自然流量下降了"
├── "排名掉了怎么办"
├── "为什么搜索不到我的网站"
├── "Google 更新后流量下降"
└── 全链路诊断引擎转发的流量异常诊断

输入要求：
├── 必须：流量下降的大致时间点
├── 推荐：GSC 数据截图（曝光量、点击量、排名变化）
├── 推荐：受影响的页面或关键词
├── 可选：Google Analytics 数据
└── 可选：最近的网站变更记录

执行流程：
├── 应用诊断模式 1-6 的决策树
├── 排除外部因素（算法更新、季节性、行业变化）
├── 定位根本原因（技术/内容/链接/竞争）
├── 评估影响范围和严重程度
└── 制定分步恢复策略

输出格式：
├── 流量下降诊断报告
│   ├── 下降时间线和范围
│   ├── 根因分析（含证据链）
│   ├── 影响评估（受影响的页面、关键词、收入）
│   └── 分步恢复行动计划（按优先级和时间线排序）
└── 监控计划（恢复期间需要跟踪的指标和频率）
```

### 模式 5：内容策略规划 (Content Strategy)

```
触发条件：
├── "帮我规划内容策略"
├── "我应该写什么博客"
├── "做一个内容日历"
├── "规划程序化 SEO"
└── 季度/年度内容规划

输入要求：
├── 必须：品牌定位和核心产品
├── 必须：关键词研究结果（或在此模式中同时执行）
├── 推荐：现有内容清单
├── 推荐：竞品内容分析
└── 可选：预算和资源约束

执行流程：
├── 1. 内容审计：评估现有内容的表现和差距
├── 2. 主题集群规划：设计支柱页面和集群内容体系
├── 3. 内容类型规划：教育指南、对比评测、案例研究、行业趋势

├── 5. 内容日历：按月/季度排列内容生产计划
├── 6. 刷新计划：识别需要更新的现有内容
└── 7. 资源分配：估算每篇内容的工作量和成本

输出格式：
├── 内容策略文档
│   ├── 主题集群地图
│   ├── 内容类型分配
│   ├── pSEO 方案（如适用）
│   ├── 12 个月内容日历
│   ├── 内容刷新计划
│   └── 资源和预算估算
└── 内容简报模板（用于每篇内容的执行）
```

## 四、淡季 SEO 优化策略

> **激活条件**：当 Hub 传入 `seasonal_mode = off_season` 时，本模块自动激活。淡季是低成本建设 SEO 资产的最佳窗口——SEO 的效果需要 3-6 个月才能显现，淡季投入的 SEO 工作会在旺季开花结果。

### 4.1 淡季 SEO 核心策略

```
淡季 SEO 三大方向：

1. 内容建设（最高优先）
   → 淡季有更多时间创作高质量内容
   → 针对旺季搜索词提前布局内容
   → 内容类型：购买指南、产品对比、使用教程、品类知识
   → 目标：旺季来临时内容已被索引并获得排名

2. 技术 SEO 优化
   → 淡季流量低 = 技术改动影响最小
   → 网站速度优化（Core Web Vitals）
   → 内部链接结构优化
   → Schema Markup 完善
   → 移动端体验优化

3. 产品页 SEO 刷新
   → 更新产品标题和描述（融入季节性关键词）
   → 优化产品图片 Alt 标签
   → 添加 FAQ Schema
   → 更新过期的促销信息和季节性内容
```

### 4.2 季节性关键词策略

| 阶段 | 关键词策略 | 示例（泳装品牌） |
|------|----------|----------------|
| **淡季初期** | 长尾信息词 + 品类教育 | "如何选择适合自己的泳装" |
| **淡季中期** | 提前布局旺季关键词 | "2026 夏季泳装趋势" |
| **预热期** | 购买意图词 + 品牌词 | "最好的比基尼品牌推荐" |
| **旺季** | 高转化商业词 | "比基尼 在线购买 包邮" |

```
淡季关键词研究流程：

1. 用 Google Search Console 分析去年同期的搜索数据
   → 找到淡季仍有搜索量的关键词
   → 识别旺季前搜索量开始上升的时间点

2. 用 Google Trends 分析品类的季节性搜索模式
   → 确定搜索量开始上升的月份
   → 提前 3 个月开始创建对应内容

3. 竞品内容差距分析
   → 竞品排名但你没有覆盖的关键词
   → 淡季是填补内容差距的好时机
```

### 4.3 淡季内容日历

```
淡季 SEO 内容生产节奏：

每月产出目标：
  ├── 2-4 篇深度博客文章（1500-3000 字）
  ├── 1 个购买指南或产品对比页面
  ├── 5-10 个产品页 SEO 刷新
  └── 1 次技术 SEO 审计和修复

内容优先级：
  1. 旺季高搜索量关键词的内容（提前布局）
  2. 常青内容（全年有搜索量的主题）
  3. 品类教育内容（建立专业权威）
  4. 客户常见问题（FAQ 页面）
```
